Part I: Mistake-Bounded Learning (Pages 1–11)

1.1 The ML Workflow and the Mistake-Bounded Protocol (Pages 1–2)

Machine learning fits a model to data. The standard workflow has six components:

  • Algorithms (problem setup) \(\rightarrow\) Training Data \(\rightarrow\) M.L. Algorithm \(\rightarrow\) Model \(\rightarrow\) Eval Model \(\rightarrow\) Validation.

Learning is formalized via bounds on failure: we want to guarantee that the number of mistakes (or the generalization error) is limited.

In mistake-bounded (online) learning, the setting is interactive. The protocol is:

  • Instance \(x\) \(\rightarrow\) learner outputs Prediction \(\widehat{y}\) \(\rightarrow\) oracle reveals Label \(y\) \(\rightarrow\) learner may Update state.

If \(\widehat{y} = y\), the state is unchanged. If \(\widehat{y} \ne y\), we count a mistake and the learner updates its internal state (e.g. its hypothesis). The goal is to guarantee that, no matter what sequence of examples nature presents (as long as the sequence is consistent with some target function in our class), the total number of mistakes is bounded.

Mistake bound (formal). A learner \(L\) has mistake bound \(M\) on concept class \(\mathcal{C}\) if, for every sequence of examples consistent with some \(f \in \mathcal{C}\) (i.e. the label of each \(x\) is \(f(x)\)), \(L\) makes at most \(M\) mistakes.
We do not assume examples are drawn from a distribution; the sequence can be adversarial. The only promise is that there exists some target function in \(\mathcal{C}\) that labels every example correctly. A good learner has a small \(M\) (and efficient update time).
Lemma (without loss of generality, only update on mistakes). Suppose an online learner \(A\) has mistake bound \(M\) on a concept class \(\mathcal{C}\), even if \(A\) is allowed to update after correct predictions. Then there is another learner \(A'\) with the same mistake bound \(M\) that updates only when it makes a mistake.

Proof. Define \(A'\) to make exactly the same prediction as \(A\) would from its current state, but to ignore every update that would have happened after a correct prediction. Thus \(A'\) changes state only on its own mistakes.

Consider the subsequence of examples on which \(A'\) makes mistakes. On that subsequence, \(A'\) evolves exactly like \(A\) run only on that same subsequence: both start from the same initial state, and after each mistaken example they apply the same update rule. Therefore, whenever \(A'\) makes its \(j\)-th mistake, algorithm \(A\) run on the mistake-only subsequence also makes its \(j\)-th mistake.

That subsequence is still consistent with the same target \(f \in \mathcal{C}\). Since \(A\) has mistake bound \(M\) on every consistent sequence, \(A\) can make at most \(M\) mistakes on that subsequence. Hence \(A'\) also makes at most \(M\) mistakes. So we may assume, without loss of generality, that updates happen only on mistakes.

This reduction is helpful in proofs: all of the algorithm's real progress can be charged to mistaken rounds, while correct rounds can be treated as passive observations.

1.2 Monotone Disjunctions and the Goal (Pages 2–3)

A natural concept class is monotone disjunctions over the Boolean domain \(X = \{0,1\}^d\). Each concept is a disjunction of (unnegated) literals: \(f(x) = x_{i_1} \lor x_{i_2} \lor \cdots\). So the function is 1 if and only if at least one of the chosen coordinates is 1.

Concrete example. Let \(f(x) = x_1 \lor x_3 \lor x_6\). Then \(f(1,0,0,\ldots) = 1\) (first bit is 1), \(f(0,0,1,0,0,0,\ldots) = 1\) (third bit is 1), and \(f(0,1,0,1,0,0,\ldots) = 0\) (none of bits 1, 3, 6 is 1). Our goal is to design a learner that has mistake bound \(MB \le d\) for this class—so we make at most \(d\) mistakes before we have identified the target (or equivalent hypothesis).

1.3 The Elimination Algorithm (Pages 3–4)

The elimination algorithm maintains a hypothesis that is a disjunction of the literals we still believe could be in the target. We start by being maximally cautious: we assume every variable might be in the disjunction.

  • Initialization: \(h = x_1 \lor x_2 \lor \cdots \lor x_d\).
  • Prediction: On instance \(x\), predict 1 if and only if at least one of the literals in \(h\) is 1 in \(x\) (i.e. the current disjunction evaluates to true on \(x\)).
  • Update (on a mistake): We only make a mistake when we predicted 1 but the true label is 0. In that case, the target \(f^*\) evaluated to 0 on \(x\), so every literal in \(f^*\) must be 0 in \(x\). Therefore every literal that was 1 in \(x\) cannot belong to \(f^*\). We remove from \(h\) every \(x_i\) for which the \(i\)-th coordinate of \(x\) was 1.
Trace. Suppose the target is \(f^* = x_2 \lor x_4 \lor x_7 \lor x_8\). We start with \(h = x_1 \lor \cdots \lor x_d\). If we see \(x = (1,0,1,0,\ldots)\) and we predict 1 (correct). Later we see \(x = (1,0,1,0,\ldots)\) with label 0—impossible for this target, so assume a different \(x\). For an instance with \(x_1 = 1, x_3 = 1\) and label 0, we remove \(x_1\) and \(x_3\) from \(h\), since they cannot be in \(f^*\). So each such mistake removes at least one literal from \(h\).
Why \(MB = d\). Every mistake (false positive) causes us to remove at least one literal from \(h\). We never remove a literal that is in \(f^*\), because when the label is 0, every literal in \(f^*\) is 0 in \(x\), so we only remove literals that were 1 in \(x\)—and those cannot be in \(f^*\). We start with \(d\) literals and only remove literals; so we can remove at most \(d\) literals, hence at most \(d\) mistakes. So the mistake bound is \(d\).

1.4 General Disjunctions and the Reduction (Pages 4–5)

If we allow negated literals, we have general disjunctions, e.g. \(f(x) = x_1 \lor \bar{x}_2 \lor x_7 \lor \bar{x}_{23}\). We reduce to the monotone case by introducing new variables: for each original variable \(X_i\), define \(Y_i = \bar{X}_i\). So every general disjunction on \(d\) variables becomes a monotone disjunction on \(2d\) “literals” (the \(x_i\) and the \(y_i\)). The mapping is \((x_1,\ldots,x_d) \mapsto (x_1,\ldots,x_d, y_1,\ldots,y_d)\) with \(y_i = 1 - x_i\). Running the elimination algorithm on these \(2d\) literals gives mistake bound \(MB = 2d\).

Why \(MB = 2d\) for general disjunctions. The reduction doubles the variable space: each literal (positive or negative) in the original formula corresponds to exactly one of the \(2d\) monotone literals. Elimination never removes a “true” literal (one in the target), and each mistake removes at least one literal from the current hypothesis. So at most \(2d\) removals, hence at most \(2d\) mistakes.
Example: \(f(x) = \bar{x}_1 \lor \bar{x}_2 \lor \cdots \lor \bar{x}_d\) uses only negated literals. In the reduced space this is a monotone disjunction on \(y_1,\ldots,y_d\), so we need at most \(d\) mistakes there; in the full \(2d\)-literal space the bound is \(2d\).
Visualization (p.5). Map \((x, \bar{x})\) to a higher-dimensional monotone vector: the point \((x_1,\ldots,x_d)\) becomes \((x_1,\ldots,x_d, 1-x_1,\ldots,1-x_d)\).

1.5 The Halving Algorithm (Pages 6–7)

For any finite concept class \(\mathcal{C}\), the halving algorithm gives an information-theoretic mistake bound. We maintain the set of functions in \(\mathcal{C}\) that are still consistent with all past examples. On each round we predict by majority vote over this set. If we are wrong, the true function was in the minority; every function that agreed with our (wrong) prediction is inconsistent with the true label and can be eliminated. At least half of the consistent functions had to agree with the majority, so at least half are eliminated.

Mistake bound \(\log_2 |\mathcal{C}|\). Let \(N_t\) be the number of functions still consistent after \(t\) mistakes. Initially \(N_0 = |\mathcal{C}|\). After each mistake, \(N_{t+1} \le N_t/2\). So after \(M\) mistakes, \(N_M \le |\mathcal{C}| \cdot 2^{-M}\). We must always have \(N_M \ge 1\) (the true function remains consistent). So \(|\mathcal{C}| \cdot 2^{-M} \ge 1\), i.e. \(2^M \le |\mathcal{C}|\), hence \(M \le \log_2 |\mathcal{C}|\).
If \(|\mathcal{C}| = 1{,}000{,}000\), then \(\log_2 |\mathcal{C}| = 20\), so we make at most 20 mistakes. The bound is information-theoretically optimal (we need at least \(\log_2 |\mathcal{C}|\) bits to identify the target), but the algorithm can be computationally expensive because we must maintain and vote over the whole consistent set.
Halving is not always optimal when duplicates are counted. Let the domain be \(Z=\{z_1,z_2,z_3\}\), and let the concept class be the following multiset of Boolean functions: \[ \underbrace{100,\dots,100}_{6\text{ copies}}, \qquad \underbrace{010,\dots,010}_{3\text{ copies}}, \qquad \underbrace{001,\dots,001}_{2\text{ copies}}, \qquad 000. \] Here \(100\) means the function with values \(f(z_1)=1\), \(f(z_2)=0\), \(f(z_3)=0\). Suppose ties in the halving vote are broken in favor of label \(1\), and the target function is \(000\).

Present the points in the order \(z_1,z_2,z_3\). Initially there are \(12\) functions, with exactly \(6\) voting \(1\) on \(z_1\), so the vote ties and halving predicts \(1\), making a mistake. After removing the six copies of \(100\), the surviving class is \(010,010,010,001,001,000\). On \(z_2\), again exactly half of the survivors vote \(1\), so halving again predicts \(1\) and makes a second mistake. The survivors are now \(001,001,000\). On \(z_3\), the majority predicts \(1\), so halving makes a third mistake.

The issue is not with the distinct concepts themselves, but with how duplicates bias the majority vote. This is why the example feels slightly artificial: halving is being applied to a multiset rather than to a set of distinct hypotheses.
A different algorithm can achieve mistake bound \(2\) on the same multiset class. Ignore duplicates and run halving on the set of distinct functions \(\{100,010,001,000\}\). This class has size \(4\), so the usual halving theorem gives mistake bound \(\log_2 4 = 2\). Thus duplicate-sensitive halving can be strictly worse than another reasonable learning strategy.

1.5A Learning a Countably Infinite Class by Doubling

The finite-class halving argument extends to countably many candidate programs by gradually enlarging the prefix that we are willing to consider.

Doubling-trick algorithm. In phase \(r\), run halving on the finite prefix \[ \mathcal{C}_r=\{f_1,f_2,\dots,f_{2^r}\}. \] If all functions in \(\mathcal{C}_r\) are eliminated, move to phase \(r+1\) and restart halving on the larger prefix. If the target is \(f_t\), then once \(r \ge \lceil \log_2 t\rceil\), the target belongs to \(\mathcal{C}_r\), so from that phase onward it is never eliminated.

In phase \(r\), the halving mistake bound is \[ \log_2 |\mathcal{C}_r|=\log_2(2^r)=r. \] Therefore, if \(R=\lceil \log_2 t\rceil\), the total number of mistakes up through phase \(R\) is at most \[ 1+2+\cdots+R=\frac{R(R+1)}{2}=O\big((\log t)^2\big). \] So this simple prefix-doubling method learns the unknown target with \(O((\log t)^2)\) mistakes.

1.5B Improving to \(O(\log t)\) Mistakes With Prior Weights

We can improve the doubling bound by assigning each candidate \(f_i\) a small initial prior weight and then running a weighted version of halving over the whole infinite class at once.

Weighted infinite halving. Give program \(f_i\) initial weight \[ w_i=\frac{1}{i(i+1)}. \] These weights sum to \(1\), because \[ \sum_{i=1}^{\infty}\frac{1}{i(i+1)}=\sum_{i=1}^{\infty}\left(\frac{1}{i}-\frac{1}{i+1}\right)=1. \] On each round, among all hypotheses still consistent with the labels seen so far, predict by weighted majority vote. If the prediction is wrong, eliminate every surviving hypothesis that agreed with the wrong label.

Let \(W_M\) be the total surviving weight after \(M\) mistakes. Initially \(W_0=1\). On every mistake, at least half of the current surviving weight voted for the wrong label, so after eliminating that side, \[ W_{j+1}\le \frac{W_j}{2}. \] Hence after \(M\) mistakes, \[ W_M \le 2^{-M}. \] If the target is \(f_t\), then its weight is never eliminated, so \[ \frac{1}{t(t+1)} \le W_M \le 2^{-M}. \] Therefore \[ M \le \log_2(t(t+1)) \le 1 + 2\log_2 t = O(\log t). \] This is the clean countable-class analogue of halving: the mistake bound is governed by the logarithm of the inverse prior weight of the target.

Goals for learning algorithms: we want both a small mistake bound and efficiency (polynomial time per round). Halving gives the optimal mistake bound for finite \(\mathcal{C}\) but may be intractable; elimination for disjunctions is efficient and achieves \(MB = d\) or \(2d\).

1.6 Decision Lists, DNF, and the Bag Algorithm (Pages 7–11)

Why these three belong together. So far we learned disjunctions (OR of literals) with elimination and any finite class with halving. The next step is: what about richer Boolean function classes? This section introduces two such classes—decision lists and DNF—and one algorithm, the bag algorithm. The link is: decision lists are the concept class we care about here; the bag algorithm is the mistake-bounded learning algorithm for that class (so “decision lists” = what we learn, “bag algorithm” = how we learn them). DNF is a different, also rich, representation (OR of ANDs); it appears in the same place in the notes to show the progression from “OR of literals” to “OR of ANDs” and to contrast with decision lists—we have a concrete algorithm for decision lists, while learning DNF in general is hard. So: one concept class + its algorithm (decision lists + bag), plus a related class for context (DNF).

Decision lists (the concept class). A decision list is a sequence of rules \((L_1, b_1), (L_2, b_2), \ldots\), where each \(L_i\) is a conjunction of literals and \(b_i \in \{0,1\}\). On input \(x\), we find the first \(i\) such that \(L_i(x)\) is true and output \(b_i\). So the list is like an if-then-else chain (e.g. “if \(\bar{x}_4\) then …; else if \(x_2\) then …; …”). This is a different concept class from disjunctions: decision lists can express more complex functions (order matters; we can have nested conditions). We want to learn an unknown decision list in the mistake-bounded model—that is what the bag algorithm does.

DNF (a related representation, for contrast). DNF (Disjunctive Normal Form) is “OR of ANDs,” e.g. \(x_1 x_2 x_7 \lor x_{11} \lor x_4 \bar{x}_5\). So we go from disjunctions (OR of literals) to DNF (OR of *terms*, where each term is an AND). The number of possible \(k\)-term DNFs can be as large as \(d^{O(k)}\). Learning \(k\)-term DNF in the worst case is hard; the notes mention this for scope. So in this section: decision lists get a concrete algorithm (bag); DNF is the “other” rich Boolean class that appears alongside for comparison and complexity context.

The bag algorithm (how we learn decision lists). The bag algorithm is the online/mistake-bounded algorithm for learning decision lists of length \(k\). One maintains \(k\) “bags” of rules. Each round: receive instance \(x\) → Predict using the current list (first rule that fires) → receive true label \(y\) → on a mistake, Update by demoting the rule that fired (the one whose condition was satisfied) to a later bag, so it is consulted later in the order. So the order of rules is updated based on errors. Failure trace (p.10): If \(L_i(x)=1\) and we predict TRUE but the true label is FALSE, we made a mistake and demote that rule.

Counting decision lists (p.11). The number of distinct decision lists of length \(k\) on \(n\) variables depends on the exact model (e.g. 1-decision lists: each \(L_i\) is a single literal). In general, each rule has a condition (conjunction over literals) and an output bit; the order of rules matters. This count feeds into mistake bounds and sample complexity for the class.

Part II: Winnow & Weighted Majority (Pages 12–19)

2.1 Mistake-Bound Comparison and Winnow (Pages 12–14)

For monotone disjunctions over \(d\) variables (the same class as in Part I), the trivial elimination algorithm already achieves mistake bound \(O(d)\) (or \(O(2d)\) for general disjunctions with negations)—independent of how many literals \(k\) are in the target. So we don’t need Winnow to get a finite bound. Winnow’s advantage is attribute efficiency: if the target disjunction has only \(k\) relevant variables (e.g. \(f^* = x_{11} \lor x_2 \lor x_{17}\) with \(k=3\) and \(d\) large), Winnow achieves mistake bound \(O(k \log d)\), which can be much smaller than \(d\) when \(k \ll d\). So the comparison is: elimination \(O(d)\) vs Winnow \(O(k \log d)\); when the target is sparse, Winnow makes fewer mistakes.

Winnow maintains a weight \(w_i \ge 0\) for each of the \(d\) variables. Initially \(w_i = 1\) for all \(i\), and we use a threshold \(\theta = d\). We predict 1 if and only if \(\sum_{i=1}^d w_i x_i \ge \theta\). So we are comparing a weighted sum of the current instance to the threshold.

  • Type A (promotion, false negative): We predicted 0 but the true label is 1. So \(\sum_i w_i x_i \lt \theta\). We double \(w_i\) for every \(i\) with \(x_i = 1\). This increases the weighted sum for similar examples in the future so we are more likely to predict 1.
  • Type B (demotion, false positive): We predicted 1 but the true label is 0. So \(\sum_i w_i x_i \ge \theta\). We set \(w_i := 0\) for every \(i\) with \(x_i = 1\). So we zero out the weights of all active variables, reducing the sum sharply.
Sparse target. Suppose \(f^*(x) = x_{11} \lor x_2 \lor x_{17}\) (only 3 relevant variables, \(k=3\)). Winnow will quickly boost the weights of indices 2, 11, 17 when they appear in positive examples, and zero out others when they appear in negative examples.
Bound \(2k \log_2 d\). Let \(W = \sum_i w_i\) (the potential).

Promotions: Only variables in the true disjunction ever get doubled (others get zeroed when they appear in a negative example). Each such variable has weight at most \(\theta = d\) before we would predict 1. So each of the \(k\) relevant literals can be doubled at most \(\log_2 d\) times. Hence number of Type A mistakes \(\le k \log_2 d\).

Demotions: On a Type B mistake we had \(\sum_i w_i x_i \ge \theta = d\), and we set to 0 all \(w_i\) with \(x_i = 1\). So the total weight \(W\) drops by at least \(d\). On a Type A mistake we double some \(w_i\) with \(x_i = 1\); before the update we had \(\sum_i w_i x_i \lt d\), so the increase in \(W\) is less than \(d\). So demotions decrease \(W\) by \(\ge d\) and promotions increase \(W\) by \(\lt d\). The total weight starts at \(d\) and never goes negative, so the number of demotions is at most the number of promotions (up to a constant). So total mistakes \(\le 2 \cdot k \log_2 d = O(k \log d)\).

2.2 Weighted Majority (Pages 14–16)

In the experts setting we have \(K\) experts. Each round we see an instance and must predict; we get the true label and see which experts were wrong. The goal is to have our total mistakes close to the best expert (the one that makes the fewest mistakes \(m\)).

Weighted Majority: Maintain a weight \(w_i\) for each expert (initially 1). Predict by weighted majority vote. When we make a mistake, halve the weight of every expert that was wrong: \(w_i \leftarrow w_i/2\).

Bound \(M \le 2.4(m + \log K)\). Let \(W_t\) be the total weight after round \(t\). When we make a mistake, the wrong experts had combined weight \(\ge W_t/2\) (because we follow the weighted majority). After halving them, \(W_{t+1} \le \frac{1}{2}(\text{wrong}) + (\text{right}) = W_t - \frac{1}{2}(\text{wrong}) \le W_t - W_t/4 = \frac{3}{4} W_t\). So after \(M\) mistakes, \(W \le K (3/4)^M\). The best expert (with \(m\) mistakes) has weight \((1/2)^m\). So \((1/2)^m \le W \le K(3/4)^M\). Taking logs: \(-m \ln 2 \le \ln K + M \ln(3/4)\), so \(M \ln(4/3) \le m \ln 2 + \ln K\), hence \(M \le \frac{m + \log K}{\log(4/3)}\) (with log in same base). With natural log: \(M \le \frac{\ln 2}{\ln(4/3)}(m + \frac{\ln K}{\ln 2}) \approx 2.41(m + \log_2 K)\).
The total weight drops by a factor of at most \(3/4\) on each of our mistakes, while the best expert’s weight is \((1/2)^m\). So we cannot make too many mistakes or the total weight would fall below the best expert’s weight. The \(\log K\) term is the price for not knowing which expert is best in advance.
Why "follow the best expert so far" can fail badly. Consider the deterministic strategy that always follows the expert with the fewest mistakes so far, breaking ties in favor of the smallest index. This strategy can be forced to make a mistake on each of \(k-1\) rounds even though one expert is perfect on all of those rounds.

Let the experts be \(E_1,\dots,E_k\). For rounds \(r=1,2,\dots,k-1\), arrange the predictions so that expert \(E_r\) is wrong on round \(r\), while every expert \(E_j\) with \(j\gt r\) is correct on round \(r\). Choose the true label each round so that this pattern holds.

Before round \(1\), all experts are tied at \(0\) mistakes, so the strategy follows \(E_1\), which is wrong. Before round \(2\), expert \(E_1\) has one mistake, while \(E_2,\dots,E_k\) still have zero, so the strategy follows \(E_2\), which is wrong. Continuing inductively, before round \(r\), experts \(E_1,\dots,E_{r-1}\) each have one mistake, while \(E_r,\dots,E_k\) still have zero, so tie-breaking picks \(E_r\), which we arranged to be wrong on round \(r\).

Hence this strategy makes a mistake on every one of the first \(k-1\) rounds. Meanwhile expert \(E_k\) is correct on all those rounds and therefore makes \(0\) mistakes. This is the standard cautionary example showing why weighted majority is more robust than greedily following the current empirical leader.

2.3 Randomized Weighted Majority (Page 17)

Setup. We have \(K\) experts. Each round we see an instance and must predict \(\pm 1\); we get the true label and see which experts were wrong. As in Weighted Majority, we maintain a weight \(w_i\) per expert (initially 1). The difference: we use a parameter \(\epsilon \in (0, 1/2)\) and (i) penalize each incorrect expert by a factor \((1-\epsilon)\): if expert \(i\) is wrong, set \(w_i \leftarrow (1-\epsilon) w_i\); (ii) predict randomly: if the total weight of experts saying \(+1\) is \(w_+\) and the total weight of all experts is \(W\), we predict \(+1\) with probability \(w_+/W\) and \(-1\) with probability \(1 - w_+/W\). So we follow the weighted proportion. Let \(\ell_t\) be the expected loss of our algorithm at time \(t\) (we lose 1 if we guess incorrectly, 0 otherwise). Then \(\ell_t\) equals the (weighted) proportion of experts who answer incorrectly at that round. The goal is to bound the total expected loss \(\sum_t \ell_t\) in terms of the best expert’s loss \(\mathrm{opt}\).

(a) Weight update: \(W_t = W_{t-1}(1 - \epsilon \ell_{t-1})\).
At the end of round \(t-1\), the total weight is \(W_{t-1}\). The weighted proportion of experts who were wrong in round \(t-1\) is \(\ell_{t-1}\), so the total weight on wrong experts is \(\ell_{t-1} W_{t-1}\). After the update: every wrong expert’s weight is multiplied by \((1-\epsilon)\), and every correct expert’s weight is unchanged. So \[ W_t = (1-\epsilon)\bigl(\ell_{t-1} W_{t-1}\bigr) + \bigl(1 - \ell_{t-1}\bigr) W_{t-1} = W_{t-1}\bigl[(1-\epsilon)\ell_{t-1} + (1 - \ell_{t-1})\bigr] = W_{t-1}(1 - \epsilon \ell_{t-1}). \]
(b) Expected loss bound: \((1+\epsilon)\,\mathrm{opt} + \frac{\log K}{\epsilon}\).
By (a), \(W_t = W_{t-1}(1 - \epsilon \ell_{t-1})\). Unrolling, \(W_T = W_0 \prod_{t=1}^{T} (1 - \epsilon \ell_t)\). Initially \(W_0 = K\). So \[ W_T = K \prod_{t=1}^{T} (1 - \epsilon \ell_t). \] Let \(\mathrm{opt} = m\) be the number of mistakes of the best expert. That expert’s weight is multiplied by \((1-\epsilon)\) exactly \(m\) times, so its final weight is \((1-\epsilon)^m\). Since every weight is nonnegative, we have \((1-\epsilon)^m \le W_T\). Thus \[ (1-\epsilon)^m \le K \prod_{t=1}^{T} (1 - \epsilon \ell_t). \] Take natural log: \[ m \ln(1-\epsilon) \le \ln K + \sum_{t=1}^{T} \ln(1 - \epsilon \ell_t). \] Use the inequality (for \(x \in (0, 1/2)\)): \(-x^2 - x \le \ln(1-x) \le -x\).
• For \(x = \epsilon\): \(\ln(1-\epsilon) \ge -\epsilon^2 - \epsilon\), so \(m\ln(1-\epsilon) \ge m(-\epsilon^2 - \epsilon)\).
• For \(x = \epsilon \ell_t\) (with \(\epsilon \ell_t \le \epsilon \lt 1/2\)): \(\ln(1 - \epsilon \ell_t) \le -\epsilon \ell_t\), so \(\sum_t \ln(1 - \epsilon \ell_t) \le -\epsilon \sum_t \ell_t = -\epsilon L\), where \(L = \sum_{t=1}^{T} \ell_t\) is the total expected loss.
So \[ m(-\epsilon^2 - \epsilon) \le m\ln(1-\epsilon) \le \ln K + \sum_t \ln(1 - \epsilon \ell_t) \le \ln K - \epsilon L. \] Hence \(-m\epsilon - m\epsilon^2 \le \ln K - \epsilon L\), so \(\epsilon L \le \ln K + m\epsilon + m\epsilon^2\), and \[ L \le \frac{\ln K}{\epsilon} + m(1 + \epsilon) = (1+\epsilon)\,m + \frac{\ln K}{\epsilon} = (1+\epsilon)\,\mathrm{opt} + \frac{\log K}{\epsilon}. \] So the expected loss of the algorithm is at most \((1+\epsilon)\,\mathrm{opt} + \frac{\log K}{\epsilon}\).
Tuning \(\epsilon\): smaller \(\epsilon\) gives a better multiplicative factor \((1+\epsilon)\) but a larger additive term \(\log K/\epsilon\). So we get a smooth tradeoff between being close to the best expert (multiplicative) and the cost of not knowing which expert is best (additive).

2.4 Probability and Expectation (Pages 17–19)

Random variables and sample spaces (p.17): A random variable \(X\) is a function from outcomes (e.g. outcomes of a random experiment) to numbers. The expected value is

\[ \mathbb{E}[X] = \sum_k \mathbb{P}[k] \cdot X(k). \]

(Or the integral in the continuous case.)

Die roll (p.17). Roll a fair 6-sided die; \(X\) = outcome. Then \(\mathbb{E}[X] = \frac{1}{6}(1+2+3+4+5+6) = 3.5\). Quiz returns (p.18): 40 students; each gets their own quiz back with some probability. The expected number who get their own quiz is the sum of the individual probabilities (by linearity). So we can compute \(\mathbb{E}[\text{# matches}]\) without enumerating all permutations.

Linearity of expectation (p.19): \(\mathbb{E}[X_1 + X_2 + \cdots + X_n] = \mathbb{E}[X_1] + \mathbb{E}[X_2] + \cdots + \mathbb{E}[X_n]\). This holds even when the \(X_i\) are dependent—no independence is required. That is the “miracle”: it simplifies many calculations. We use it to analyze algorithms (e.g. expected number of mistakes) and to reason about the recursive structure of decision trees (expectation over random splits).

Depth vs size (p.20): For a Boolean decision tree, depth = longest path from root to leaf; size = number of nodes (or leaves). The best-known bound for DT learning is quasi-polynomial: \(n^{O(\log n)}\) (p.20 formula).

Tree → Decision List (p.21): Steps to convert a Decision Tree into a Decision List: expand the literal space (introduce variables for conjunctions along paths), then list rules in an order consistent with the tree structure.

Rank of a decision tree (p.22): Recursive definition. \(\mathrm{Rank}(\mathrm{leaf})=0\). If \(\mathrm{Rank}(T_L)=\mathrm{Rank}(T_R)\) then \(\mathrm{Rank}(T)=\mathrm{Rank}(T_L)+1\); else \(\mathrm{Rank}(T)=\max(\mathrm{Rank}(T_L),\mathrm{Rank}(T_R))\).
Theorem (p.23–25): Tree of size \(S\) has rank \(\le \log_2 S\). Induction: base \(S=1\), \(R=0\). For \(\mathrm{Rank}(T)=k\), both subtrees have rank \(\ge k-1\) ⇒ sizes \(\ge 2^{k-1}\) ⇒ \(S \ge 2\cdot 2^{k-1}+1 = 2^k+1\) ⇒ \(k \le \log_2 S\). Rank-\(k\) trees can be written as \(k\)-Decision Lists (induction on rank+size).

Gini splitting (p.28): Visualization of greedy tree building using Gini-based splits.

Motivation. Decision trees are another rich Boolean representation: they encode classifiers as recursive if-then-else programs. This section serves two goals. First, it explains how to build trees (greedy impurity minimization, Gini) and why this is reasonable. Second, it connects trees back to decision lists from Part I: using the notion of rank, we show that any bounded-rank tree can be converted into a bounded-length decision list. So the rank theorem and the “tree → decision list” construction explain how expressive trees relate to the decision-list class and why trees appear here.

3.1 Greedy Construction

At each node we have a subset of the data. We choose a feature (or literal) and split the data by that feature. The gain of a split measures how much we reduce error (or impurity) by splitting:

\[ \mathrm{Gain} = \mathrm{Error}(\mathrm{parent}) - \sum_{\mathrm{children}} \frac{n_{\mathrm{child}}}{n_{\mathrm{parent}}}\,\mathrm{Error}(\mathrm{child}). \]

We pick the split that maximizes Gain. Repeat until a stopping condition (e.g. pure node or max depth).

Why greedy? Finding the optimal tree is NP-hard. Greedy gain maximization is fast and in practice often gives good trees. We’re effectively asking: “which question gives the most information about the label?”
Worked Gini-splitting example. Suppose a node contains \(8\) training examples, \(4\) positive and \(4\) negative. Its Gini impurity is \[ G\!\left(\frac{4}{8}\right)=2\cdot \frac{1}{2}\cdot \frac{1}{2}=0.5. \] If a feature \(x_4\) splits the data into a left child with \(5\) points \((1\text{ positive},4\text{ negative})\) and a right child with \(3\) points \((3\text{ positive},0\text{ negative})\), then the weighted post-split impurity is \[ \frac{5}{8}G\!\left(\frac{1}{5}\right)+\frac{3}{8}G(1) = \frac{5}{8}\cdot 2\cdot \frac{1}{5}\cdot \frac{4}{5}+\frac{3}{8}\cdot 0 =0.2. \] So the gain is \(0.5-0.2=0.3\). If our stopping rule says “stop when gain \(\lt 0.1\),” then we should definitely split on this feature.

Continuing recursively, if one child is already pure, we stop there. On the impure child we repeat the same calculation with the remaining candidate features, again choosing the split with smallest weighted impurity. This is exactly the bookkeeping used in hand-worked decision-tree exercises.

3.2 Potential Functions (Splitting Criteria)

We need a way to measure “error” or “impurity” at a node. If a fraction \(a\) of the node is positive, we want a function \(\Phi(a)\) that is 0 when the node is pure (\(a=0\) or \(a=1\)) and larger when \(a\) is near \(1/2\).

  • Misclassification rate: \(\Phi(a) = \min(a, 1-a)\). This is the fraction we’d misclassify if we predict the majority. Not differentiable at \(a = 1/2\).
  • Gini index (p.27): \(\Phi(a) = G(a) = 2a(1-a)\). Smooth, differentiable. For \(a \in [0,1]\), \(2a(1-a) \ge \min(a,1-a)\).

Optimization geometry (p.28): The curve \(\min(x, 1-x)\) has a corner at \(x = 1/2\); the Gini curve \(2x(1-x)\) is smooth and approximates it from above. Visualizing both shows why Gini is preferred for gradient-based or numeric optimization while still tracking impurity.

Why \(2a(1-a) \ge \min(a,1-a)\)? For \(a \in [0, 1/2]\), \(\min(a,1-a) = a\). We need \(2a(1-a) \ge a\), i.e. \(2(1-a) \ge 1\), i.e. \(a \le 1/2\), which holds. For \(a \in [1/2, 1]\), \(\min(a,1-a) = 1-a\); we need \(2a(1-a) \ge 1-a\), i.e. \(2a \ge 1\), i.e. \(a \ge 1/2\), which holds. So minimizing weighted Gini at the children gives a proxy that upper-bounds the drop in training misclassification; minimizing Gini tends to reduce training error.
Jensen's inequality. If \(\Phi\) is concave and \(x_1,\dots,x_r\) are points with weights \(\lambda_1,\dots,\lambda_r\ge 0\) satisfying \(\sum_{j=1}^r \lambda_j=1\), then \[ \Phi\!\left(\sum_{j=1}^r \lambda_j x_j\right)\ge \sum_{j=1}^r \lambda_j \Phi(x_j). \] If \(\Phi\) is convex, the inequality reverses. In decision-tree proofs, the \(x_j\) are usually label fractions in the child nodes, and the \(\lambda_j\) are the relative child sizes.
Why splitting cannot increase impurity for concave criteria. Let a parent node have positive-label fraction \(p\). Suppose a split creates children \(1,\dots,r\), where child \(j\) receives fraction \(\lambda_j\) of the samples and has positive-label fraction \(p_j\). Then \[ p=\sum_{j=1}^r \lambda_j p_j, \qquad \sum_{j=1}^r \lambda_j=1. \] If impurity is measured by a concave function \(\Phi\), Jensen's inequality gives \[ \Phi(p)=\Phi\!\left(\sum_{j=1}^r \lambda_j p_j\right)\ge \sum_{j=1}^r \lambda_j \Phi(p_j). \] Therefore the information gain \[ \mathrm{Gain} = \Phi(p)-\sum_{j=1}^r \lambda_j \Phi(p_j) \] is always nonnegative. So refining a node can only preserve or decrease weighted impurity; it never makes the split objectively worse under a concave impurity measure such as entropy or Gini.
Why use Gini? Smoothness makes optimization (e.g. gradient-based or numeric) easier. The fact that it’s an upper bound means that if we reduce Gini, we’re also reducing (or not increasing) the misclassification error we care about.
This is the standard proof pattern when you want to show that splitting a node into more children should help us learn. The parent label proportion is an average of the child proportions, and concavity says impurity at the average is at least the average impurity after splitting.

3.3 Random Forests

Ensemble of trees: (1) Bagging: train each tree on a bootstrap sample (sample \(n\) examples with replacement from the training set). (2) Feature randomness: at each split, consider only a random subset of features. Predict by majority vote over trees.

Why it works. Trees have high variance (small change in data can change the tree a lot). Averaging many trees reduces variance. Bagging and feature randomness make trees diverse, so the average is more stable than a single tree.

3.4 Decision Trees and Decision Lists (Rank Bound)

Rank: measuring “balanced depth”. The rank of a decision tree is defined recursively. Every leaf has rank 0. If an internal node has children of ranks \(r_1\) and \(r_2\), then its rank is \[ \mathrm{Rank}(T) = \begin{cases} r_1 + 1 & \text{if } r_1 = r_2,\\ \max(r_1, r_2) & \text{otherwise.} \end{cases} \] So we “add 1” only when both subtrees have the same rank (a kind of symmetry); otherwise we just take the larger rank. Intuitively, rank counts how many times we can go down to two subtrees of equal depth.

Theorem: \(\mathrm{Rank}(T) \le \log_2 S\) where \(S\) is the number of leaves (size). Proof by induction on \(S\).
Base case. If \(S = 1\), the tree is a single leaf, so \(\mathrm{Rank}(T) = 0\) and \(0 \le \log_2 1 = 0\).

Inductive step. Assume the theorem holds for all trees with fewer than \(S\) leaves. Let \(T\) have left subtree \(T_L\) and right subtree \(T_R\) with sizes \(S_L, S_R\) and ranks \(r_L, r_R\). Then \(S = S_L + S_R + 1\). We consider two cases.
  • Case 1: \(r_L = r_R\). Then \(\mathrm{Rank}(T) = r_L + 1\). For the rank to be \(k\), we must have \(r_L = r_R = k-1\). By the induction hypothesis applied to each subtree, a subtree of rank \(\ge k-1\) must have at least \(2^{k-1}\) leaves. So \(S_L \ge 2^{k-1}\) and \(S_R \ge 2^{k-1}\). Hence \[ S = S_L + S_R + 1 \;\ge\; 2^{k-1} + 2^{k-1} + 1 = 2^k + 1, \] so \(k \le \log_2 S\).
  • Case 2: \(r_L \ne r_R\). Then \(\mathrm{Rank}(T) = \max(r_L, r_R)\). By the induction hypothesis, \(r_L \le \log_2 S_L\) and \(r_R \le \log_2 S_R\). Since \(S_L, S_R \lt S\) and \(S = S_L + S_R + 1\), each of \(\log_2 S_L\) and \(\log_2 S_R\) is at most \(\log_2 S\). Thus \(\mathrm{Rank}(T) = \max(r_L, r_R) \le \log_2 S\).
In both cases, \(\mathrm{Rank}(T) \le \log_2 S\), completing the induction.
Interpretation. Rank roughly measures “balanced depth” of the tree: how many times you can go down to two subtrees of comparable depth. A tree with \(S\) leaves cannot have too many such balanced levels—at most \(\log_2 S\). This is why rank is at most \(\log_2 S\).

From trees to decision lists. A decision tree of rank \(r\) can be turned into a decision list in which each rule is a conjunction of at most \(r\) literals. Conceptually, each root-to-leaf path corresponds to a rule of the form “if (all tests along that path hold) then output (leaf label).” The rank bound guarantees that, after a suitable simplification, these conjunctions never need more than \(r\) literals.

Conversion idea (sketch). Traverse the tree and, for each leaf labeled 1, form a conjunction of literals describing the path from the root to that leaf (for each internal node on the path, include \(x_i\) if we took the “\(x_i=1\)” child, or \(\bar{x}_i\) if we took the “\(x_i=0\)” child). Order these conjunctions so that more specific paths come earlier, and add a final default rule for the remaining points. One can prove, by induction on the tree using the rank definition, that when \(\mathrm{Rank}(T) \le r\), we can choose these conjunctions so that each has length at most \(r\). Thus the resulting representation is a \(r\)-decision list computing the same function as the tree. This connects the expressiveness of trees (via rank) to the decision-list class studied earlier.

Part IV: PAC Learning and Generalization (Pages 29–41)

Assumptions (p.29): Fixed distribution \(D\); i.i.d. draws; ground truth \(c\).

Statistical evaluation (p.30): Classifier \(h\) with true error \(\ge 0.3\). Over 20 random draws, the probability that \(h\) remains consistent with all 20 is at most \((1 - 0.3)^{20} = (0.7)^{20} \approx 2 \times 10^{-5}\). So we can “catch” a bad model with high probability.

Domain and sample (p.31): \(X = \{0,1\}^n\); coordinates; labels. PAC workflow: Domain \(X\) \(\rightarrow\) Coordinates \(\rightarrow\) Label \(\rightarrow\) PAC Algorithm \(\rightarrow\) Hypothesis \(h\).

Hold-out and validation (p.32): Hold-out methodology: Reserve a subset of the data (not used for training); evaluate the model on this set to estimate true error. The probability that an inaccurate hypothesis (error \(\gt \epsilon\)) remains consistent with \(t\) hold-out samples is \(\le (1-\epsilon)^t\).

Input parameters (p.33): Error \(\epsilon\), Confidence \(\delta\). Example: 95% confidence, 0.03 error ⇒ \(m \ge \frac{1}{\epsilon}\ln\frac{|\mathcal{C}|}{\delta}\).

PAC learner requirements (p.34): Polynomial sample and time complexity. Consistency model: Output any \(h \in \mathcal{C}\) consistent with the sample.

Consistency / Union bound (p.35–37): Probability mass of “bad” hypotheses consistent with \(m\) samples \(\le K(1-\epsilon)^m\). Set \(K e^{-\epsilon m} \le \delta\) ⇒ \(m \ge \frac{1}{\epsilon} \ln\frac{|\mathcal{C}|}{\delta}\).
Naive PAC bound via “challenges”. Fix a finite concept class \(\mathcal{C}\) and a target function \(c \in \mathcal{C}\). Let a hypothesis \(h \in \mathcal{C}\) be called bad if its true error is larger than \(\epsilon\): \(\mathrm{err}_D(h) = \mathbb{P}_{x \sim D}[h(x) \ne c(x)] \gt \epsilon\). Think of each training example as a challenge to every bad hypothesis: on that example, there is at least an \(\epsilon\) chance (under the draw from \(D\)) that the bad hypothesis disagrees with the target and so is “caught.” The probability that a fixed bad hypothesis escapes a single challenge (agrees with \(c\) on that example) is at most \(1-\epsilon\). If we draw \(m\) i.i.d. challenges, the probability that this same bad hypothesis passes all of them (is consistent with all \(m\) samples) is at most \((1-\epsilon)^m\). Now there may be many bad hypotheses. In the worst case, every hypothesis in \(\mathcal{C}\) is bad; there are \(|\mathcal{C}|\) of them. By the union bound, the probability that some bad hypothesis is consistent with all \(m\) samples is at most \(|\mathcal{C}|(1-\epsilon)^m\). Using \(1-x \le e^{-x}\), we get \((1-\epsilon)^m \le e^{-\epsilon m}\), so this failure probability is \(\le |\mathcal{C}| e^{-\epsilon m}\). To make this \(\le \delta\), we require \[ |\mathcal{C}| e^{-\epsilon m} \le \delta \quad\Longrightarrow\quad m \ge \frac{1}{\epsilon} \ln \frac{|\mathcal{C}|}{\delta}. \] Thus, with \(m \ge \frac{1}{\epsilon} \ln \frac{|\mathcal{C}|}{\delta}\) i.i.d. labeled examples, any consistent learner (one that only outputs hypotheses in \(\mathcal{C}\) that agree with all training examples) will, with probability at least \(1-\delta\), output a hypothesis whose true error is at most \(\epsilon\). This is the basic finite-class PAC sample complexity bound.
Online-to-batch conversion from a mistake bound. Suppose an online algorithm makes at most \(t\) mistakes on every realizable sequence. Given a sample \[ S=\{(x_1,c(x_1)),\dots,(x_m,c(x_m))\} \] of i.i.d. labeled examples from \(D\), run the online learner on the examples in the order \(x_1,\dots,x_m\), producing hypotheses \(h_1,\dots,h_m\) just before seeing each labeled point. Then output one of these hypotheses chosen uniformly at random.

Let \(M_i\) be the indicator that \(h_i(x_i)\neq c(x_i)\). Because the sample is realizable, the online learner makes at most \(t\) mistakes, so \[ \sum_{i=1}^m M_i \le t. \] Also, since \(x_i\) is independent of the past and \(h_i\) depends only on the previous examples, we have \[ \mathbb{E}[M_i] = \mathbb{E}[\mathrm{err}_D(h_i)]. \] Therefore \[ \frac{1}{m}\sum_{i=1}^m \mathbb{E}[\mathrm{err}_D(h_i)] = \frac{1}{m}\sum_{i=1}^m \mathbb{E}[M_i] \le \frac{t}{m}. \] If we output a uniformly random \(h_i\), its expected error is at most \(t/m\). If we instead run the learner on the first \(m\) examples and choose uniformly from the \(m+1\) hypotheses that arise before each draw \(x_1,\dots,x_{m+1}\), the same calculation yields the cleaner bound \[ \mathbb{E}[\mathrm{err}_D(h)] \le \frac{t}{m+1}. \]

A mistake-bounded learner can only be wrong a limited number of times on the sample path. Averaging over the hypotheses it visits converts this online guarantee into a small expected fresh-sample error.
Infinite concept class (p.38): Axis-parallel rectangles in \(\mathbb{R}^2\).

Tightest Fitting Rectangle (p.39): Algorithm: take the smallest axis-aligned rectangle containing all positive examples. “Bad event” strips: four regions each with probability mass \(\epsilon/4\).

Axis-parallel bound (p.40): Probability we miss a given strip is \((1-\epsilon/4)^m\). By union bound, failure probability \(\le 4(1-\epsilon/4)^m \le 4 e^{-\epsilon m/4}\). Set \(4 e^{-\epsilon m/4} \le \delta\) to get \(m \ge \frac{4}{\epsilon}\ln\frac{4}{\delta}\).
Tightest-fitting learner for a disjunction of intervals on the number line. Let the target class be unions of at most \(k\) disjoint intervals: \[ h(x)=\mathbf{1}[x\in I_1\cup\cdots\cup I_k]. \] The natural consistent learner is the one-dimensional analogue of the tightest rectangle learner: sort the positive examples and output the smallest union of at most \(k\) intervals that contains all observed positives.
PAC bound for \(k\) intervals. Each true interval contributes two boundary events, one near its left endpoint and one near its right endpoint, so there are at most \(2k\) critical boundary strips. Choose these strips so that each has probability mass \(\epsilon/(2k)\). If the sample contains at least one positive example in every such strip, then the uncovered part of the target has total probability at most \(\epsilon\).

The probability of missing one fixed strip is \[ \left(1-\frac{\epsilon}{2k}\right)^m. \] By the union bound over the \(2k\) strips, \[ \Pr[\text{failure}] \le 2k\left(1-\frac{\epsilon}{2k}\right)^m \le 2k\,e^{-\epsilon m/(2k)}. \] Therefore it suffices to choose \[ m\ge \frac{2k}{\epsilon}\ln\frac{2k}{\delta} \] to guarantee error at most \(\epsilon\) with confidence at least \(1-\delta\). This is exactly the tightest-fitting boundary-strip argument specialized to a disjunction of intervals on \(\mathbb{R}\).

Loss definitions (p.41): 0/1 loss: \(\ell(h(x),y) = \mathbf{1}[h(x) \ne y]\). Empirical loss: \(\widehat{L}(h) = \frac{1}{m}\sum_i \ell(h(x_i),y_i)\). Population loss: \(L_D(h) = \mathbb{E}_{(x,y)\sim D}[\ell(h(x),y)]\).

Motivation. Move from mistake bounds on sequences to generalization: we draw a sample from a distribution \(D\) and want that, with high probability, our hypothesis has low error on \(D\) (not just on the sample).

4.1 PAC Framework

Probably Approximately Correct (PAC). Goal: with probability \(\ge 1 - \delta\) over the random sample, output a hypothesis \(h\) such that \(\mathrm{err}_D(h) \le \epsilon\). We want sample complexity (number of examples) and runtime polynomial in \(1/\epsilon\), \(1/\delta\), and (for finite \(\mathcal{C}\)) \(\log|\mathcal{C}|\).
Example (p.33): 95% confidence (\(1-\delta = 0.95\)) and error \(\epsilon = 0.03\) ⇒ \(m \ge \frac{1}{\epsilon}\ln\frac{|\mathcal{C}|}{\delta}\).
Consistency model (p.34): Output any \(h \in \mathcal{C}\) that agrees with the sample.
Why “probably” and “approximately”? “Probably” = with confidence \(1-\delta\); we allow a small chance of failure. “Approximately” = we allow error up to \(\epsilon\); we don’t demand perfect accuracy.

Bridge to noise and unrealizability. The noisy and agnostic viewpoint is developed in Part VI. In the realizable case, those guarantees reduce to the standard PAC guarantees because the optimal in-class error becomes zero.

4.2 Cross-Validation

Split data into \(k\) folds. For each fold, train on the other \(k-1\) folds and evaluate on that fold. Average the errors. Used to estimate true error and tune hyperparameters (e.g. \(\lambda\)).

Idea. The test fold acts as a hold-out set, so its error is an unbiased estimate of generalization error (for the model trained on the rest). Averaging over folds reduces variance.

4.3 Chernoff / Hoeffding Bounds

For i.i.d. \(X_1,\ldots,X_n\) with \(X_i \in [0,1]\), let \(\mu = \mathbb{E}[\frac{1}{n}\sum_i X_i]\). Hoeffding: \(\mathbb{P}\bigl[\bigl|\frac{1}{n}\sum_i X_i - \mu\bigr| \ge \Delta\bigr] \le 2 e^{-2n\Delta^2}\).

Why exponential? The probability that the sample mean deviates from \(\mu\) by \(\Delta\) decays exponentially in \(n\). So with a modest increase in sample size, we get much better concentration.
Use for hold-out. Let \(X_i = 1\) if example \(i\) is misclassified, 0 else. Then \(\mu = \mathrm{err}_D(h)\) and \(\frac{1}{n}\sum_i X_i = \widehat{\mathrm{err}}\) (empirical error). So \(\mathbb{P}[|\widehat{\mathrm{err}} - \mathrm{err}_D(h)| \ge \Delta] \le 2e^{-2n\Delta^2}\). To have this \(\le \delta\), set \(2e^{-2n\Delta^2} = \delta\), so \(n = \frac{1}{2\Delta^2} \ln\frac{2}{\delta}\). So with \(n\) hold-out examples, empirical error is within \(\Delta\) of true error with probability \(\ge 1-\delta\).

4.4 PAC Learning From a Mistake-Bounded Learner

A mistake-bound algorithm is stronger than a PAC learner in the realizable setting: once we can force only finitely many online mistakes on every consistent sequence, we can turn that online procedure into a distributional learner.

Key block argument. Fix any hypothesis \(h\) with true error \(\mathrm{err}_D(h) \gt \epsilon\). Then the probability that \(h\) correctly labels \(k\) fresh i.i.d. examples is at most \[ (1-\epsilon)^k \le e^{-\epsilon k}. \] Therefore, if \[ k \ge \frac{1}{\epsilon}\ln\frac{1}{\delta}, \] then the probability that \(h\) gets an entire block of \(k\) random examples correct is at most \(\delta\).

Now suppose algorithm \(A\) has mistake bound \(t\) and only updates on mistakes. Feed it examples one by one. Every time \(A\) makes a mistake, its internal state changes. Since there can be at most \(t\) mistakes, there are at most \(t+1\) consecutive blocks of examples on which the hypothesis remains fixed. Hence after seeing \[ (t+1)k \] examples, we are guaranteed that at least one of those \(t+1\) blocks has length at least \(k\).

If the hypothesis used on that length-\(k\) block had true error greater than \(\epsilon\), then by the previous calculation the probability it would classify all \(k\) examples in the block correctly would be at most \(\delta\). Therefore, with probability at least \(1-\delta\), any block of \(k\) consecutive correct predictions must come from a hypothesis whose true error is at most \(\epsilon\).

PAC learner. Choose \[ k=\left\lceil \frac{1}{\epsilon}\ln\frac{1}{\delta}\right\rceil. \] Run the mistake-bounded learner \(A\) on fresh labeled examples until it achieves \(k\) consecutive correct predictions. Output the current hypothesis. Because \(A\) makes at most \(t\) mistakes, this happens after at most \((t+1)k\) examples. By the block argument above, with probability at least \(1-\delta\) the output hypothesis has error at most \(\epsilon\).

Thus any class learnable with mistake bound \(t\) is PAC-learnable in the realizable case with sample complexity \[ O\!\left(\frac{t}{\epsilon}\log\frac{1}{\delta}\right), \] and runtime polynomial in the runtime of the online learner and in \(1/\epsilon,1/\delta\).

Remark. The systematic lower-bound arguments, including No-Free-Lunch and VC-based sample-complexity lower bounds, are collected later in Part VII so they appear in one place rather than being repeated inside the PAC section.

5.1 VC Dimension

Big picture (p.42): VC dimension \(d = \mathrm{VC}(\mathcal{C})\) measures how rich the class is (how many points it can shatter). In the realizable setting, with a consistent learner that always returns some \(h \in \mathcal{C}\) fitting the sample, the Fundamental Theorem of Statistical Learning says that, with probability at least \(1-\delta\) over the random sample, we have \(\mathrm{err}_D(h) \le \epsilon\) as soon as \[ m = O\!\left(\frac{d + \log(1/\delta)}{\epsilon}\right). \] So more complex classes (larger \(d\)) or smaller \(\epsilon\) both require more data; this matches the form written in the lecture notes.

Geometric definitions (p.47): A set \(S \subseteq \mathbb{R}^d\) is convex if for all \(x, y \in S\) and \(\lambda \in [0,1]\), \(\lambda x + (1-\lambda)y \in S\). A convex combination of points \(x_1,\ldots,x_k\) is \(\sum_i \alpha_i x_i\) with \(\alpha_i \ge 0\) and \(\sum_i \alpha_i = 1\). The convex hull \(\mathrm{conv}(S)\) is the set of all convex combinations of points in \(S\).
Halfspaces and convexity (p.50): If \(h(x) = +1\) and \(h(y) = +1\) for a halfspace \(h(z) = \mathrm{sign}(w \cdot z + b)\), then \(w \cdot x + b \ge 0\) and \(w \cdot y + b \ge 0\). For any convex combination \(z = \lambda x + (1-\lambda)y\) with \(\lambda \in [0,1]\), \(w \cdot z + b = \lambda(w \cdot x + b) + (1-\lambda)(w \cdot y + b) \ge 0\). So \(h(z) = +1\). Thus the set of points labeled \(+1\) by a halfspace is convex; similarly for \(-1\). So halfspace labelings are convex—this is used in Radon’s argument.
A set \(S\) is shattered by \(\mathcal{C}\) if for every labeling of \(S\) (each of the \(2^{|S|}\) ways to assign 0/1 to points in \(S\)), there exists a function in \(\mathcal{C}\) that realizes that labeling. The VC dimension of \(\mathcal{C}\), \(\mathrm{VC}(\mathcal{C})\), is the maximum size of a set that can be shattered by \(\mathcal{C}\).
Intuition. VC dimension measures the “capacity” of the class: how many points can be labeled in all possible ways? Intervals on the line can shatter 2 points but not 3 (you can’t label 3 points as + − + with one interval). So VC dimension of intervals = 2.

Examples: Intervals on \(\mathbb{R}\): VC dim 2. Axis-aligned rectangles: VC dim 4. Halfspaces in \(\mathbb{R}^n\): VC dim \(n+1\). (p.42–43) Fundamental Theorem of Statistical Learning; shattering. Set of 9 unlabeled points and labeling variation over \(\mathcal{C}\); VC-dim \(k\) = largest shattered set size. (p.44–46) Intervals (VC=2), rectangles (VC=4), triangles in the plane, 8 points and shattered triangles. (p.47–48) Convex sets and convex combinations; convex hull \(\mathrm{conv}(S)\). Halfspaces, linear thresholds, Perceptron model. (p.50–53) Halfspaces are convex separators (\(z = \lambda x + (1-\lambda)y\)). Weight vector \(w\) and bias \(b\). PAC for halfspaces in \(d\) dimensions. Solving linear inequalities in polynomial time. Agnostic learning: no assumption target is in \(\mathcal{C}\).

Fundamental theorem (proof). If \(h\) has true error \(\gt \epsilon\), then the probability that \(h\) is consistent with one random example is \(\le 1-\epsilon\). So the probability that \(h\) is consistent with \(m\) i.i.d. examples is \(\le (1-\epsilon)^m\). Using the approximation \(1+x \le e^x\) (for all \(x\)), \((1-\epsilon)^m \le e^{-\epsilon m}\). By the union bound over all \(|\mathcal{C}|\) hypotheses, the probability that some bad hypothesis (error \(\gt \epsilon\)) is consistent with the sample is \(\le |\mathcal{C}| e^{-\epsilon m}\). Set this \(\le \delta\): \(|\mathcal{C}| e^{-\epsilon m} \le \delta\) \(\Rightarrow\) \(m \ge \frac{1}{\epsilon} \ln\frac{|\mathcal{C}|}{\delta}\).

5.2 Axis-Parallel Rectangles (PAC Example)

Concept class: Axis-aligned rectangles in \(\mathbb{R}^2\) (or \(\mathbb{R}^d\)). Algorithm: Take the tightest-fitting rectangle that contains all positive examples. To bound error by \(\epsilon\), define four “strips” (top, bottom, left, right) such that each has probability mass \(\epsilon/4\) under \(D\). If we see at least one example in each strip, our rectangle has error \(\le \epsilon\). The probability we miss a given strip is \((1-\epsilon/4)^m\). So failure probability \(\le 4(1-\epsilon/4)^m\). Set \(4(1-\epsilon/4)^m \le \delta\) to get \(m \ge \frac{4}{\epsilon} \ln\frac{4}{\delta}\).

5.3 VC Dimension (continued)

Equivalent definition: \(\mathcal{C}\) shatters \(S\) iff the restriction \(|\mathcal{C}_S| = 2^{|S|}\).

Finite-class upper bound. For every finite hypothesis class \(H\), \[ \mathrm{VCdim}(H)\le \log_2 |H|. \] Indeed, if \(d=\mathrm{VCdim}(H)\), then \(H\) shatters some set of size \(d\), so it must realize all \(2^d\) labelings of that set. Hence \(2^d\le |H|\), and taking \(\log_2\) gives \(d\le \log_2|H|\).
Disjunctions over \(n\) literals have VC dimension at least \(n\). Consider the \(n\) standard basis vectors \[ e_1,\dots,e_n \in \{0,1\}^n, \] where \(e_i\) has a \(1\) in coordinate \(i\) and \(0\) elsewhere. We claim that the class of disjunctions shatters this set.

Take any labeling of \(\{e_1,\dots,e_n\}\). Let \[ I=\{i : e_i \text{ should be labeled } 1\}. \] Define the disjunction \[ h_I(x)=\bigvee_{i\in I} x_i. \] Then for each basis vector \(e_j\), we have \(h_I(e_j)=1\) exactly when \(j\in I\), because \(e_j\) has only one active coordinate. Therefore every labeling of the \(n\) basis vectors is realized by some disjunction.

Hence the set \(\{e_1,\dots,e_n\}\) is shattered, so \[ \mathrm{VCdim}(\text{disjunctions on } n \text{ variables}) \ge n. \] If negations are also allowed, the lower bound still holds a fortiori.

5.4 Radon’s Theorem and VC Dimension of Halfspaces

Claim: The VC dimension of halfspaces in \(\mathbb{R}^d\) is \(d+1\).

Radon’s Theorem. Any \(d+2\) points in \(\mathbb{R}^d\) are affinely dependent: there exist coefficients \(\alpha_i\) (not all zero) with \(\sum_i \alpha_i x_i = 0\) and \(\sum_i \alpha_i = 0\) (linear dependency: \((x_i,1)\) are dependent in \(\mathbb{R}^{d+1}\)). Partition indices into \(I^+ = \{i : \alpha_i \ge 0\}\) and \(I^- = \{i : \alpha_i \lt 0\}\). Then \(\sum_{i \in I^+} \alpha_i x_i = \sum_{j \in I^-} |\alpha_j| x_j\). Divide by \(A = \sum_{i \in I^+} \alpha_i = \sum_{j \in I^-} |\alpha_j|\); we get \(p = \sum_{i \in I^+} (\alpha_i/A) x_i = \sum_{j \in I^-} (|\alpha_j|/A) x_j\). So \(p\) is in both \(\mathrm{conv}(\{x_i : i \in I^+\})\) and \(\mathrm{conv}(\{x_j : j \in I^-\})\).

Implication. If we label \(I^+\) as \(+1\) and \(I^-\) as \(-1\), any halfspace that labels all of \(I^+\) positive and all of \(I^-\) negative would have to label \(p\) (in both convex hulls) as both—impossible. So no set of \(d+2\) points is shattered. One can show \(d+1\) points can be shattered. So VC-dim of halfspaces in \(\mathbb{R}^d\) is \(d+1\).

Sample complexity (VC dimension): For \(\mathcal{C}\) with VC dimension \(d\), to get \(\mathrm{err}_D(h) \le \epsilon\) with probability \(\ge 1-\delta\), sample complexity is \(n = O\bigl(\frac{1}{\epsilon^2}\bigl(d \ln\frac{1}{\epsilon} + \ln\frac{1}{\delta}\bigr)\bigr)\) (agnostic/form in p.52: \(O\bigl(\frac{\mathrm{VC} + \log 1/\delta}{\epsilon^2}\bigr)\)). In the realizable case, \(O\bigl(\frac{1}{\epsilon}\bigl(d \ln\frac{1}{\epsilon} + \ln\frac{1}{\delta}\bigr)\bigr)\) suffices.

Exact VC dimension of axis-aligned rectangles in \(\mathbb{R}^d\). Let \[ H_d^{\mathrm{rec}}=\{h_{a,b}: a,b\in\mathbb{R}^d,\ a\le b\}, \qquad h_{a,b}(x)=\prod_{i=1}^d \mathbf{1}[a_i\le x_i\le b_i]. \] Then \[ \mathrm{VCdim}(H_d^{\mathrm{rec}})=2d. \]

Lower bound \(2d\). For each coordinate \(i\), choose two points \(u_i,v_i\in\mathbb{R}^d\) that are extreme only in coordinate \(i\): for example, \[ u_i=(-1 \text{ in coordinate } i,\ 0 \text{ elsewhere}), \qquad v_i=(+1 \text{ in coordinate } i,\ 0 \text{ elsewhere}). \] This gives \(2d\) points. For any subset of these points that we want labeled positive, choose in each coordinate \(i\) the interval \([a_i,b_i]\) to include exactly the selected extremes among \(\{u_i,v_i\}\):

  • If neither \(u_i\) nor \(v_i\) should be positive, choose \([a_i,b_i]\) disjoint from \([-1,1]\).
  • If only \(u_i\) should be positive, choose \([a_i,b_i]\) containing \(-1\) but not \(+1\).
  • If only \(v_i\) should be positive, choose \([a_i,b_i]\) containing \(+1\) but not \(-1\).
  • If both should be positive, choose \([a_i,b_i]\) containing both \(-1\) and \(+1\).

Since every point differs from the others only through these coordinatewise extremes, the Cartesian product rectangle realizes any labeling of the \(2d\) points. So \(2d\) points can be shattered.

Upper bound \(2d\). Take any set \(S\) of \(2d+1\) points in \(\mathbb{R}^d\). For each coordinate \(i\), choose one point with minimum \(i\)-th coordinate and one point with maximum \(i\)-th coordinate. This marks at most \(2d\) points total, so some point \(x^\star\in S\) is unmarked.

Label all marked extreme points positive and label \(x^\star\) negative. Any axis-aligned rectangle that contains all marked extremes must also contain every coordinate value between each minimum and maximum, and because \(x^\star\) lies coordinatewise between those extremes in every dimension, it must also lie inside the rectangle. So this labeling cannot be realized. Hence no set of \(2d+1\) points is shattered.

Combining the lower and upper bounds gives \[ \mathrm{VCdim}(H_d^{\mathrm{rec}})=2d. \]

Loss definitions (p.41): 0/1 loss: \(\ell(h(x),y) = \mathbf{1}[h(x) \ne y]\). Empirical loss: \(\widehat{L}(h) = \frac{1}{m}\sum_{i=1}^m \ell(h(x_i),y_i)\). Population loss: \(L_D(h) = \mathbb{E}_{(x,y)\sim D}[\ell(h(x),y)]\).

5.5 Loss Functions and Logistic Regression

Hinge loss: \(\ell(z) = \max\{1 - z, 0\}\). Used in SVMs; penalizes margin violations.

Logistic loss: \(\ell(z) = \ln(1 + e^{-z})\). Smooth approximation to 0-1 loss; used in logistic regression.

Logistic regression: Model \(\mathbb{P}[y=1 \mid x] = \sigma(w \cdot x) = \frac{1}{1 + e^{-w \cdot x}}\). Maximizing log-likelihood is equivalent to minimizing \(-\sum_i \ln(1 + e^{-y_i w^T x_i})\). This objective is convex; optimize with gradient descent.

Part VI: Learning With Noise

6.1 The Agnostic Viewpoint

In the realizable PAC model we assume that some hypothesis in the class labels every example correctly. Learning with noise is the systematic way to relax that assumption. Instead of supposing that there exists \(h^\star\in H\) with zero error, we allow the data distribution \(D\) over \(X\times \{\pm 1\}\) to be arbitrary. The best we can ask for is a hypothesis whose risk is close to the minimum achievable inside \(H\).

Agnostic objective. For a hypothesis class \(H\), the risk of \(h\in H\) is \[ L_D(h)=\mathbb E_{(x,y)\sim D}\big[\mathbf 1[h(x)\neq y]\big]. \] The agnostic goal is to output \(h\) such that \[ L_D(h)\le \inf_{g\in H} L_D(g)+\epsilon \] with probability at least \(1-\delta\).

The quantity \[ \inf_{g\in H}L_D(g) \] is the approximation floor imposed by noise and model misspecification. When labels are noisy or the true labeling rule is not in \(H\), this term is positive. Agnostic learning therefore separates two sources of error: the irreducible error inside \(H\), and the additional \(\epsilon\) caused by finite samples.

Noise converts the learning problem from “find a perfectly consistent classifier” to “compete with the best classifier in the hypothesis class.” This is the right formulation whenever labels may be corrupted or the model class is only approximate.

6.2 Classification Noise and Random Label Flips

The cleanest noise model is random classification noise. Assume there is an underlying target function \(f:X\to\{\pm 1\}\), but the observed label is flipped independently with probability \(\eta\), where \(0\le \eta \le 1/2\). Thus the observed label \(\widetilde y\) satisfies \[ \widetilde y= \begin{cases} f(x), & \text{with probability }1-\eta,\\ -f(x), & \text{with probability }\eta. \end{cases} \]

Noise identity. For every classifier \(h\), \[ L_\eta(h)=\eta+(1-2\eta)L_f(h), \] where \(L_f(h)=\mathbb P[h(x)\neq f(x)]\) is the error with respect to the clean target and \(L_\eta(h)\) is the observed noisy-label error.

Proof. Let \(A=\{x:h(x)\neq f(x)\}\) and \(A^c=\{x:h(x)=f(x)\}\). If \(x\in A^c\), then \(h\) is correct relative to the clean label and becomes wrong only if the label flips, which happens with probability \(\eta\). If \(x\in A\), then \(h\) is wrong relative to the clean label and becomes correct only if the label flips, so its observed error probability is \(1-\eta\). Therefore \[ L_\eta(h)=\eta\,\mathbb P(A^c)+(1-\eta)\,\mathbb P(A). \] Since \(\mathbb P(A)=L_f(h)\) and \(\mathbb P(A^c)=1-L_f(h)\), \[ L_\eta(h)=\eta(1-L_f(h))+(1-\eta)L_f(h) =\eta+(1-2\eta)L_f(h). \] This proves the identity.

Corollary. If \(\eta\lt 1/2\), then minimizing noisy risk is equivalent to minimizing clean risk, because \(1-2\eta\gt 0\). Hence the optimal classifier does not change under class-independent random label noise below rate \(1/2\). At \(\eta=1/2\), labels are pure coin flips and learning becomes impossible.

Example. If the clean error of \(h\) is \(L_f(h)=0.1\) and the flip rate is \(\eta=0.2\), then \[ L_\eta(h)=0.2+(1-0.4)\cdot 0.1=0.26. \] A perfectly clean classifier still incurs noisy error \(0.2\), because even correct labels are flipped \(20\%\) of the time.

6.3 Bayes Optimal Prediction Under Noise

For a general noisy distribution \(D\), define the conditional class probability \[ \eta(x)=\mathbb P[y=1\mid x]. \] The classifier that minimizes the \(0/1\) risk over all measurable rules is the Bayes classifier \[ h_B(x)= \begin{cases} 1, & \eta(x)\ge 1/2,\\ -1, & \eta(x)\lt 1/2. \end{cases} \]

Why thresholding at \(1/2\) is optimal. Fix \(x\). If we predict \(1\), the conditional probability of error is \[ \mathbb P[y=-1\mid x]=1-\eta(x). \] If we predict \(-1\), the conditional probability of error is \[ \mathbb P[y=1\mid x]=\eta(x). \] Therefore the smaller conditional error is \[ \min\{\eta(x),1-\eta(x)\}, \] achieved by predicting the more likely label. Integrating over \(x\) shows that the Bayes rule minimizes the overall risk.

This gives the conceptual meaning of learning with noise. Labels are no longer deterministic functions of \(x\); instead, the best possible prediction is the more likely label at each \(x\), and the unavoidable Bayes risk is the average of the minority-label probabilities.

6.4 ERM and Agnostic Learning Guarantees

Once noise is present, empirical risk minimization is no longer trying to drive the training error to zero. It is trying to approximate \[ \arg\min_{h\in H} L_D(h) \] by minimizing the empirical loss \[ \widehat L_S(h)=\frac1m\sum_{i=1}^m \mathbf 1[h(x_i)\neq y_i]. \]

Uniform convergence implication. If \[ \sup_{h\in H}\big|L_D(h)-\widehat L_S(h)\big|\le \frac{\epsilon}{2}, \] then every empirical risk minimizer \(\widehat h\in \arg\min_{h\in H}\widehat L_S(h)\) satisfies \[ L_D(\widehat h)\le \inf_{g\in H} L_D(g)+\epsilon. \]

Proof. Let \(h^\star\in \arg\min_{h\in H}L_D(h)\). Then \[ L_D(\widehat h)\le \widehat L_S(\widehat h)+\frac{\epsilon}{2} \le \widehat L_S(h^\star)+\frac{\epsilon}{2} \le L_D(h^\star)+\epsilon = \inf_{g\in H}L_D(g)+\epsilon. \] The first and third inequalities use uniform convergence, and the middle inequality uses the ERM property of \(\widehat h\).

In the noisy setting, ERM succeeds for exactly the same structural reason as in the realizable setting: if empirical losses are uniformly close to true losses, then the empirical minimizer is close to the true minimizer. The only difference is that the true minimum risk is no longer \(0\).

6.5 SQ Learning

Statistical query (SQ) learning replaces direct access to labeled examples by access to expectations of bounded predicates. Instead of asking for a random labeled example \((x,y)\), the learner chooses a query function \(\phi:X\times\{\pm 1\}\to[-1,1]\) and a tolerance \(\tau>0\), and receives a number \(v\) satisfying \[ \bigl|v-\mathbb E_{(x,y)\sim D}[\phi(x,y)]\bigr|\le \tau. \] So an SQ oracle returns an approximate correlation or approximate average, not a raw example.

SQ oracle. Denote the oracle by \(\mathrm{STAT}_D(\tau)\). On input \(\phi\), it returns any value within \(\tau\) of \(\mathbb E_D[\phi]\). An SQ learner is efficient if it asks only polynomially many queries and each query has inverse-polynomial tolerance.

The motivation is robustness to noise. Empirical averages are stable under independent sampling, so if we can estimate the expectation of every query accurately, then the learner can reason from aggregate statistics rather than from possibly corrupted individual labels.

Why examples simulate SQ queries. If we have i.i.d. samples \((x_1,y_1),\dots,(x_m,y_m)\), then for any fixed query \(\phi\) we can estimate \(\mathbb E_D[\phi]\) by the empirical mean \[ \widehat\mu=\frac{1}{m}\sum_{i=1}^m \phi(x_i,y_i). \] Since \(\phi\in[-1,1]\), Hoeffding's inequality gives \[ \Pr\bigl[|\widehat\mu-\mathbb E_D[\phi]|\gt \tau\bigr]\le 2e^{-2m\tau^2}. \] Thus \(m=O(\tau^{-2}\log(1/\delta))\) samples suffice to answer one SQ up to tolerance \(\tau\) with failure probability at most \(\delta\).

This means SQ learning is not stronger than ordinary PAC learning from examples: examples can emulate SQ access by averaging. The important direction is the converse intuition: some algorithms can be designed using only such averages, which makes them naturally noise-tolerant.

Reduction from PAC examples to SQ access. Suppose an SQ algorithm asks \(q\) queries, each with tolerance at least \(\tau\). A PAC learner with ordinary i.i.d. labeled examples can simulate each query by replacing \(\mathbb E_D[\phi]\) with the empirical mean of \(\phi\) on a fresh sample. If each query is answered to accuracy at most \(\tau\), then the SQ algorithm behaves exactly as if it had oracle access to \(\mathrm{STAT}_D(\tau)\). By Hoeffding plus a union bound over the \(q\) queries, it is enough to use \[ m=O\!\left(\frac{1}{\tau^2}\log\frac{q}{\delta}\right) \] samples per query to make all answers simultaneously valid with probability at least \(1-\delta\). Therefore an efficient SQ learner can be converted into an efficient PAC learner.
Reduction from PAC learning to approximate error estimation. In the opposite direction, an SQ oracle can estimate the error of any fixed hypothesis \(h\) by querying \[ \phi_h(x,y)=\mathbf 1[h(x)\neq y]. \] Then \[ \mathbb E_D[\phi_h]=L_D(h). \] So SQ access lets us evaluate the risk of any candidate hypothesis up to additive tolerance \(\tau\). If a PAC procedure can be implemented using only approximate estimates of such expectations, then it can be turned into an SQ learner. This is why many classical algorithms that depend only on correlations, moments, or estimated losses admit SQ versions.
Random classification noise only rescales correlations. Suppose the observed label is \(\widetilde y = b\,y\), where \(b\in\{\pm 1\}\) is independent of \(x,y\) and \(\Pr[b=-1]=\eta\lt 1/2\). For any query of the form \(\phi(x,y)=y\,g(x)\) with \(g(x)\in[-1,1]\), \[ \mathbb E[\widetilde y\,g(x)] = \mathbb E[b]\,\mathbb E[y\,g(x)] = (1-2\eta)\mathbb E[y\,g(x)]. \] So under random label noise, the true correlation is shrunk by the factor \(1-2\eta\), but its sign is preserved as long as \(\eta\lt 1/2\).

That identity is the basic reason SQ algorithms tolerate random classification noise: if the learner only needs approximate correlations, then noise does not destroy the signal; it only attenuates it by a known scalar factor.

Example: learning a halfspace by SQ estimates of perceptron updates. Suppose the target is a halfspace \(h^\star(x)=\mathrm{sign}(w^\star\cdot x)\) over inputs with \(\|x\|_2\le 1\). The ordinary perceptron update on a mistaken example is \[ w \leftarrow w + yx. \] An SQ analogue does not ask for a single mislabeled point. Instead, relative to the current weight vector \(w\), define the set of points the current classifier gets wrong: \[ M_w=\{(x,y): y(w\cdot x)\le 0\}. \] Then query the conditional average update direction \[ \mu_w=\mathbb E[yx\mid (x,y)\in M_w]. \] If the current classifier still makes noticeable error, this vector points in a direction positively correlated with \(w^\star\), just as the average of perceptron mistake updates does. So an SQ-style algorithm can use an approximate version of \(\mu_w\) and update by \[ w \leftarrow w + \widehat{\mu}_w. \] This is the same geometric idea as perceptron: move toward the average signed feature vector of examples your current separator classifies incorrectly. The difference is that SQ replaces one noisy update \(yx\) by a stable average direction.
Why \(\tau=0\) is not allowed. If \(\tau=0\), then the oracle must return the exact value of \(\mathbb E_D[\phi]\) for every bounded query \(\phi\). That is much stronger than what finite samples can provide: by Hoeffding, estimating an expectation to additive error \(\tau\) needs on the order of \(1/\tau^2\) samples, which blows up as \(\tau\to 0\). So exact SQ answers generally require infinitely precise distributional information, not realistic sample access.
Choosing the largest useful tolerance \(\tau\). In SQ learning, larger \(\tau\) is better computationally because simulating one query needs only \(O(\tau^{-2}\log(1/\delta))\) samples. So the right strategy is to make \(\tau\) as large as possible while still preserving the information the learner needs.

Suppose the learner relies on the sign of a correlation \[ \mu=\mathbb E[y\,g(x)], \] and assume \(|\mu|\ge \gamma\) in the clean distribution for some margin or advantage parameter \(\gamma>0\). Under random classification noise at rate \(\eta\lt 1/2\), the observed correlation becomes \[ \mu_{\mathrm{noisy}}=(1-2\eta)\mu, \] so its magnitude is at least \((1-2\eta)\gamma\). If the SQ oracle answers with additive error at most \(\tau\), then to guarantee that the sign is still correct we need \[ \tau < |\mu_{\mathrm{noisy}}|. \] A standard safe choice is to require \[ \tau \le \frac{(1-2\eta)\gamma}{2}, \] because then the oracle error is at most half the available signal, so the returned value must have the same sign as \(\mu_{\mathrm{noisy}}\).

This is the “optimal” tolerance up to constants: if \(\tau\) is much smaller, the algorithm still works but wastes samples; if \(\tau\) is larger than a constant multiple of \((1-2\eta)\gamma\), the noise in the query answer can overwhelm the correlation and the learner can no longer reliably tell which direction is correct. Therefore the largest order of tolerance compatible with correct learning is \[ \tau = \Theta((1-2\eta)\gamma), \] with the convenient explicit choice \[ \tau=\frac{(1-2\eta)\gamma}{2}. \]

Conceptually, the SQ model is meant to capture algorithms that depend on robust statistics. Requiring exact expectations would make the model unrealistically strong and would break the reduction from PAC examples to SQ simulation, because finite data can only approximate expectations, never determine them exactly in general.

Example: estimating a coordinate correlation. For Boolean features \(x\in\{\pm 1\}^d\), take \[ \phi_j(x,y)=y x_j. \] Then \(\mathbb E[yx_j]\) measures how correlated coordinate \(j\) is with the label. A simple SQ-style learner can inspect these correlations and choose a feature whose correlation magnitude is noticeably nonzero. Under random classification noise at rate \(\eta\), the observed statistic becomes \[ \mathbb E[\widetilde y x_j]=(1-2\eta)\mathbb E[yx_j]. \] So the ranking of strong coordinates is unchanged, although the margin shrinks.
SQ learning sits conceptually between PAC learning and noise-tolerant learning. PAC gives the sample framework, agnostic/noisy learning explains why perfect consistency may be impossible, and SQ learning shows that many useful algorithms can be driven entirely by approximate expectations rather than by exact labels on individual examples.
For exam purposes, the key relationship is: \[ \text{examples } \Longrightarrow \text{ simulate SQ queries by averaging,} \] while \[ \text{SQ access } \Longrightarrow \text{ estimate errors/correlations without inspecting single labels directly.} \] So SQ is best viewed as a restricted information model inside PAC, designed to expose which learning procedures rely only on stable statistics.

Part VII: Lower Bounds for Learning

7.1 The No-Free-Lunch Principle

Lower bounds explain why structural assumptions are necessary. If we allow the hypothesis class to be all functions on a large domain, then no learning algorithm can be uniformly successful from a small sample.

No-Free-Lunch theorem. Let \(X\) be a finite domain and let \(A\) be any learning algorithm that receives a sample of size \(m\). Assume \(2m\lt |X|\). Then there exists a labeling function \(f:X\to\{0,1\}\) and a distribution \(D\) on \(X\) such that, with constant probability over the sample, the output hypothesis of \(A\) has constant error.

Proof. Choose a set \(C\subseteq X\) of size \(2m\), and let \(D\) be the uniform distribution on \(C\). Let the sample points be drawn from \(D\). Since the sample has size \(m\), there are always at least \(m\) points of \(C\) that are unseen if the sample points are distinct, and in general a constant fraction of \(C\) remains unseen with nontrivial probability. Denote by \(U\subseteq C\) the set of unseen points.

Fix the realized sample \(S\). The algorithm only observes the labels on the sampled points, so on the unseen set \(U\) it cannot distinguish between two target labelings that agree on \(S\) and disagree on all of \(U\). Let \(f_0,f_1\) be two such labelings. Since \(A\) outputs the same hypothesis after seeing \(S\) under either target, the average of its errors against \(f_0\) and \(f_1\) on the unseen set is exactly \(1/2\). Therefore for at least one of the two labelings, \[ \mathrm{err}_D(A(S))\ge \frac{|U|}{2|C|}. \]

It remains to note that with constant probability the sample leaves a constant fraction of \(C\) unseen. On that event, \[ \mathrm{err}_D(A(S))\ge \frac{|U|}{2|C|}, \] which is bounded below by a positive universal constant. A standard occupancy calculation yields such an event with positive constant probability, proving the theorem.

The proof is pure information limitation. If the learner has never seen a large block of the domain, then on that block there are multiple target functions compatible with the sample. No algorithm can guess correctly on all such completions.

7.2 VC Dimension as a Lower-Bound Mechanism

The No-Free-Lunch theorem becomes a class-dependent lower bound once we restrict attention to a hypothesis class \(H\) with finite VC dimension. If \(H\) shatters a set of size \(d\), then on that shattered set the class behaves like the set of all binary labelings.

VC lower bound from shattering. Let \(H\) be a binary hypothesis class with \(\mathrm{VCdim}(H)=d\). If \(m\lt d/2\), then for every learning algorithm \(A\) there exists a realizable learning problem over \(H\) on which \(A\) has constant error with constant probability.

Proof. Since \(\mathrm{VCdim}(H)=d\), there exists a shattered set \(C=\{x_1,\dots,x_d\}\). Choose any subset \(C'\subseteq C\) of size \(2m\). Because \(C\) is shattered, every labeling of \(C'\) is realized by some hypothesis in \(H\). Therefore the restriction of \(H\) to \(C'\) is exactly the class of all binary functions on \(C'\).

Apply the No-Free-Lunch theorem to the finite domain \(C'\). For the uniform distribution on \(C'\), any learner using only \(m\) examples must fail on some realizable labeling with constant probability and constant error. Since every labeling of \(C'\) is induced by a member of \(H\), this failure instance is realizable by the class \(H\) itself. Hence any learner for \(H\) requires at least on the order of \(d\) samples even to avoid constant error.

This proves the central message: VC dimension is not only an upper-bound parameter through uniform convergence; it is also a lower-bound parameter because shattered sets contain many mutually indistinguishable target functions.

7.3 Realizable and Agnostic Sample-Complexity Lower Bounds

The sharp rates are different in the realizable and agnostic settings. For binary classification with VC dimension \(d\), the sample complexity scales as \[ \Theta\!\left(\frac{d+\log(1/\delta)}{\epsilon}\right) \] in the realizable case and as \[ \Theta\!\left(\frac{d+\log(1/\delta)}{\epsilon^2}\right) \] in the agnostic case, up to logarithmic refinements depending on the exact theorem statement.

Why the agnostic rate has \(1/\epsilon^2\). In the agnostic setting, the learner must distinguish between two distributions whose risks differ by only \(\epsilon\). This is a statistical estimation problem, and the variance of Bernoulli observations forces \(\Theta(1/\epsilon^2)\) samples even before model complexity is considered. The factor \(d\) enters by embedding many such difficult binary decisions into a shattered set of size \(d\).
Realizable learning is easier because a perfect hypothesis exists and the learner only has to identify one consistent explanation. Agnostic learning is harder because even the best hypothesis makes errors, so the learner must estimate and compare close risks rather than just eliminate impossible hypotheses.

7.4 Why Lower Bounds Matter

Lower bounds are not pessimistic decoration; they tell us what kinds of improvements are impossible. If a theorem states that every learner needs at least order \(d/\epsilon\) or \(d/\epsilon^2\) examples, then no clever algorithm can beat that dependence uniformly over the whole class. The only way to improve is to add more structure: margins, sparsity, noise assumptions, compression, distributional assumptions, or a smaller hypothesis class.

Example. Decision stumps can be learned with very small sample complexity because their VC dimension is \(2\). Halfspaces in \(\mathbb R^d\) require sample complexity proportional to \(d\) because their VC dimension is \(d+1\). This is not an artifact of analysis; it is forced by lower bounds.

Part VIII: Compression, Oracles, Domain Adaptation, TDS (Pages 55–61)

Compression learning (p.55–56): A compression scheme is a way to represent a hypothesis by a small subset of the training set (the “compression set”). If the algorithm’s output can be reconstructed from a set of size \(O(\sqrt{m}\,d^3)\) (or similar), one can derive a PAC bound: the probability of a bad hypothesis surviving is controlled by the number of ways to choose that many examples. So we get sample complexity that depends on the compression set size rather than the full class size. This connects to Occam’s razor: simpler (compressible) hypotheses generalize better.

Learning from oracles (p.57–58): Instead of labeled examples, we may have access to a positive oracle \(O_+\) (returns a positive example on request) and a negative oracle \(O_-\) (returns a negative example). The goal is to find a classifier with small error relative to both oracles. Algorithms typically query both oracles and maintain a hypothesis consistent with the examples seen; the sample complexity can be analyzed similarly to PAC.

8.1 Domain Adaptation and the Joint Error (Page 59)

The problem of shift. Standard PAC learning assumes the training and test distributions are the same. Domain adaptation addresses the case where they differ: the learner receives a labeled training set from \(D_{\mathrm{train}}\) and an unlabeled (or partially labeled) test set from \(D_{\mathrm{test}}\). The question is how well we can perform on \(D_{\mathrm{test}}\) when we only have labels from \(D_{\mathrm{train}}\).

The joint error (\(\lambda\)). A fundamental limit of learning under shift is the joint error: the minimum combined error that any single concept in the class can achieve on both distributions: \[ \lambda = \min_{c \in \mathcal{C}} \;\bigl( \mathrm{err}_{D_{\mathrm{train}}}(c) + \mathrm{err}_{D_{\mathrm{test}}}(c) \bigr). \] If \(\lambda\) is large, no single function in \(\mathcal{C}\) can do well on both domains at once—there is an inherent tension between fitting the training domain and the test domain. If \(\lambda = 0\), a perfect classifier exists for both; that special case is often called covariate shift (only the input distribution changes, not the conditional label distribution).

8.2 Generalization Bounds and Discrepancy (Page 60)

Generalization bound under shift. The error on the test distribution can be bounded in terms of the training error, the joint error \(\lambda\), and a measure of how far the two distributions are: \[ \mathrm{err}_{D_{\mathrm{test}}}(h) \le \mathrm{err}_{D_{\mathrm{train}}}(h) + \lambda + \mathrm{dist}(D_{\mathrm{train}}, D_{\mathrm{test}}). \] So if we fit the training set well and the distributions are close (and \(\lambda\) is small), we can still hope for good test performance.

Discrepancy distance. The discrepancy is a distance between \(D_{\mathrm{train}}\) and \(D_{\mathrm{test}}\) that is tailored to the concept class \(\mathcal{C}\). It measures the maximum difference, over pairs of concepts in \(\mathcal{C}\), in the probability that they disagree on a random example: \[ \mathrm{disc}(D, D') = \sup_{f_1, f_2 \in \mathcal{C}} \;\Bigl| \mathbb{P}_{x \sim D}[f_1(x) \neq f_2(x)] - \mathbb{P}_{x \sim D'}[f_1(x) \neq f_2(x)] \Bigr|. \] So we compare how often \(f_1\) and \(f_2\) disagree under \(D\) versus under \(D'\). If this is small, the two distributions “look similar” to the class \(\mathcal{C}\). When \(\lambda = 0\) (a perfect joint classifier exists), the setting is often referred to as covariate shift.

Proof of the generalization bound under shift. We show \(\mathrm{err}_{D_{\mathrm{test}}}(h) \le \mathrm{err}_{D_{\mathrm{train}}}(h) + \lambda + \mathrm{disc}(D_{\mathrm{train}}, D_{\mathrm{test}})\).

Step 1 — Use the best \(c\) to bound test error. For any \(h\) and any \(c \in \mathcal{C}\), the triangle inequality for 0-1 loss gives \(\mathbf{1}[h(x) \neq y] \le \mathbf{1}[h(x) \neq c(x)] + \mathbf{1}[c(x) \neq y]\). Taking expectation over \(D_{\mathrm{test}}\): \[ \mathrm{err}_{D_{\mathrm{test}}}(h) = \mathbb{E}_{D_{\mathrm{test}}}[\mathbf{1}[h \neq y]] \le \mathbb{P}_{D_{\mathrm{test}}}[h \neq c] + \mathrm{err}_{D_{\mathrm{test}}}(c). \] So for every \(c \in \mathcal{C}\), \(\mathrm{err}_{D_{\mathrm{test}}}(h) \le \mathbb{P}_{D_{\mathrm{test}}}[h \neq c] + \mathrm{err}_{D_{\mathrm{test}}}(c)\). We choose \(c = c^*\) that minimizes \(\mathrm{err}_{D_{\mathrm{train}}}(c) + \mathrm{err}_{D_{\mathrm{test}}}(c)\), so \(\lambda = \mathrm{err}_{D_{\mathrm{train}}}(c^*) + \mathrm{err}_{D_{\mathrm{test}}}(c^*)\).

Step 2 — Add and subtract the train error. Write the test-side disagreement as train disagreement plus the difference: \[ \mathbb{P}_{D_{\mathrm{test}}}[h \neq c^*] = \mathbb{P}_{D_{\mathrm{train}}}[h \neq c^*] + \Bigl( \mathbb{P}_{D_{\mathrm{test}}}[h \neq c^*] - \mathbb{P}_{D_{\mathrm{train}}}[h \neq c^*] \Bigr). \] The term in parentheses is bounded by the discrepancy: \(\bigl| \mathbb{P}_{D_{\mathrm{test}}}[h \neq c^*] - \mathbb{P}_{D_{\mathrm{train}}}[h \neq c^*] \bigr| \le \mathrm{disc}(D_{\mathrm{train}}, D_{\mathrm{test}})\).

Step 3 — Relate train disagreement to training loss. On the training distribution, \(\mathbf{1}[h(x) \neq c^*(x)] \le \mathbf{1}[h(x) \neq y] + \mathbf{1}[c^*(x) \neq y]\), so \[ \mathbb{P}_{D_{\mathrm{train}}}[h \neq c^*] \le \mathrm{err}_{D_{\mathrm{train}}}(h) + \mathrm{err}_{D_{\mathrm{train}}}(c^*). \] Step 4 — Rearrange. Putting it together: \[ \mathrm{err}_{D_{\mathrm{test}}}(h) \le \mathbb{P}_{D_{\mathrm{test}}}[h \neq c^*] + \mathrm{err}_{D_{\mathrm{test}}}(c^*) \] \[ \le \mathbb{P}_{D_{\mathrm{train}}}[h \neq c^*] + \mathrm{disc}(D_{\mathrm{train}}, D_{\mathrm{test}}) + \mathrm{err}_{D_{\mathrm{test}}}(c^*) \] \[ \le \mathrm{err}_{D_{\mathrm{train}}}(h) + \mathrm{err}_{D_{\mathrm{train}}}(c^*) + \mathrm{err}_{D_{\mathrm{test}}}(c^*) + \mathrm{disc}(D_{\mathrm{train}}, D_{\mathrm{test}}) = \mathrm{err}_{D_{\mathrm{train}}}(h) + \lambda + \mathrm{disc}(D_{\mathrm{train}}, D_{\mathrm{test}}). \] So test error is bounded by training error plus the joint error \(\lambda\) plus the discrepancy.

8.3 TDS Learning and the Perceptron (Page 61)

Testable learning with distribution shift (TDS). In the TDS model, the learner can certify its performance. The algorithm receives labeled training data from the training domain and unlabeled data from the test domain. It must either ACCEPT—and then guarantee low error on the test distribution—or REJECT when the shift is too large to certify (e.g. when certification would be impossible or unreliable). So we get a testable guarantee: if the algorithm accepts, we have a bound on test error; if it rejects, we know not to trust the model under the observed shift.

Error proof logic. A typical proof uses the triangle inequality to decompose test error. For a hypothesis \(h\), the test error can be written as the sum of (1) the disagreement between \(h\) and the best-fitting function \(c^* \in \mathcal{C}\) on the test distribution, and (2) that function’s own error on the test distribution (which is at most \(\lambda\) when we use the minimizer defining \(\lambda\)). So \(\mathrm{err}_{D_{\mathrm{test}}}(h)\) is bounded by the disagreement with the best \(c \in \mathcal{C}\) plus \(\lambda\). If we can control the disagreement (e.g. via unlabeled test data and the discrepancy), we can certify that test error is low; otherwise we reject.

Bridge to the Perceptron. The notes then connect to practical algorithms for learning halfspaces. The Perceptron (Rosenblatt, 1958) is an online algorithm that learns a linear separator. Under a margin assumption (there exists a unit vector \(w^*\) such that \(y\,(w^* \cdot x) \ge \rho\) for all examples), the Perceptron’s mistakes are bounded by \(1/\rho^2\). So we move from the statistical setting (domain adaptation, discrepancy, TDS) to the mistake-bounded setting: the Perceptron provides a concrete learner with a clean guarantee when data are separable with margin \(\rho\).

Part IX: The Perceptron Algorithm (Pages 62–64)

Halfspaces with margin (p.62): Margin \(\rho\) on the unit sphere. Update rule (p.62): \(w_{\mathrm{new}} = w_{\mathrm{old}} + yx\).

Perceptron margin bound (p.63): \(M \le 1/\rho^2\) using Cauchy–Schwarz (see full proof below).

Polynomial threshold functions (p.64): Higher-order / feature expansion for non-linear boundaries. Example: To learn circles \(x^2 + y^2 \le r^2\), map \((x,y) \mapsto (x^2, y^2)\) or use a feature expansion so that a linear separator in the new space corresponds to the circle in the original space.

Motivation. We want an efficient, mistake-bounded algorithm for halfspaces (linear separators). Linear programming can find a separator in polynomial time, but the perceptron gives a very simple update with a clean mistake bound.

9.1 Function Class: Halfspaces

\(\mathcal{C}\) is the set of halfspaces: \( f(x) = \mathrm{sign}(w \cdot x + \theta) \). For convenience we use labels \(y \in \{-1,+1\}\). We can absorb the threshold into \(w\) by adding a coordinate that is always 1: \(f(x) = \mathrm{sign}(w \cdot x)\). The vector \(w\) is unknown.

Alternative (LP): Finding a separating \(w\) from labeled examples \((x_i, y_i)\) can be written as linear inequalities \(y_i(w \cdot x_i) \ge 0\) (or \(\ge 1\) for margin). Solving this with linear programming is polynomial-time but more expensive than the simple perceptron update.

9.2 Algorithm

  • Initialize \(w\) (e.g. \(\mathbf{0}\) or any unit vector).
  • On example \(x\): predict \(\widehat{y} = \mathrm{sign}(w \cdot x)\).
  • On a mistake: update \( w_{\mathrm{new}} = w_{\mathrm{old}} + y\, x \), where \(y\) is the true label. (If we predicted +1 but true was \(-1\), then \(y=-1\) so we subtract \(x\); if we predicted \(-1\) but true was \(+1\), we add \(x\).)
Unified update: on every mistake, \( w \leftarrow w + y\, x \).
Why add \(yx\)? We want \(w \cdot x\) to have the same sign as \(y\). If we predicted +1 but \(y = -1\), then \(w \cdot x \gt 0\); adding \(yx = -x\) reduces \(w \cdot x\) for this \(x\) (and similar points). If we predicted -1 but \(y = +1\), then \(w \cdot x \lt 0\); adding \(yx = +x\) increases \(w \cdot x\). So we nudge \(w\) in the direction that would have made our prediction correct.

9.3 Margin Assumption

There exists a unit-norm separator \(w^*\) (\(\|w^*\| = 1\)) and a margin \(\rho \gt 0\) such that for every example \((x,y)\), \( y\,(w^* \cdot x) \ge \rho \). We assume \(\|x\| \le 1\) (or \(\|x\| = 1\) for simplicity). Then every point is at distance at least \(\rho\) from the separating hyperplane \(\{ z : w^* \cdot z = 0 \}\).
Why margin? If data are only barely separable (\(y\,(w^* \cdot x)\) close to 0), the perceptron might need many small nudges. A large \(\rho\) means data are “far” from the boundary, so each update makes more progress and we get a finite mistake bound.

9.4 Perceptron Convergence Theorem (Mistake Bound \(O(1/\rho^2)\))

Theorem. Under the margin assumption (and \(\|x\| = 1\)), the perceptron makes at most \(1/\rho^2\) mistakes.

Proof idea. We track two quantities: (1) \(w \cdot w^*\)—how aligned our weight is with the true separator. Each mistake increases this by at least \(\rho\). (2) \(\|w\|^2\)—how large our weight is. Each mistake increases this by at most 1. So after \(t\) mistakes we have \(w \cdot w^* \ge t\rho\) and \(\|w\|^2 \le t\). By Cauchy–Schwarz, \(w \cdot w^* \le \|w\|\). So \(t\rho \le \sqrt{t}\), hence \(t \le 1/\rho^2\).
Proof (step by step). Let \(w\) be the current weight vector and \(w^*\) the true unit vector. Consider one mistake on \((x,y)\).

Claim 1: On every mistake, \(w \cdot w^*\) increases by at least \(\rho\).
After the update, \(w' = w + yx\). So \[ w' \cdot w^* = (w + yx) \cdot w^* = w \cdot w^* + y\,(x \cdot w^*). \] The margin assumption says \(y\,(x \cdot w^*) \ge \rho\). So \(w' \cdot w^* \ge w \cdot w^* + \rho\). So each mistake adds at least \(\rho\) to the alignment with \(w^*\). After \(t\) mistakes, \(w \cdot w^* \ge t\rho\) (assuming we start from 0 or positive).

Claim 2: On every mistake, \(\|w\|^2\) increases by at most 1.
\[ \|w'\|^2 = \|w + yx\|^2 = \|w\|^2 + 2y\,(w \cdot x) + y^2\|x\|^2. \] We had a mistake, so our prediction \(\mathrm{sign}(w \cdot x)\) was wrong. So \(y\) and \(w \cdot x\) have opposite signs, i.e. \(y\,(w \cdot x) \le 0\). Also \(y^2 = 1\) and \(\|x\|^2 = 1\). So \[ \|w'\|^2 \le \|w\|^2 + 0 + 1 = \|w\|^2 + 1. \] After \(t\) mistakes, \(\|w\|^2 \le t\), so \(\|w\| \le \sqrt{t}\).

Synthesis. After \(t\) mistakes we have \(w \cdot w^* \ge t\rho\) and \(\|w\| \le \sqrt{t}\). By the Cauchy–Schwarz inequality, \(w \cdot w^* \le \|w\|\,\|w^*\| = \|w\| \cdot 1 = \|w\|\). So \[ t\rho \le w \cdot w^* \le \|w\| \le \sqrt{t}. \] Thus \(t\rho \le \sqrt{t}\). Dividing by \(\sqrt{t}\): \(\rho\sqrt{t} \le 1\), so \(\sqrt{t} \le 1/\rho\), hence \(t \le 1/\rho^2\). So the number of mistakes is at most \(1/\rho^2\).

Note. The bound is dimension-independent. If \(\rho\) is very small (e.g. exponentially small in dimension), the bound can be large. If \(\rho\) is constant or inverse polynomial, we get a polynomial mistake bound.

Why perceptron can have an infinite mistake bound on a fixed dataset. If the data are not linearly separable, repeatedly cycling through the same finite training set can cause the perceptron to keep making mistakes forever.

A minimal example is the one-point contradictory dataset \[ S=\{(x,+1),(x,-1)\} \] for any nonzero vector \(x\). No linear classifier can label the same input both positive and negative, so the dataset is not realizable by a halfspace.

Start from \(w=0\). On the first copy \((x,+1)\), the perceptron predicts incorrectly and updates to \(w \leftarrow w + x\). On the second copy \((x,-1)\), it now predicts \(+1\), makes a mistake, and updates to \(w \leftarrow w - x = 0\). We have returned to the initial state. Thus every full pass through the dataset repeats the same two mistakes, so the mistake process never terminates.

The perceptron convergence theorem requires separability with positive margin. Once that assumption fails, the algorithm is no longer guaranteed to settle down.

9.5 Kernel Perceptron

To learn non-linear boundaries, map \(x \mapsto \phi(x)\) into a high-dimensional feature space and run perceptron in that space. The weight vector is then a linear combination of the mistaken examples: \(w = \sum_j \alpha_j y_j \phi(x_j)\), so predictions are \(\mathrm{sign}\bigl(\sum_j \alpha_j y_j \,\phi(x_j) \cdot \phi(x)\bigr)\). The update only needs inner products \(\phi(x) \cdot \phi(z)\). A kernel \(K(x,z) = \phi(x) \cdot \phi(z)\) computes this without explicitly constructing \(\phi\). Example: \(K(x,z) = (x \cdot z)^2\) corresponds to a quadratic feature map; we get polynomial boundaries with cost in the original dimension.

Kernel trick. We never need to store or compute \(\phi(x)\); we only need \(K(x,z)\). For polynomial kernels \(K(x,z) = (x \cdot z)^d\), the implicit \(\phi\) has dimension \(O(n^d)\), but \(K(x,z)\) is computed in \(O(n)\) time.

Part X: Linear Regression and Regularization (Pages 65–74)

Motivation. In supervised regression we observe pairs \((x^1, y^1), \ldots, (x^m, y^m)\) and want to fit a function \(f(x)\) so that predictions \(f(x^i)\) are close to the labels \(y^i\). The standard approach is to minimize squared error; we then add regularization to control overfitting and study how that affects bias and variance.

10.1 Linear Regression Foundations (Pages 65–67)

Objective. We choose \(f\) to minimize the mean squared error on the data: \[ \frac{1}{m} \sum_{i=1}^{m} \bigl(f(x^i) - y^i\bigr)^2. \] When we restrict to linear functions \(f(x) = w^T x\) (or \(f(x) = \beta_0 + \beta_1 x\) in one dimension), this becomes a minimization over the weight vector \(w\).

Optimal function in principle. The function that minimizes mean squared error over all (not necessarily linear) predictors is the conditional mean: \(f^*(x) = \mathbb{E}[y \mid x]\). Linear regression is the best linear approximation to this when we work in the space of linear functions.

Why the conditional mean minimizes squared loss. For any measurable function \(g(X)\), \[ Y-g(X)=\bigl(Y-\mathbb{E}[Y\mid X]\bigr)+\bigl(\mathbb{E}[Y\mid X]-g(X)\bigr). \] Squaring and taking expectation gives \[ \mathbb{E}\bigl[(Y-g(X))^2\bigr] = \mathbb{E}\bigl[(Y-\mathbb{E}[Y\mid X])^2\bigr] + \mathbb{E}\bigl[(\mathbb{E}[Y\mid X]-g(X))^2\bigr] + 2\,\mathbb{E}\Bigl[(Y-\mathbb{E}[Y\mid X])(\mathbb{E}[Y\mid X]-g(X))\Bigr]. \] The cross term is \(0\), because \(\mathbb{E}[Y-\mathbb{E}[Y\mid X]\mid X]=0\). Hence \[ \mathbb{E}\bigl[(Y-g(X))^2\bigr] = \mathbb{E}\bigl[(Y-\mathbb{E}[Y\mid X])^2\bigr] + \mathbb{E}\bigl[(\mathbb{E}[Y\mid X]-g(X))^2\bigr] \ge \mathbb{E}\bigl[(Y-\mathbb{E}[Y\mid X])^2\bigr]. \] Therefore \(g^\star(X)=\mathbb{E}[Y\mid X]\) minimizes mean squared error over all predictors \(g\).

Simple linear regression (scalar \(x\)). Take \(f(x) = \beta_0 + \beta_1 x\). Setting the partial derivatives of \(\sum_i (y_i - \beta_0 - \beta_1 x_i)^2\) to zero gives the closed-form solutions: \[ \beta_1 = \frac{\mathrm{cov}(x,y)}{\mathrm{var}(x)}, \qquad \beta_0 = \bar{y} - \beta_1 \bar{x}. \] So the slope is the covariance of \(x\) and \(y\) normalized by the variance of \(x\); the intercept centers the line at the means.

10.2 Matrix Form and Normal Equations (Pages 68–69)

With \(X\) the \(m \times d\) matrix of inputs and \(y\) the vector of labels, we minimize \(\|Xw - y\|_2^2\). Geometrically, we are projecting the vector \(y\) onto the column space (span) of \(X\). The optimal \(w\) is the one for which the residual vector \((y - Xw)\) is orthogonal to every column of \(X\)—that is, the residual lies in the orthogonal complement of the column space. So \(X^T(y - Xw) = 0\), which rearranges to the normal equations: \[ X^T X w = X^T y \quad\Longrightarrow\quad w = (X^T X)^{-1} X^T y \] (when \(X^T X\) is invertible).

Maximum likelihood and OLS. If we assume the data are generated as \(y^i = w^T x^i + \epsilon^i\) with \(\epsilon^i \sim N(0, \sigma^2)\) i.i.d., then the log-likelihood of the data is a constant minus \(\frac{1}{2\sigma^2}\sum_i (y_i - w^T x_i)^2\). Maximizing the log-likelihood is therefore equivalent to minimizing the sum of squared residuals. So under Gaussian noise, MLE and ordinary least squares (OLS) give the same solution.

10.3 Ordinary Least Squares (OLS) — Gradient Derivation

Minimize \(\|Xw - y\|^2 = \sum_i (x_i^T w - y_i)^2\). Taking the gradient with respect to \(w\):

\[ \nabla_w \|Xw - y\|^2 = 2X^T(Xw - y). \]

Setting to zero gives the normal equations \(X^T X w = X^T y\). If \(X^T X\) is invertible, \(w = (X^T X)^{-1} X^T y\). If not (e.g. \(n \lt d\) or collinearity), the solution is not unique or we use Ridge.

Why the normal equations? We are projecting \(y\) onto the column space of \(X\). The residual \(Xw - y\) must be orthogonal to every column of \(X\), i.e. \(X^T(Xw - y) = 0\). That is exactly \(X^T X w = X^T y\).

10.4 Ridge Regression (L2) and Regularization (Pages 70–72)

Objective. To reduce overfitting or handle ill-posed problems (e.g. \(X^T X\) singular), we add a penalty on the norm of \(w\): \[ \min_w \;\|Xw - y\|_2^2 + \lambda \|w\|_2^2. \] Taking the gradient and setting it to zero gives \(X^T(Xw - y) + \lambda w = 0\), i.e. \((X^T X + \lambda I) w = X^T y\), so \[ w = (X^T X + \lambda I)^{-1} X^T y. \] Adding \(\lambda I\) ensures the matrix is invertible even when \(X^T X\) is not; it also shrinks the weights toward zero.

Bias–variance in a toy model (p.71–72). For a model of the form \(y^i = a^* x^i + \epsilon^i\) with \(\epsilon^i \sim N(0, \sigma^2)\), the Ridge estimator’s mean squared error decomposes into variance plus bias squared. With \(\beta = \frac{1}{m}\sum_i (x^i)^2\), one has \[ \mathrm{Variance} = \frac{m\beta\sigma^2}{(\lambda + m\beta)^2}, \qquad \mathrm{Bias}^2 = \frac{\lambda^2 (a^*)^2}{(\lambda + m\beta)^2}. \] So \(\mathrm{MSE} = \mathrm{Variance} + \mathrm{Bias}^2\). When \(\lambda = 0\) (OLS), variance can be large and bias zero; when \(\lambda \to \infty\), variance goes to zero but bias grows. The optimal \(\lambda^*\) that minimizes MSE in this setting is \[ \lambda^* = \frac{\sigma^2}{(a^*)^2}. \] So when there is noise (\(\sigma^2 \gt 0\)), a positive \(\lambda\) can achieve lower MSE than OLS—Ridge can outperform unregularized least squares by trading a little bias for a reduction in variance.

Why Ridge helps. When \(X^T X\) is nearly singular, small changes in \(y\) can cause huge changes in \(w\). Adding \(\lambda I\) stabilizes the inverse and shrinks \(w\) toward zero; the optimal \(\lambda^*\) balances variance and bias.

10.5 Lasso (L1)

Minimize \(\|Xw - y\|^2 + \lambda \|w\|_1\). The L1 penalty \(\|w\|_1 = \sum_j |w_j|\) drives many coefficients to exactly zero (sparse solutions). No closed form; optimize with (sub)gradient or coordinate descent.

Why sparsity? The L1 ball has “corners” on the axes; when the level sets of the loss hit such a corner, some \(w_j\) become exactly 0. L2 (Ridge) gives a smooth ball, so weights shrink but rarely hit zero exactly.

10.6 Bias–Variance Decomposition

For an estimator \(\hat{a}\) of a quantity \(a\), expand \(\mathbb{E}[(\hat{a} - a)^2]\):

Derivation. Write \(\hat{a} - a = (\hat{a} - \mathbb{E}[\hat{a}]) + (\mathbb{E}[\hat{a}] - a)\). Then \[ (\hat{a} - a)^2 = (\hat{a} - \mathbb{E}[\hat{a}])^2 + (\mathbb{E}[\hat{a}] - a)^2 + 2(\hat{a} - \mathbb{E}[\hat{a}])(\mathbb{E}[\hat{a}] - a). \] Taking expectation: the cross term vanishes because \(\mathbb{E}[\hat{a} - \mathbb{E}[\hat{a}]] = 0\). So \[ \mathrm{MSE}(\hat{a}) = \mathbb{E}[(\hat{a} - a)^2] = \mathrm{Var}(\hat{a}) + \bigl(\mathbb{E}[\hat{a}] - a\bigr)^2 = \mathrm{Var}(\hat{a}) + \mathrm{Bias}(\hat{a})^2. \]

OLS (\(\lambda = 0\)): unbiased, possibly high variance. Ridge/Lasso (\(\lambda \gt 0\)): bias toward zero, lower variance. Optimal \(\lambda\) minimizes MSE.

Tradeoff. As \(\lambda\) increases, the estimator is pulled toward zero (bias increases) but becomes less sensitive to the sample (variance decreases). The best \(\lambda\) balances the two.

10.7 Kernel Methods (Pages 73–74)

Feature expansion. Linear boundaries in the original space may be inadequate. We can map inputs \(x\) to a higher-dimensional vector \(\phi(x)\) (e.g. quadratic features like \(x_1^2, x_1 x_2, x_2^2\)) and fit a linear model in that space. That yields non-linear decision boundaries in the original \(x\)-space.

The kernel trick. Many algorithms (including regression and the perceptron) only need inner products \(\langle \phi(x^1), \phi(x^2) \rangle\). A kernel function \(K(x^1, x^2)\) is defined to equal this inner product in some feature space: \(K(x^1, x^2) = \langle \phi(x^1), \phi(x^2) \rangle\). We can then work in the high-dimensional space without ever forming \(\phi(x)\) explicitly—we only evaluate \(K\). For example, \(K(x,z) = (x \cdot z)^2\) corresponds to a quadratic feature map.

Weight representation. In the feature space, the optimal weight vector \(w\) can be expressed as a linear combination of the training instance mappings: \(w = \sum_i y^i \phi(x^i)\) (with coefficients given by the labels or by the dual solution). Prediction on a new point \(z\) is then a sum of kernel evaluations: \(f(z) = \sum_i y^i K(x^i, z)\). We never need to store or compute the high-dimensional \(\phi\); we only need the kernel \(K\).

Part XI: Optimization & Loss (Pages 75–84)

Motivation. Training a model usually means choosing parameters \(w\) that minimize a loss function \(f(w)\). We do this iteratively: from the current \(w\), we take a step that reduces \(f\). This section explains why moving in the direction of the negative gradient works, how to do it (gradient descent and its stochastic variant), how fast we converge under smoothness, and how common loss functions and logistic regression fit into this picture.

11.1 Convexity and the Intuition of Gradient Descent (Pages 75–76)

Convexity. A function \(f\) is convex if the graph lies below the chord between any two points. In other words, the value at the midpoint is at most the average of the values at the endpoints: \[ f\bigl(\tfrac{1}{2}x_1 + \tfrac{1}{2}x_2\bigr) \le \tfrac{1}{2}f(x_1) + \tfrac{1}{2}f(x_2). \] (More generally, \(f(\alpha x_1 + (1-\alpha)x_2) \le \alpha f(x_1) + (1-\alpha)f(x_2)\) for \(\alpha \in [0,1].\)) For differentiable \(f\), an equivalent characterization is the first-order condition: \(f\) is convex if and only if \[ f(y) \ge f(x) + \nabla f(x)^T (y - x) \] for all \(x, y\). So the linear approximation at \(x\) is a global under-estimator of \(f\). A consequence: any local minimum of a convex function is a global minimum.

Why the negative gradient? To minimize \(f\), we want to move in a direction that decreases \(f\) as much as possible. For a small step \(\epsilon d\) with \(\|d\| = 1\), Taylor expansion gives \(f(x + \epsilon d) \approx f(x) + \epsilon \nabla f(x)^T d\). To make this decrease, we want \(\nabla f(x)^T d\) as negative as possible. That is achieved by choosing \(d\) in the direction of \(-\nabla f(x)\). So moving in the direction of the negative gradient is the direction of steepest descent. For a sufficiently small step size, such a step is guaranteed to reduce \(f\).

11.2 The Gradient Descent Algorithm (Page 77)

Protocol.

  1. Initialize \(w\) (e.g. at random).
  2. If \(\|\nabla f(w)\|_2\) is below a chosen threshold, stop (we are at an approximate stationary point, i.e. near a local minimum for convex \(f\)).
  3. Otherwise, update: \(w_{\mathrm{new}} = w_{\mathrm{old}} - \eta \nabla f(w)\), where \(\eta \gt 0\) is the step size. Then repeat from step 2.

Quadratic example. For \(f(x) = x^T A x - b^T x\) (with \(A\) symmetric), the gradient is \(\nabla f(x) = 2Ax - b\). So the GD update is \(x \leftarrow x - \eta(2Ax - b)\). For convex quadratics (e.g. \(A \succeq 0\)), this converges to the minimizer when \(\eta\) is chosen appropriately.

11.3 Gradient Descent vs. Stochastic Gradient Descent (Page 78)

Batch gradient for MSE. In linear regression we minimize the mean squared error \(\mathrm{MSE}(w) = \frac{1}{m}\sum_{j=1}^m (w^T x^j + b - y^j)^2\). The gradient is the average of the per-example gradients: \[ \nabla \mathrm{MSE}(w) = \frac{2}{m} \sum_{j=1}^{m} (w^T x^j + b - y^j)\, x^j. \] Batch gradient descent uses this full gradient at each iteration—expensive when \(m\) is large.

Stochastic gradient descent (SGD). Instead of summing over all \(m\) points, SGD picks one index \(j\) uniformly at random and updates \(w\) using only that example’s gradient. So each update is much cheaper. The key fact: the expected value of the SGD update equals the full GD update—\(\mathbb{E}[w_{\mathrm{new},\mathrm{SGD}}] = w_{\mathrm{new},\mathrm{GD}}\) when the expectation is over the random choice of \(j\). So SGD is an unbiased, efficient approximation to full gradient descent and is standard for large datasets.

Sigmoid perceptron with MSE. Suppose labels are \(y_i\in\{0,1\}\), the score is \(w^T x_i\), and the prediction is the sigmoid \[ \sigma(z)=\frac{1}{1+e^{-z}}. \] The mean squared error on a dataset \(S=\{(x_i,y_i)\}_{i=1}^m\) is \[ \mathrm{MSE}(w)=\frac{1}{m}\sum_{i=1}^m \bigl(\sigma(w^T x_i)-y_i\bigr)^2. \]

Differentiating the sigmoid gives \[ \sigma'(z)=\frac{e^{-z}}{(1+e^{-z})^2} =\frac{1}{1+e^{-z}}\left(1-\frac{1}{1+e^{-z}}\right) =\sigma(z)\bigl(1-\sigma(z)\bigr). \] Therefore, by the chain rule, \[ \nabla \mathrm{MSE}(w) = \frac{2}{m}\sum_{i=1}^m \bigl(\sigma(w^T x_i)-y_i\bigr)\, \sigma(w^T x_i)\bigl(1-\sigma(w^T x_i)\bigr)\,x_i. \] Gradient descent with step size \(\eta\gt 0\) becomes \[ w \leftarrow w - \eta\, \frac{2}{m}\sum_{i=1}^m \bigl(\sigma(w^T x_i)-y_i\bigr)\, \sigma(w^T x_i)\bigl(1-\sigma(w^T x_i)\bigr)\,x_i. \] This is the batch update rule for a single sigmoid unit trained under mean squared error.

11.4 Smoothness and Convergence Analysis (Pages 79–81)

\(L\)-smoothness. A differentiable function \(f\) has \(L\)-Lipschitz gradients (or is \(L\)-smooth) if \[ \|\nabla f(x) - \nabla f(y)\| \le L \|x - y\|_2 \] for all \(x, y\). So the gradient does not change too rapidly; the slope of \(f\) is “controlled” by \(L\).

Descent lemma. For an \(L\)-smooth function, if we take a step of size \(\eta = 1/L\) in the negative gradient direction—\(w^{k+1} = w^k - \frac{1}{L}\nabla f(w^k)\)—then the function value decreases by at least a fixed amount at each step: \[ f(w^{k+1}) \le f(w^k) - \frac{1}{2L} \|\nabla f(w^k)\|^2. \] So each step guarantees a decrease of at least \(\frac{1}{2L}\|\nabla f(w^k)\|^2\).

Convergence rate. Summing this inequality over \(k = 1, \ldots, T\) and using \(f(w^{T+1}) \ge f(w^*)\) (where \(w^*\) is a minimizer) gives \(\sum_{k=1}^T \|\nabla f(w^k)\|^2 \le 2L\bigl(f(w^1) - f(w^*)\bigr)\). So the sum of squared gradient norms is bounded. In particular, the minimum squared gradient over those \(T\) iterations satisfies \[ \min_{j=1,\ldots,T} \|\nabla f(w^j)\|^2 \le \frac{2L\bigl(f(w^1) - f(w^*)\bigr)}{T}. \] So we get an \(O(1/T)\) rate: after \(T\) steps, at least one iterate has gradient norm squared of order \(1/T\). To reach a point where \(\|\nabla f(w)\|^2 \le \epsilon\), we need \(T = O(1/\epsilon)\) iterations.

Subgradient (p.79). For non-differentiable losses (e.g. hinge \(\max\{0, 1-z\}\)), we use a subgradient: a vector \(v\) such that \(f(w') \ge f(w) + v^T(w' - w)\) for all \(w'\). Subgradient descent uses \(v\) in place of \(\nabla f(w)\) when the gradient does not exist.
Perceptron as SGD on the loss \(\ell(z)=\max(0,-z)\). Consider binary labels \(y_i\in\{\pm1\}\) and objective \[ \frac{1}{m}\sum_{i=1}^m \ell\bigl(y_i w^T x_i\bigr), \qquad \ell(z)=\max(0,-z). \] For one example \((x_i,y_i)\), the loss is \[ \ell_i(w)=\max(0,-y_i w^T x_i). \] A subgradient is \[ \partial \ell_i(w)= \begin{cases} \{0\}, & y_i w^T x_i \gt 0,\\ \{-y_i x_i\}, & y_i w^T x_i \lt 0. \end{cases} \] Therefore an SGD step with step size \(\eta\) is \[ w \leftarrow \begin{cases} w, & y_i w^T x_i \gt 0,\\ w + \eta y_i x_i, & y_i w^T x_i \lt 0. \end{cases} \]

This is exactly the perceptron update rule, up to the learning-rate factor \(\eta\). In particular, when \(\eta=1\), stochastic subgradient descent on the loss \(\ell(z)=\max(0,-z)\) coincides with the perceptron algorithm.

The perceptron is not just an ad hoc rule: it is SGD on a convex loss that penalizes only currently misclassified examples.

11.5 Loss Functions and Logistic Regression (Pages 82–84)

Loss comparison. In classification we predict with \(\mathrm{sign}(w^T x)\) and measure quality with a loss \(\ell(z)\) where \(z = y\, (w^T x)\) (so a correct prediction has \(z \gt 0\)).

  • 0/1 loss: \(\ell(z) = \mathbf{1}[z \le 0]\)—counts mistakes directly. It is non-convex and non-smooth, so hard to optimize with gradient methods.
  • Hinge loss: \(\ell(z) = \max\{0, 1 - z\}\). Convex upper bound on 0/1 loss; used in SVMs. Non-differentiable at \(z = 1\).
  • Logistic loss: \(\ell(z) = \ln(1 + e^{-z})\). Convex and smooth; well-suited for gradient descent.
  • Square loss: \(\ell(z) = (1-z)^2\). Smooth and convex; often used in regression-style classification.

Logistic regression. We model the probability of the positive class with the sigmoid \(\sigma(t) = \frac{1}{1 + e^{-t}}\). So \(\mathbb{P}[y = 1 \mid x] = \sigma(w \cdot x)\). Given data \((x^i, y^i)\), we maximize the probability of the observed labels, or equivalently minimize the negative log-likelihood: \[ -\sum_i \ln\bigl(1 + e^{-y^i w^T x^i}\bigr). \] This is a sum of logistic losses (one per example). The function is convex in \(w\) (the Hessian is positive semi-definite), so gradient descent finds the global minimum—the maximum likelihood estimate. Thus logistic regression reduces to convex optimization and can be solved efficiently with GD or SGD.

11.6 Norms, Errors, and Losses: A Reference Sheet

A norm measures the size of a vector or matrix. A loss measures the penalty for a prediction. In many proofs, the norm controls geometry while the loss controls optimization and statistical risk.

Main vector norms. For \(x=(x_1,\dots,x_d)\in\mathbb{R}^d\), \[ \|x\|_1 = \sum_{j=1}^d |x_j|, \qquad \|x\|_2 = \Bigl(\sum_{j=1}^d x_j^2\Bigr)^{1/2}, \qquad \|x\|_\infty = \max_j |x_j|. \] More generally, for \(1\le p\lt\infty\), \[ \|x\|_p = \Bigl(\sum_{j=1}^d |x_j|^p\Bigr)^{1/p}. \]
\(\|x\|_1\) measures total magnitude and promotes sparsity, \(\|x\|_2\) measures Euclidean length and is rotation invariant, and \(\|x\|_\infty\) measures the largest coordinate error. In finite dimensions all these norms are equivalent, but they emphasize different geometry.
Norm comparisons in \(\mathbb{R}^d\). For every \(x\in\mathbb{R}^d\), \[ \|x\|_\infty \le \|x\|_2 \le \|x\|_1 \le \sqrt{d}\,\|x\|_2 \le d\,\|x\|_\infty. \] Proof. Since \(\max_j |x_j|^2 \le \sum_j x_j^2\), we get \(\|x\|_\infty \le \|x\|_2\). Also \[ \|x\|_2^2=\sum_j x_j^2 \le \Bigl(\sum_j |x_j|\Bigr)^2=\|x\|_1^2, \] so \(\|x\|_2\le \|x\|_1\). By Cauchy-Schwarz, \[ \|x\|_1=\sum_j |x_j|\cdot 1 \le \Bigl(\sum_j x_j^2\Bigr)^{1/2}\Bigl(\sum_j 1^2\Bigr)^{1/2} =\sqrt{d}\,\|x\|_2. \] Finally, \(\|x\|_1\le d\,\|x\|_\infty\), hence \(\|x\|_2\le \|x\|_1\le d\,\|x\|_\infty\). These inequalities are used constantly when converting one error guarantee into another.
Main matrix norms. For \(A=(A_{ij})\in\mathbb{R}^{m\times n}\), \[ \|A\|_F = \Bigl(\sum_{i,j} A_{ij}^2\Bigr)^{1/2}, \qquad \|A\|_{2\to 2} = \sup_{\|x\|_2=1}\|Ax\|_2. \] The Frobenius norm is the Euclidean norm of the entries. The operator norm is the largest stretch of a unit vector. Under SVD, \[ \|A\|_F^2=\sum_i \sigma_i^2, \qquad \|A\|_{2\to 2}=\sigma_1. \]
When each norm appears.

\(\text{Ridge} \longleftrightarrow \|w\|_2^2\).

\(\text{Lasso} \longleftrightarrow \|w\|_1\).

\(\text{Margin bounds / geometry} \longleftrightarrow \|w\|_2\).

\(\text{Matrix approximation} \longleftrightarrow \|A-B\|_F\) or \(\|A-B\|_{2\to2}\).

If a problem is about sparsity, expect \(\ell_1\). If it is about Euclidean geometry, projections, or spectral behavior, expect \(\ell_2\) or operator norm.
Regression losses. If the true response is \(y\) and the prediction is \(\hat y\), common losses are \[ \ell_{\mathrm{L1}}(\hat y,y)=|\hat y-y|, \qquad \ell_{\mathrm{L2}}(\hat y,y)=(\hat y-y)^2, \qquad \ell_{\mathrm{abs}}(\hat y,y)=|\hat y-y|. \] On a sample \((x_i,y_i)\), these produce \[ \text{MAE}=\frac1m\sum_{i=1}^m |\hat y_i-y_i|, \qquad \text{MSE}=\frac1m\sum_{i=1}^m (\hat y_i-y_i)^2, \qquad \text{RMSE}=\sqrt{\text{MSE}}. \]

Why L1 and L2 behave differently.

Squared loss penalizes large residuals more strongly because the penalty grows quadratically. Absolute loss grows only linearly, so it is more robust to outliers. The optimizer of expected squared loss is the conditional mean, while the optimizer of expected absolute loss is a conditional median.

Proof for absolute loss. Fix a real random variable \(Y\) and consider

\[ \phi(a)=\mathbb{E}|Y-a|. \]

For differentiable points of \(\phi\),

\[ \phi'(a)=\mathbb{P}(Y\lt a)-\mathbb{P}(Y\gt a). \]

Thus \(a\) is optimal exactly when the mass on each side is at most \(1/2\), which is the defining property of a median.

Example. If the residuals are \(1,1,1,10\), then \[ \text{MAE}=\frac{13}{4}=3.25, \qquad \text{MSE}=\frac{1+1+1+100}{4}=25.75. \] The outlier \(10\) dominates the squared loss much more than the absolute loss.
Classification losses in margin form. Let \(z=y\,f(x)\), where \(y\in\{\pm1\}\). Then \[ \ell_{0/1}(z)=\mathbf{1}[z\le 0], \qquad \ell_{\mathrm{hinge}}(z)=\max(0,1-z), \] \[ \ell_{\mathrm{log}}(z)=\log(1+e^{-z}), \qquad \ell_{\mathrm{exp}}(z)=e^{-z}, \qquad \ell_{\mathrm{square}}(z)=(1-z)^2. \]
What each classification loss is for.
  • \(0/1\) loss measures actual mistakes but is hard to optimize directly.
  • Hinge loss is convex and upper-bounds \(0/1\) loss, so it is useful for margin methods such as SVMs.
  • Logistic loss is convex and smooth, so it is standard for gradient methods and probabilistic modeling.
  • Exponential loss is the natural loss behind AdaBoost.
  • Square loss is algebraically convenient but can over-penalize large negative margins.
Margin-loss comparison at one point. If \(z=2\), then \[ \ell_{0/1}(2)=0,\quad \ell_{\mathrm{hinge}}(2)=0,\quad \ell_{\mathrm{log}}(2)=\log(1+e^{-2}),\quad \ell_{\mathrm{exp}}(2)=e^{-2}. \] If \(z=-1\), then \[ \ell_{0/1}(-1)=1,\quad \ell_{\mathrm{hinge}}(-1)=2,\quad \ell_{\mathrm{log}}(-1)=\log(1+e),\quad \ell_{\mathrm{exp}}(-1)=e. \] So all surrogate losses penalize a negative margin, but they do so with different severity and smoothness.
Huber loss. To interpolate between L1 and L2, define for parameter \(\delta\gt 0\) \[ \ell_\delta(r)= \begin{cases} \frac12 r^2, & |r|\le \delta,\\ \delta |r|-\frac12\delta^2, & |r|\gt \delta. \end{cases} \] It is quadratic near \(0\) and linear in the tails, so it is smooth near the optimum but more robust to outliers than squared loss.
Cheat-sheet rule. Use \(\ell_2\) when you want smooth calculus, projections, or Gaussian models. Use \(\ell_1\) when you want robustness or sparsity. Use hinge/logistic/exponential when you are replacing hard classification error by a tractable surrogate. Use operator norm for worst-case stretching and Frobenius norm for entrywise or energy-style error.

11.7 Major Inequalities: A Proof Toolkit

Many arguments in learning theory reduce to choosing the right inequality. The recurring pattern is: bound a hard quantity by a simpler one, then simplify that bound using convexity, orthogonality, or independence.

Jensen's inequality. If \(f\) is convex and \(X\) is integrable, then \[ f(\mathbb{E}X)\le \mathbb{E}[f(X)]. \] If \(f\) is concave, the inequality reverses: \[ f(\mathbb{E}X)\ge \mathbb{E}[f(X)]. \]
How Jensen is used. Jensen converts averages inside a nonlinear function into averages outside it. In decision trees, if \(\Phi\) is a concave impurity function and a node with class-probability vector \(p\) is split into children with weights \(\alpha_j\) and class vectors \(p^{(j)}\), then \[ p=\sum_j \alpha_j p^{(j)} \qquad\Longrightarrow\qquad \Phi(p)\ge \sum_j \alpha_j \Phi\bigl(p^{(j)}\bigr). \] Therefore weighted impurity cannot increase after splitting. This is the clean proof that further splitting helps for entropy and Gini impurity.
Example. For binary entropy \[ H(p)=-p\log p-(1-p)\log(1-p), \] concavity gives \[ H\Bigl(\sum_j \alpha_j p_j\Bigr)\ge \sum_j \alpha_j H(p_j). \] So the parent entropy is at least the weighted average child entropy.
Cauchy-Schwarz. For vectors \(x,y\in\mathbb{R}^d\), \[ |\langle x,y\rangle|\le \|x\|_2\|y\|_2. \] Equality holds iff one vector is a scalar multiple of the other.
How Cauchy-Schwarz is used. This is the default way to turn an inner product into norm bounds. It appears in margin arguments, norm comparisons, regression, and SVD. For example, if \(\|w\|_2\le B\) and \(\|x\|_2\le R\), then \[ |w^\top x|\le BR. \] This is the standard step when bounding predictions, margins, gradients, or Lipschitz constants.
Example. In the perceptron proof, \[ w^{(t)}\cdot w^* \le \|w^{(t)}\|_2\,\|w^*\|_2. \] After lower-bounding the left-hand side by \(t\rho\), this yields an upper bound on the number of mistakes.
Triangle inequality and Minkowski. \[ \|x+y\|_p\le \|x\|_p+\|y\|_p. \] In particular, \[ \|x+y\|_2\le \|x\|_2+\|y\|_2. \]
How the triangle inequality is used. Use it when you decompose error into approximation error plus estimation error, or empirical error plus generalization gap. For example, \[ \|A-C\|_F \le \|A-B\|_F+\|B-C\|_F. \] Many multi-step approximation arguments are just repeated applications of this inequality.
AM-GM and Young's inequality. For nonnegative \(a,b\), \[ ab\le \frac{a^2+b^2}{2}. \] More generally, Young's inequality lets you split a product into a sum of powers.
How AM-GM is used. This is the standard way to eliminate a cross term. If \[ 2ab \] appears inside a quadratic expansion, replace it by \(a^2+b^2\). This is common when bounding \(\|x+y\|_2^2\), gradients, or residual terms.
Union bound. For events \(E_1,\dots,E_m\), \[ \mathbb{P}\Bigl(\bigcup_{i=1}^m E_i\Bigr)\le \sum_{i=1}^m \mathbb{P}(E_i). \]
How the union bound is used. This is how one passes from a guarantee for a single hypothesis or a single vector to a simultaneous guarantee over a finite set. In learning theory, if each bad event has probability at most \(\delta/|H|\), then all hypotheses in a finite class behave well simultaneously with probability at least \(1-\delta\).
Markov and Chebyshev. If \(X\ge 0\), then for \(t\gt 0\), \[ \mathbb{P}(X\ge t)\le \frac{\mathbb{E}X}{t}. \] If \(X\) has finite variance, then for \(t\gt 0\), \[ \mathbb{P}(|X-\mathbb{E}X|\ge t)\le \frac{\operatorname{Var}(X)}{t^2}. \]
How Markov and Chebyshev are used. Markov converts a bound on expectation into a tail bound. Chebyshev converts variance control into concentration. They are weaker than Hoeffding but require fewer assumptions, so they are useful when variables are unbounded or only second moments are available.
Hoeffding's inequality. If \(X_1,\dots,X_m\) are independent and \(a_i\le X_i\le b_i\), then for \(S_m=\sum_{i=1}^m X_i\), \[ \mathbb{P}\bigl(S_m-\mathbb{E}S_m \ge t\bigr) \le \exp\Bigl(-\frac{2t^2}{\sum_{i=1}^m (b_i-a_i)^2}\Bigr). \] In the i.i.d. Bernoulli or bounded-loss case, for the empirical mean \(\widehat{\mu}\), \[ \mathbb{P}\bigl(|\widehat{\mu}-\mu|\ge \epsilon\bigr)\le 2e^{-2m\epsilon^2}. \]
How Hoeffding is used. This is the main concentration inequality behind finite-class PAC bounds and confidence intervals for bounded random variables. If the loss lies in \([0,1]\), then empirical loss \(\widehat{L}(h)\) concentrates around population loss \(L(h)\): \[ \mathbb{P}\bigl(|\widehat{L}(h)-L(h)|\ge \epsilon\bigr)\le 2e^{-2m\epsilon^2}. \] Then apply the union bound over a finite hypothesis class.
Example. If \(|H|=1000\) and you want \[ |\widehat{L}(h)-L(h)|\le \epsilon \] for all \(h\in H\) with probability at least \(1-\delta\), it is enough that \[ 2|H|e^{-2m\epsilon^2}\le \delta. \] Solving gives \[ m\ge \frac{1}{2\epsilon^2}\log\frac{2|H|}{\delta}. \]
Chernoff-style multiplicative bounds. For sums of independent Bernoulli variables, one often uses stronger relative-error bounds than Hoeffding. If \(X=\sum_{i=1}^m X_i\) with \(X_i\in\{0,1\}\) independent and \(\mu=\mathbb{E}X\), then for \(0\lt\epsilon\lt 1\), \[ \mathbb{P}(X\ge (1+\epsilon)\mu)\le \exp\Bigl(-\frac{\epsilon^2\mu}{3}\Bigr), \qquad \mathbb{P}(X\le (1-\epsilon)\mu)\le \exp\Bigl(-\frac{\epsilon^2\mu}{2}\Bigr). \]
Which inequality to reach for. Use Jensen for convexity/concavity arguments. Use Cauchy-Schwarz for inner products and norm bounds. Use the triangle inequality for decomposing error. Use AM-GM to remove cross terms. Use the union bound to make finitely many events hold simultaneously. Use Markov or Chebyshev when you only know moments. Use Hoeffding or Chernoff when you have independence and bounded variables.

12.1 Feedforward Networks and the Hypothesis Class

A feedforward neural network is a directed acyclic graph \(G=(V,E)\) arranged in layers \(V_0,V_1,\dots,V_T\). The layer \(V_0\) is the input layer. Its coordinates are the input variables and, typically, one extra constant neuron whose output is always \(1\). Layers \(V_1,\dots,V_{T-1}\) are hidden layers, and \(V_T\) is the output layer.

Fix an activation function \(\sigma:\mathbb{R}\to\mathbb{R}\). For a neuron \(v_{t+1,j}\), define

\[ a_{t+1,j}(x)=\sum_{r:(v_{t,r},v_{t+1,j})\in E} w((v_{t,r},v_{t+1,j}))\,o_{t,r}(x), \qquad o_{t+1,j}(x)=\sigma(a_{t+1,j}(x)). \]

Each symbol has a concrete role. \(x\) is the input example. \(o_{t,r}(x)\) is the output produced by neuron \(r\) in layer \(t\) when the network is fed the input \(x\). The quantity \(w((v_{t,r},v_{t+1,j}))\) is the weight on the edge from neuron \(r\) in layer \(t\) to neuron \(j\) in layer \(t+1\). The sum \[ a_{t+1,j}(x) \] is the pre-activation or affine score of neuron \(j\): it is the weighted combination of all outputs feeding into that neuron before any nonlinearity is applied. The output \[ o_{t+1,j}(x)=\sigma(a_{t+1,j}(x)) \] is the value sent forward to the next layer after applying the activation function \(\sigma\).

Read the formula in two stages. First, compute a weighted vote from the previous layer. Second, pass that score through a nonlinearity. Without the nonlinearity, the whole network would collapse to a single linear map.

Thus the computation is layerwise: weighted sum first, activation second. Typical activations in the classical theory are the sign function, the threshold function, and the sigmoid.

Definition. For a fixed architecture \((V,E,\sigma)\), the neural network hypothesis class is \[ H_{V,E,\sigma}=\{h_{V,E,\sigma,w}: w:E\to \mathbb{R}\}. \] The architecture is fixed; the learned parameters are the edge weights.

This notation should be read carefully. \(V\) specifies the neurons and their layer structure, \(E\) specifies which neurons are connected, \(\sigma\) specifies the activation rule used at each neuron, and \(w:E\to\mathbb{R}\) assigns one real parameter to each edge. So \(H_{V,E,\sigma}\) is the set of all functions realizable by that fixed graph as the edge weights vary.

This is a definition rather than a theorem, but it has an immediate corollary worth stating explicitly: every claim about learnability, VC dimension, or optimization must be interpreted relative to a fixed architecture unless the hypothesis class is enlarged by allowing the graph itself to vary.

This distinction is important. The phrase “learn a neural network” means “search over the set of possible weight assignments of a fixed graph,” not “choose the graph itself,” unless the problem explicitly says that the architecture may vary.

12.2 Why Neural Networks Matter: Approximation, Estimation, Optimization

Neural networks are best understood by splitting the analysis into three separate questions.

  1. Approximation: what functions can the class represent?
  2. Estimation: how many examples are needed to learn within the class?
  3. Optimization: can we actually find good weights efficiently?

The first two questions are favorable. The third is the hard one. Many confusions about neural networks come from mixing these three issues together.

Expressive power is not the same as learnability in polynomial time. A class may contain excellent predictors and still be hard to optimize over.

12.3 Expressive Power: Universality and Circuit Simulation

A depth-2 sign network can represent every Boolean function. The construction is explicit: for each positive input pattern \(u\), construct one hidden neuron that fires exactly on \(u\), then let the output neuron compute the disjunction of those hidden neurons.

Construction for an arbitrary Boolean function. Let \(f:\{\pm1\}^n\to\{\pm1\}\). Let \(u_1,\dots,u_k\) be the inputs on which \(f\) equals \(1\). For each \(u_i\), build a hidden neuron \(g_i\) such that \(g_i(x)=1\) iff \(x=u_i\). Then the output neuron computes \(g_1(x)\vee \cdots \vee g_k(x)\).

This proves universality, but it is weak as an efficiency statement because \(k\) may be exponential in \(n\). So one also needs a size-sensitive expressivity result.

Key expressivity fact. Every Boolean function computable in time \(T(n)\) can be represented by a sign network of size \(O(T(n)^2)\).

Proof. Let \(f\) be computed by a Turing machine in at most \(T(n)\) steps on inputs of length \(n\). A standard simulation from circuit complexity converts this computation into a Boolean circuit of size \(O(T(n)^2)\): one tracks the machine configuration at each time step, and each update from time \(t\) to \(t+1\) depends only on a constant-size local neighborhood of the current configuration. Since there are \(O(T(n))\) relevant tape cells and \(O(T(n))\) time steps, the resulting circuit has quadratic size.

It remains to simulate Boolean gates with sign neurons. For inputs \(x_i\in\{\pm1\}\), define \[ \operatorname{AND}(x_1,\dots,x_k) = \operatorname{sign}\!\left(\sum_{i=1}^k x_i-k+1\right). \] If all inputs are \(1\), the expression inside the sign is \(1\); otherwise it is at most \(-1\). Similarly, \[ \operatorname{OR}(x_1,\dots,x_k) = \operatorname{sign}\!\left(\sum_{i=1}^k x_i+k-1\right), \] because the sum is at least \(0\) when some input is \(1\) and at most \(-2\) when all are \(-1\). Negation is implemented by multiplying by \(-1\). Therefore each Boolean gate in the circuit can be replaced by a sign neuron, giving a sign network of size \(O(T(n)^2)\) computing \(f\). This proves the claim.

Corollary. Polynomial-size sign networks are rich enough to contain every efficiently computable Boolean predictor. Thus the approximation power of neural networks is not the bottleneck for most algorithmically realistic tasks.

The idea is via Boolean circuits. A Turing computation can be unrolled into a layered circuit recording the machine state at each time step. Since sign neurons can implement conjunction, disjunction, and negation, the resulting circuit can be simulated by a neural network.

This gives the right interpretation: neural networks are expressive enough to model efficiently computable predictors without requiring exponential size in every relevant case.

Example. If the target is “accept exactly these \(k\) inputs,” then the universal construction uses \(k\) hidden units. This is efficient only when \(k\) is small. If \(k\) is exponential, the construction proves representability but not useful learnability.

12.4 Geometric Intuition for Depth

A sign neuron defines a halfspace. Therefore, one hidden layer forms several halfspaces and then combines the corresponding binary indicators. If the output computes a conjunction, the network represents an intersection of halfspaces, hence a convex polytope. Adding another layer allows unions of such pieces. This is the simplest geometric explanation for why extra depth increases the complexity of decision regions.

One hidden layer gives intersections of simple separators. More layers compose those regions into richer structures such as unions of polytopes and other nested decision sets.

12.5 Sample Complexity and VC Dimension

For binary classification with sign activation at the output, sample complexity is controlled by VC dimension. The standard bound is:

Theorem. \[ \mathrm{VCdim}(H_{V,E,\mathrm{sign}})=O(|E|\log |E|). \]

Proof. Let \(H=H_{V,E,\mathrm{sign}}\). For each layer \(t\), let \(H^{(t)}\) be the class of maps from outputs of layer \(t-1\) to outputs of layer \(t\) obtained by varying only the weights entering layer \(t\). Since the full network is the composition of these layer maps, \[ h = h^{(T)}\circ \cdots \circ h^{(1)}, \qquad h^{(t)}\in H^{(t)}, \] the growth function satisfies \[ \tau_H(m)\le \prod_{t=1}^T \tau_{H^{(t)}}(m). \] Now fix a layer \(t\). If that layer has neurons \(1,\dots,k_t\), then \(H^{(t)}\) is a product class of the coordinate classes \(H^{(t,1)},\dots,H^{(t,k_t)}\), so \[ \tau_{H^{(t)}}(m)\le \prod_{j=1}^{k_t}\tau_{H^{(t,j)}}(m). \] If neuron \(j\) in layer \(t\) has \(d_{t,j}\) incoming edges, then \(H^{(t,j)}\) is the class of sign halfspaces in dimension \(d_{t,j}\), whose VC dimension is \(d_{t,j}\). Sauer's lemma therefore yields \[ \tau_{H^{(t,j)}}(m)\le \left(\frac{em}{d_{t,j}}\right)^{d_{t,j}} \le (em)^{d_{t,j}}. \] Multiplying over all neurons and all layers gives \[ \tau_H(m)\le (em)^{\sum_{t,j} d_{t,j}} = (em)^{|E|}. \] If \(m\) points are shattered, then \(\tau_H(m)=2^m\), so \[ 2^m \le (em)^{|E|}. \] Taking logarithms, \[ m\log 2 \le |E|(1+\log m). \] Solving this inequality for \(m\) gives \(m=O(|E|\log |E|)\). Hence \(\mathrm{VCdim}(H_{V,E,\mathrm{sign}})=O(|E|\log |E|)\).

Corollary. Standard VC theory then implies sample complexity bounds polynomial in the number of edges. So sign networks are statistically learnable with data requirements controlled by network size.

The proof decomposes the network by layers. Each neuron behaves like a halfspace over the outputs of the previous layer, and the growth function of the whole network is bounded by multiplying the growth functions of the component classes. Sauer’s lemma then yields the stated VC bound.

The main takeaway is that the statistical complexity of sign networks scales roughly with the number of trainable parameters, up to a logarithmic factor.

12.6 Hardness of Training

Despite the positive approximation and sample-complexity results, optimization is hard. Even exact empirical risk minimization for very small networks with sign activations is NP-hard. More strongly, even approximate ERM is computationally hard in general.

This is the central negative result: the main obstacle to neural-network learning is computational, not representational or statistical.

12.7 SGD and Backpropagation

The practical training rule is stochastic gradient descent. One chooses a differentiable loss \(L_D(w)\) as a function of the weights and repeatedly updates \(w\) in the negative gradient direction using random examples. Since the loss is nonconvex, SGD is a heuristic rather than a globally correct optimization procedure.

Backpropagation is the efficient gradient-computation method used inside SGD. It computes all partial derivatives using one forward pass and one backward pass through the network.

Forward pass notation. For neuron \(i\) in layer \(t\), write \[ a_{t,i}=\sum_j w_{t,i,j}\,o_{t-1,j}+b_{t,i}, \qquad o_{t,i}=\sigma(a_{t,i}). \] Here \(a_{t,i}\) is the pre-activation and \(o_{t,i}\) is the post-activation. If the network output is \(\widehat y=o_{T,1}\) and the loss on one example is \(\ell(\widehat y,y)\), then the goal of backpropagation is to compute all derivatives of \(\ell\) with respect to all weights and biases.
Indexing convention for \(w_{t,i,j}\). The weight \(w_{t,i,j}\) is the weight on the edge \[ \text{(neuron \(j\) in layer \(t-1\))} \longrightarrow \text{(neuron \(i\) in layer \(t\))}. \] So:
  • \(t\) tells you which layer receives the signal.
  • \(i\) is the destination neuron index in layer \(t\).
  • \(j\) is the source neuron index in layer \(t-1\).
Equivalently, the matrix \(W_t=(w_{t,i,j})\) maps activations from layer \(t-1\) to pre-activations in layer \(t\).
Backpropagation error signals. Define \[ \delta_{t,i}=\frac{\partial \ell}{\partial a_{t,i}}. \] These are the key hidden quantities propagated backward through the network.
Output-layer formula. At the output layer, \[ \delta_{T,i} = \frac{\partial \ell}{\partial o_{T,i}}\sigma'(a_{T,i}). \] This is just the chain rule: \[ \frac{\partial \ell}{\partial a_{T,i}} = \frac{\partial \ell}{\partial o_{T,i}} \frac{\partial o_{T,i}}{\partial a_{T,i}} = \frac{\partial \ell}{\partial o_{T,i}}\sigma'(a_{T,i}). \]
Hidden-layer recursion. For a hidden neuron \(i\) in layer \(t\lt T\), \[ \delta_{t,i} = \sigma'(a_{t,i})\sum_k w_{t+1,k,i}\,\delta_{t+1,k}. \] Proof. The variable \(a_{t,i}\) affects the loss only through the later pre-activations \(a_{t+1,k}\). By the chain rule, \[ \frac{\partial \ell}{\partial a_{t,i}} = \sum_k \frac{\partial \ell}{\partial a_{t+1,k}} \frac{\partial a_{t+1,k}}{\partial a_{t,i}}. \] Since \[ a_{t+1,k}=\sum_r w_{t+1,k,r}o_{t,r}+b_{t+1,k} \qquad\text{and}\qquad o_{t,r}=\sigma(a_{t,r}), \] we get \[ \frac{\partial a_{t+1,k}}{\partial a_{t,i}} = w_{t+1,k,i}\sigma'(a_{t,i}). \] Substituting gives \[ \delta_{t,i} = \sum_k \delta_{t+1,k} w_{t+1,k,i}\sigma'(a_{t,i}) = \sigma'(a_{t,i})\sum_k w_{t+1,k,i}\delta_{t+1,k}. \]

Once the error signal \(\delta_{t,i}\) is known, the derivatives with respect to incoming parameters are local.

Gradient formulas for weights and biases. For an edge from neuron \(j\) in layer \(t-1\) to neuron \(i\) in layer \(t\), \[ \frac{\partial \ell}{\partial w_{t,i,j}} = \delta_{t,i}\,o_{t-1,j}, \qquad \frac{\partial \ell}{\partial b_{t,i}} = \delta_{t,i}. \] Proof. Since \[ a_{t,i}=\sum_j w_{t,i,j}o_{t-1,j}+b_{t,i}, \] we have \[ \frac{\partial a_{t,i}}{\partial w_{t,i,j}}=o_{t-1,j}, \qquad \frac{\partial a_{t,i}}{\partial b_{t,i}}=1. \] Therefore \[ \frac{\partial \ell}{\partial w_{t,i,j}} = \frac{\partial \ell}{\partial a_{t,i}} \frac{\partial a_{t,i}}{\partial w_{t,i,j}} = \delta_{t,i}o_{t-1,j}, \] and similarly \(\partial \ell/\partial b_{t,i}=\delta_{t,i}\).
Backpropagation algorithm. For one training example \((x,y)\):
  1. Run a forward pass to compute all \(a_{t,i}\) and \(o_{t,i}\).
  2. Compute the output-layer errors \(\delta_{T,i}\).
  3. For \(t=T-1,T-2,\dots,1\), compute hidden-layer errors using \[ \delta_{t,i}=\sigma'(a_{t,i})\sum_k w_{t+1,k,i}\delta_{t+1,k}. \]
  4. Compute gradients by \[ \frac{\partial \ell}{\partial w_{t,i,j}}=\delta_{t,i}o_{t-1,j}, \qquad \frac{\partial \ell}{\partial b_{t,i}}=\delta_{t,i}. \]
  5. Update with SGD: \[ w_{t,i,j}\leftarrow w_{t,i,j}-\eta \frac{\partial \ell}{\partial w_{t,i,j}}, \qquad b_{t,i}\leftarrow b_{t,i}-\eta \frac{\partial \ell}{\partial b_{t,i}}. \]
One-hidden-layer formulas. Suppose \[ h_i=\sigma\!\left(\sum_j w^{(1)}_{i,j}x_j+b^{(1)}_i\right), \qquad \widehat y=\sigma\!\left(\sum_i w^{(2)}_i h_i+b^{(2)}\right). \] Then \[ \delta^{(2)}=\frac{\partial \ell}{\partial \widehat y}\sigma'(a^{(2)}), \] and for each hidden unit \(i\), \[ \delta^{(1)}_i=\sigma'(a^{(1)}_i)\,w^{(2)}_i\,\delta^{(2)}. \] Therefore \[ \frac{\partial \ell}{\partial w^{(2)}_i}=\delta^{(2)}h_i, \qquad \frac{\partial \ell}{\partial w^{(1)}_{i,j}}=\delta^{(1)}_i x_j. \] This is the exam pattern to memorize: output error first, then push it backward through the weights, then multiply by the incoming activation.
Backpropagation is just repeated chain rule on a layered graph. The reason it is efficient is that each intermediate derivative \(\delta_{t,i}\) is computed once and then reused for every incoming weight at that neuron.

12.8 Smoothness and the Basic Gradient-Descent Descent Lemma

If the gradient is Lipschitz, then a single step of gradient descent provably decreases the objective. This is the basic deterministic estimate behind gradient-based training.

Definition. A differentiable function \(f:\mathbb R^d\to\mathbb R\) is \(L\)-smooth if \[ \|\nabla f(x)-\nabla f(y)\|_2 \le L\|x-y\|_2 \qquad \text{for all }x,y\in\mathbb R^d. \]

If \(f\) is twice differentiable, this is equivalent to the operator-norm bound \[ \|\nabla^2 f(x)\|_{\mathrm{op}} \le L \qquad \text{for all }x, \] so the largest eigenvalue of the Hessian is at most \(L\).

Descent lemma. If \(f\) is \(L\)-smooth, then for all \(x,y\), \[ f(x)\le f(y)+\langle \nabla f(y),x-y\rangle + \frac{L}{2}\|x-y\|_2^2. \] In particular, the gradient-descent update \[ w^{k+1}=w^k-\frac{1}{L}\nabla f(w^k) \] satisfies \[ f(w^{k+1})\le f(w^k)-\frac{1}{2L}\|\nabla f(w^k)\|_2^2. \]

Proof. Define \[ \phi(t)=f(y+t(x-y)), \qquad t\in[0,1]. \] Then \[ \phi'(t)=\langle \nabla f(y+t(x-y)),x-y\rangle. \] Hence \[ f(x)-f(y) = \int_0^1 \phi'(t)\,dt = \langle \nabla f(y),x-y\rangle + \int_0^1 \langle \nabla f(y+t(x-y))-\nabla f(y),x-y\rangle\,dt. \] By Cauchy-Schwarz and \(L\)-smoothness, \[ \left| \langle \nabla f(y+t(x-y))-\nabla f(y),x-y\rangle \right| \le Lt\|x-y\|_2^2. \] Integrating \(Lt\) from \(0\) to \(1\) gives \(L\|x-y\|_2^2/2\), proving the first inequality.

Now substitute \[ x=w^{k+1}=w^k-\frac1L\nabla f(w^k), \qquad y=w^k. \] Then \[ x-y=-\frac1L\nabla f(w^k), \] so \[ \langle \nabla f(w^k),x-y\rangle = -\frac1L\|\nabla f(w^k)\|_2^2 \] and \[ \frac{L}{2}\|x-y\|_2^2 = \frac{1}{2L}\|\nabla f(w^k)\|_2^2. \] Combining the two terms gives \[ f(w^{k+1})\le f(w^k)-\frac{1}{2L}\|\nabla f(w^k)\|_2^2. \]

Corollary. With step size \(1/L\), gradient descent strictly decreases \(f\) unless the gradient is already zero.

Part XIII: Boosting

13.1 Weak Learnability

What boosting is. Boosting is a procedure for converting many simple predictors, each only slightly better than random guessing, into one strong predictor with very small training error and often much better predictive performance. The base learner is called repeatedly on reweighted versions of the same sample, and the final classifier is a weighted vote of the hypotheses returned in those rounds.

In symbols, if the weak learner returns hypotheses \(h_1,\dots,h_T\), then boosting forms an aggregate score \[ F_T(x)=\sum_{t=1}^T \alpha_t h_t(x) \] and predicts by the sign of that score: \[ H_T(x)=\operatorname{sign}(F_T(x)). \] So boosting is not choosing the single best weak hypothesis. It is building a new classifier from a weighted combination of many weak hypotheses.

A weak learner is only required to do slightly better than chance. In binary classification this means producing a hypothesis with error at most \(1/2-\gamma\) for some fixed \(\gamma\gt 0\).

Definition. A learning algorithm is a \(\gamma\)-weak learner if, under realizability, it returns with high probability a hypothesis \(h\) such that \[ L(h)\le \frac12-\gamma. \]

Here \(L(h)\) denotes the classification error of \(h\) under the distribution currently being considered. In the realizable boosting setting this usually means either the weighted empirical error on the current sample distribution \(D^{(t)}\), or the true distributional error in an abstract definition of weak learning. The parameter \(\gamma\) is the edge of the weak learner over random guessing. Random guessing has error \(1/2\); a weak learner improves this by at least \(\gamma\).

“Weak” does not mean useless. It means the learner only needs a uniform advantage over chance, no matter how small. As long as that advantage is positive and repeatable, boosting can compound it across rounds.

A useful corollary of the definition is conceptual rather than algebraic: if a weak learner can be made efficient on a simple base class, then boosting can leverage it to construct a much richer predictor without ever directly optimizing over the richer class.

Weak learning is not easier than strong learning from the perspective of VC dimension alone. The real advantage is computational: a weak learner may be easy to implement even when full ERM over a richer class is not.

13.2 Confidence Amplification by Repeated Runs

Before one reaches AdaBoost itself, there is a simpler amplification principle: if a randomized learner succeeds with constant probability bounded away from \(1/2\), then repeated independent runs reduce the confidence parameter from a constant to any desired \(\delta\).

Amplification lemma. Suppose a learning algorithm succeeds with probability at least \(1-\eta\), where \(\eta<1\). If we run it independently \(t\) times, then the probability that all \(t\) runs fail is at most \(\eta^t\). Therefore it suffices to choose \[ t \ge \frac{\log(1/\delta)}{\log(\eta^{-1})} \] to make the failure probability at most \(\delta\).

Proof. Independence gives \[ \mathbb{P}[\text{all }t\text{ runs fail}] = \eta^t. \] Thus \(\eta^t\le \delta\) exactly when \[ t\log(\eta)\le \log(\delta), \] which rearranges to \[ t \ge \frac{\log(\delta)}{\log(\eta)} = \frac{\log(1/\delta)}{\log(\eta^{-1})}. \]

Example. If one run succeeds with probability \(0.6\), then one run fails with probability \(0.4\). After \(t\) independent runs, the probability that every run fails is at most \[ (0.4)^t. \] Hence \(t=\Theta(\log(1/\delta))\) runs suffice to reach confidence \(1-\delta\).

13.3 Decision Stumps as the Canonical Base Class

A decision stump thresholds one coordinate and predicts according to the sign of the comparison. Weighted ERM over stumps is efficient because, after sorting along a coordinate, the weighted empirical error can be updated incrementally as the threshold sweeps from left to right.

Decision stumps are also genuine weak learners for richer classes. Consider a 3-piece classifier on the line, with labels \(+\,-\,+\). At least one of the three regions has probability mass at most \(1/3\). A stump can match the target on the other two regions, so its error is at most \(1/3\), which is strictly below \(1/2\).

Example. This 3-region example is important because it shows exactly what a weak learner needs to do: not solve the problem completely, but consistently beat random guessing.

13.4 AdaBoost: Update Rule and Interpretation

AdaBoost is the canonical boosting algorithm. It keeps a probability distribution \(D^{(t)}\) over the training examples. Examples that were hard for earlier hypotheses receive larger weight in later rounds, forcing the next weak learner to pay more attention to them. After \(T\) rounds, AdaBoost outputs a weighted majority vote of the \(T\) weak hypotheses.

AdaBoost maintains a distribution \(D^{(t)}\) over training examples. At round \(t\), the weak learner is trained relative to \(D^{(t)}\) and returns a hypothesis \(h_t\) with weighted error \(\epsilon_t\). The algorithm assigns vote

\[ w_t=\frac12\log\!\left(\frac{1}{\epsilon_t}-1\right) \] and updates the sample weights by \[ D_i^{(t+1)} \propto D_i^{(t)} e^{-w_t y_i h_t(x_i)}. \] The final hypothesis is \[ h_S(x)=\operatorname{sign}\!\left(\sum_{t=1}^T w_t h_t(x)\right). \]

The terms in this update matter. \(D_i^{(t)}\) is the weight of training point \(i\) at round \(t\). The label \(y_i\in\{\pm 1\}\) is the true class, and \(h_t(x_i)\in\{\pm1\}\) is the round-\(t\) weak learner's prediction on that point. Therefore \(y_i h_t(x_i)=+1\) when the point is classified correctly and \(-1\) when it is misclassified. The coefficient \(w_t\) is the vote strength assigned to \(h_t\): lower weighted error \(\epsilon_t\) produces larger \(w_t\), so more accurate weak hypotheses count more in the final classifier.

The exponential update is the mechanism that makes boosting adaptive. Misclassified points are amplified, correctly classified points are damped, and the next weak learner is fit to the resulting harder distribution. The final classifier is therefore a sequence of corrections to earlier mistakes, not a repetition of the same rule.

If example \(i\) is misclassified, then \(y_i h_t(x_i)=-1\), so the exponential factor is \(e^{w_t}\) and its weight increases. If example \(i\) is correctly classified, the factor is \(e^{-w_t}\) and its weight decreases. So difficult examples become more influential in later rounds.

Computation. If \(\epsilon_t=0.2\), then \[ w_t=\frac12\log\left(\frac{1}{0.2}-1\right)=\frac12\log 4=\log 2. \]

13.5 Accuracy of the Current Weak Learner Under the Next Distribution

One of the cleanest algebraic facts about AdaBoost is that after the reweighting step, the current weak classifier \(h_t\) has weighted accuracy exactly \(1/2\) under the new distribution \(D^{(t+1)}\). This is not merely an interpretation; it is an exact identity.

Lemma. Let \[ \epsilon_t = \mathbb{P}_{i\sim D^{(t)}}[h_t(x_i)\neq y_i]. \] Suppose AdaBoost updates the weights by leaving misclassified examples unchanged and multiplying correctly classified examples by \[ \beta_t=\frac{\epsilon_t}{1-\epsilon_t}, \] followed by normalization. Then \[ \mathbb{P}_{i\sim D^{(t+1)}}[h_t(x_i)=y_i]=\mathbb{P}_{i\sim D^{(t+1)}}[h_t(x_i)\neq y_i]=\frac12. \]

Proof. Let \[ C_t=\{i:h_t(x_i)=y_i\}, \qquad M_t=\{i:h_t(x_i)\neq y_i\}. \] By definition of \(\epsilon_t\), \[ \sum_{i\in C_t} D_i^{(t)} = 1-\epsilon_t, \qquad \sum_{i\in M_t} D_i^{(t)} = \epsilon_t. \] Before normalization, the new weights are \[ \widetilde D_i^{(t+1)}= \begin{cases} \beta_t D_i^{(t)} & i\in C_t,\\ D_i^{(t)} & i\in M_t. \end{cases} \] Therefore the total unnormalized mass on correctly classified examples is \[ \sum_{i\in C_t} \widetilde D_i^{(t+1)} = \beta_t(1-\epsilon_t) = \frac{\epsilon_t}{1-\epsilon_t}(1-\epsilon_t) = \epsilon_t. \] The total unnormalized mass on misclassified examples is \[ \sum_{i\in M_t}\widetilde D_i^{(t+1)} = \epsilon_t. \] Hence the total unnormalized mass is \(2\epsilon_t\). After normalization, \[ \mathbb{P}_{i\sim D^{(t+1)}}[h_t(x_i)=y_i] = \frac{\epsilon_t}{2\epsilon_t} = \frac12, \] and likewise \[ \mathbb{P}_{i\sim D^{(t+1)}}[h_t(x_i)\neq y_i] = \frac{\epsilon_t}{2\epsilon_t} = \frac12. \] This proves the lemma.

Corollary. Relative to the next distribution, the current weak learner has zero edge. This is exactly why AdaBoost needs a new classifier at the next round: the reweighting removes the advantage of the old one.

The conceptual point is precise. The update does not just “focus on hard points.” It rebalances the sample so that the current classifier is no better than random guessing with respect to the new weights.

13.6 Why the Training Error Falls Exponentially

The analysis tracks an exponential potential. Each round shrinks the potential by a factor \(\sqrt{1-4\gamma^2}\), which is at most \(e^{-2\gamma^2}\). Since 0-1 training loss is upper bounded by the exponential loss, one obtains:

Theorem. If every round satisfies \(\epsilon_t\le 1/2-\gamma\), then \[ L_S(h_S)\le e^{-2\gamma^2 T}. \]

Proof. Define the ensemble score \(f_T(x)=\sum_{t=1}^T w_t h_t(x)\) and the exponential potential \[ Z_T=\frac1m\sum_{i=1}^m e^{-y_i f_T(x_i)}. \] Since \(h_S(x)=\operatorname{sign}(f_T(x))\), whenever \(h_S(x_i)\neq y_i\) we have \(y_i f_T(x_i)\le 0\), and therefore \[ 1[h_S(x_i)\neq y_i]\le e^{-y_i f_T(x_i)}. \] Averaging over \(i\) gives \(L_S(h_S)\le Z_T\). Thus it suffices to bound \(Z_T\).

Let \(f_t(x)=\sum_{s=1}^t w_s h_s(x)\), with \(f_0\equiv 0\), and define \[ Z_t=\frac1m\sum_{i=1}^m e^{-y_i f_t(x_i)}. \] Then \(Z_0=1\) and \[ Z_T = \prod_{t=1}^T \frac{Z_t}{Z_{t-1}}. \] For a fixed round \(t\), \[ \frac{Z_t}{Z_{t-1}} = \frac{\sum_i e^{-y_i f_t(x_i)}}{\sum_i e^{-y_i f_{t-1}(x_i)}} = \sum_i D_i^{(t)} e^{-w_t y_i h_t(x_i)}, \] where \(D^{(t)}\) is the AdaBoost distribution induced by round \(t\). Splitting the sum over correct and incorrect predictions yields \[ \frac{Z_t}{Z_{t-1}} = (1-\epsilon_t)e^{-w_t} + \epsilon_t e^{w_t}. \] Since \[ w_t=\frac12\log\!\left(\frac{1-\epsilon_t}{\epsilon_t}\right), \] we have \[ e^{w_t}=\sqrt{\frac{1-\epsilon_t}{\epsilon_t}}, \qquad e^{-w_t}=\sqrt{\frac{\epsilon_t}{1-\epsilon_t}}. \] Therefore \[ \frac{Z_t}{Z_{t-1}} = (1-\epsilon_t)\sqrt{\frac{\epsilon_t}{1-\epsilon_t}} + \epsilon_t\sqrt{\frac{1-\epsilon_t}{\epsilon_t}} = 2\sqrt{\epsilon_t(1-\epsilon_t)}. \] Since \(\epsilon_t\le 1/2-\gamma\), \[ \epsilon_t(1-\epsilon_t)\le \left(\frac12-\gamma\right)\left(\frac12+\gamma\right)=\frac14-\gamma^2, \] hence \[ \frac{Z_t}{Z_{t-1}}\le 2\sqrt{\frac14-\gamma^2}=\sqrt{1-4\gamma^2}. \] Using \(\sqrt{1-a}\le e^{-a/2}\) for \(a\in[0,1]\), we obtain \[ \frac{Z_t}{Z_{t-1}}\le e^{-2\gamma^2}. \] Multiplying over \(t\) gives \[ Z_T\le e^{-2\gamma^2 T}. \] Since \(L_S(h_S)\le Z_T\), the theorem follows.

Corollary. To drive the training error below \(\eta\), it suffices to take \[ T \ge \frac{1}{2\gamma^2}\log\!\left(\frac{1}{\eta}\right). \] Thus even a small fixed edge \(\gamma\) can be amplified into very small training error.

This is the central reason AdaBoost is effective on the training sample: repeated weak improvement compounds multiplicatively.

13.7 The Weighted-Mass Calculation Behind AdaBoost

The weighted-mass calculation underlying the exponential training-error theorem is worth isolating because it captures the one-step contraction mechanism directly.

Lemma. Let the weighted error at round \(t\) be \[ \epsilon_t=\frac12-\gamma_t, \qquad \gamma_t\gt 0, \] and set \[ \beta_t = \frac{\epsilon_t}{1-\epsilon_t} = \frac{1/2-\gamma_t}{1/2+\gamma_t}. \] If correctly classified examples have their weights multiplied by \(\beta_t\) and incorrectly classified examples keep their weight, then the total unnormalized weight after the update is \[ W_t\cdot 2\epsilon_t = W_t\cdot 2\left(\frac12-\gamma_t\right). \] In the equivalent exponential-weight formulation of AdaBoost, the contraction factor is \[ 2\sqrt{\epsilon_t(1-\epsilon_t)} \le \sqrt{1-4\gamma_t^2}. \]

Proof. The correctly classified portion of the mass is \(1-\epsilon_t\), while the misclassified portion is \(\epsilon_t\). Therefore after multiplying the correct points by \(\beta_t\), the new unnormalized total weight is \[ W_t\big((1-\epsilon_t)\beta_t+\epsilon_t\big). \] Substituting \(\beta_t=\epsilon_t/(1-\epsilon_t)\) gives \[ (1-\epsilon_t)\beta_t+\epsilon_t = (1-\epsilon_t)\frac{\epsilon_t}{1-\epsilon_t}+\epsilon_t = 2\epsilon_t. \] This proves the first formula.

For the exponential-weight form, one instead writes the round-\(t\) normalization factor as \[ (1-\epsilon_t)e^{-w_t}+\epsilon_t e^{w_t} \] with \[ w_t=\frac12\log\!\left(\frac{1-\epsilon_t}{\epsilon_t}\right). \] Substituting \[ e^{w_t}=\sqrt{\frac{1-\epsilon_t}{\epsilon_t}}, \qquad e^{-w_t}=\sqrt{\frac{\epsilon_t}{1-\epsilon_t}} \] yields \[ (1-\epsilon_t)e^{-w_t}+\epsilon_t e^{w_t} = 2\sqrt{\epsilon_t(1-\epsilon_t)}. \] Since \(\epsilon_t(1-\epsilon_t)=1/4-\gamma_t^2\), this equals \(\sqrt{1-4\gamma_t^2}\).

The significance of the lemma is that every full AdaBoost proof reduces to showing that this one-step contraction compounds over rounds.

13.8 Capacity of the Final Class

The output of AdaBoost lies in

\[ L(B,T) = \left\{ x\mapsto \operatorname{sign}\!\left(\sum_{t=1}^T w_t h_t(x)\right) : h_t\in B \right\}. \]

Thus the algorithm is really constructing a halfspace over \(T\) base hypotheses. As \(T\) grows, the class becomes richer, so approximation can improve but estimation error may also increase. Up to logarithmic factors, one has

\[ \mathrm{VCdim}(L(B,T)) \lesssim T\cdot \mathrm{VCdim}(B). \]

Proof. Let \(d=\mathrm{VCdim}(B)\), and let \(C=\{x_1,\dots,x_m\}\) be shattered by \(L(B,T)\). To specify a hypothesis in \(L(B,T)\), one first chooses \(T\) base hypotheses \(h_1,\dots,h_T\in B\). Over the set \(C\), each base hypothesis induces one dichotomy. By Sauer's lemma, the number of dichotomies induced by \(B\) on \(m\) points is at most \((em/d)^d\). Therefore the number of possible \(T\)-tuples \((h_1,\dots,h_T)\) restricted to \(C\) is at most \[ \left(\frac{em}{d}\right)^{dT}. \]

Once these \(T\) base hypotheses are fixed, each point \(x_i\) is mapped to a binary vector \[ \psi(x_i)=(h_1(x_i),\dots,h_T(x_i))\in\{\pm 1\}^T. \] The final prediction is a halfspace on these \(T\)-dimensional feature vectors. The VC dimension of halfspaces in \(\mathbb{R}^T\) is \(T+1\), so Sauer's lemma implies that the number of labelings obtainable at this stage is at most \[ \left(\frac{em}{T+1}\right)^{T+1}. \] Consequently the total number of dichotomies realizable by \(L(B,T)\) on \(C\) is at most \[ \left(\frac{em}{d}\right)^{dT} \left(\frac{em}{T+1}\right)^{T+1}. \] Since \(C\) is shattered, this quantity must be at least \(2^m\). Taking logarithms gives \[ m\log 2 \le dT\log\!\left(\frac{em}{d}\right) + (T+1)\log\!\left(\frac{em}{T+1}\right). \] Solving this inequality for \(m\) yields \[ m = O\!\big(Td\log(Td)\big), \] which proves the stated upper bound up to logarithmic factors.

Corollary. The parameter \(T\) simultaneously controls approximation power and estimation error. This is why boosting exposes a genuine bias-complexity tradeoff rather than monotonically improving all aspects of learning.

This is the precise sense in which boosting exposes the bias-complexity tradeoff through the parameter \(T\).

Part XIV: Confidence Intervals and Concentration

14.1 General Template

A confidence interval is a high-probability interval for an unknown population quantity. If \(\hat\theta\) is an estimator of \(\theta\), then a level-\(1-\delta\) interval is any interval \([L,U]\) satisfying \[ \mathbb{P}(\theta\in [L,U])\ge 1-\delta. \] In learning theory, the unknown quantity is often the true error \(L_D(h)\), and the observed quantity is the empirical error \(L_S(h)\).

The general recipe is: start from a concentration inequality, solve it for the deviation parameter \(\epsilon\), then rewrite the result as an interval.

14.2 Exact Gaussian Confidence Intervals for Linear Regression Coefficients

Concentration inequalities give distribution-free intervals. When the noise model is genuinely Gaussian, one can do better: the estimator itself is Gaussian, so the confidence interval is obtained from an exact normal quantile rather than from a generic tail inequality.

Consider the simple linear model with intercept \[ y_i = a + \alpha x_i + \epsilon_i, \qquad \epsilon_i\stackrel{\text{i.i.d.}}{\sim} N(0,\sigma^2), \] where the \(x_i\) are fixed and only the noise is random. The ordinary least-squares estimator of the slope is \[ \widehat\alpha = \frac{\sum_{i=1}^n (x_i-\bar x)(y_i-\bar y)}{\sum_{i=1}^n (x_i-\bar x)^2}, \qquad \bar x = \frac1n\sum_{i=1}^n x_i, \qquad \bar y = \frac1n\sum_{i=1}^n y_i. \]

Theorem. Under the Gaussian model above, \[ \widehat\alpha \sim N\!\left(\alpha,\frac{\sigma^2}{\sum_{i=1}^n (x_i-\bar x)^2}\right). \] Equivalently, \[ \widehat\alpha \sim N\!\left(\alpha,\frac{\sigma^2}{\sum_{i=1}^n x_i^2-\frac1n\left(\sum_{i=1}^n x_i\right)^2}\right). \]

Proof. Because \[ y_i-\bar y = \alpha(x_i-\bar x)+(\epsilon_i-\bar\epsilon), \qquad \bar\epsilon=\frac1n\sum_{j=1}^n \epsilon_j, \] we obtain \[ \sum_{i=1}^n (x_i-\bar x)(y_i-\bar y) = \alpha\sum_{i=1}^n (x_i-\bar x)^2 + \sum_{i=1}^n (x_i-\bar x)(\epsilon_i-\bar\epsilon). \] Since \(\sum_i (x_i-\bar x)=0\), \[ \sum_{i=1}^n (x_i-\bar x)\bar\epsilon = \bar\epsilon \sum_{i=1}^n (x_i-\bar x) = 0. \] Therefore \[ \widehat\alpha = \alpha + \frac{\sum_{i=1}^n (x_i-\bar x)\epsilon_i}{\sum_{i=1}^n (x_i-\bar x)^2}. \] The numerator is a linear combination of the independent Gaussian variables \(\epsilon_1,\dots,\epsilon_n\), hence it is Gaussian. Therefore \(\widehat\alpha\) is Gaussian as well.

Its mean is \[ \mathbb{E}[\widehat\alpha] = \alpha + \frac{\sum_i (x_i-\bar x)\mathbb{E}[\epsilon_i]}{\sum_i (x_i-\bar x)^2} = \alpha. \] Its variance is \[ \mathrm{Var}(\widehat\alpha) = \frac{\mathrm{Var}\!\left(\sum_i (x_i-\bar x)\epsilon_i\right)} {\left(\sum_i (x_i-\bar x)^2\right)^2}. \] Independence of the \(\epsilon_i\) gives \[ \mathrm{Var}\!\left(\sum_i (x_i-\bar x)\epsilon_i\right) = \sum_i (x_i-\bar x)^2 \mathrm{Var}(\epsilon_i) = \sigma^2\sum_i (x_i-\bar x)^2. \] Hence \[ \mathrm{Var}(\widehat\alpha) = \frac{\sigma^2}{\sum_i (x_i-\bar x)^2}. \] Finally, \[ \sum_{i=1}^n (x_i-\bar x)^2 = \sum_{i=1}^n x_i^2 - n\bar x^2 = \sum_{i=1}^n x_i^2-\frac1n\left(\sum_{i=1}^n x_i\right)^2. \] This proves the theorem.

Corollary. If \(z_{1-\delta/2}\) denotes the \((1-\delta/2)\)-quantile of the standard normal, then a level-\(1-\delta\) confidence interval is \[ \widehat\alpha \pm z_{1-\delta/2}\, \frac{\sigma}{\sqrt{\sum_i (x_i-\bar x)^2}}. \]

The proof of the corollary is immediate: standardize the Gaussian variable, \[ \frac{\widehat\alpha-\alpha}{\sigma/\sqrt{\sum_i (x_i-\bar x)^2}} \sim N(0,1), \] and invert the central normal-probability event.

Example. Suppose \(n=100\), \(\sigma=4\), \(\sum_i x_i=30\), and \(\sum_i x_i^2=25\). Then \[ \sum_i (x_i-\bar x)^2 = 25-\frac{30^2}{100} = 16. \] Therefore \[ \mathrm{Var}(\widehat\alpha)=\frac{16}{16}=1, \qquad \widehat\alpha\sim N(\alpha,1). \] A \(98\%\) confidence interval uses \(z_{0.99}\approx 2.326\), so the interval is \[ [\widehat\alpha-2.326,\widehat\alpha+2.326]. \]

14.3 Why Gaussian Maximum Likelihood Reduces to Least Squares

Under Gaussian noise, minimizing the sum of squared residuals is not an ad hoc choice; it is exactly the maximum-likelihood estimator.

Consider the model \[ y_i = \beta_0 + \beta_1 x_i + \epsilon_i, \qquad \epsilon_i \stackrel{\text{i.i.d.}}{\sim} N(0,\sigma^2). \] Then each conditional density is \[ p(y_i\mid x_i,\beta_0,\beta_1) = \frac{1}{\sqrt{2\pi}\sigma} \exp\!\left( -\frac{(y_i-(\beta_0+\beta_1x_i))^2}{2\sigma^2} \right). \]

Proposition. Maximizing the likelihood of the sample is equivalent to minimizing \[ \sum_{i=1}^m (y_i-(\beta_0+\beta_1x_i))^2. \] More generally, in matrix form the Gaussian maximum-likelihood estimator solves \[ \min_w \|Xw-y\|_2^2, \] whose normal-equation solution is \[ w=(X^\top X)^{-1}X^\top y \] whenever \(X^\top X\) is invertible.

Proof. Independence gives the likelihood \[ \mathcal L(\beta_0,\beta_1) = \prod_{i=1}^m p(y_i\mid x_i,\beta_0,\beta_1) = \left(\frac{1}{\sqrt{2\pi}\sigma}\right)^m \exp\!\left( -\frac{1}{2\sigma^2}\sum_{i=1}^m (y_i-(\beta_0+\beta_1x_i))^2 \right). \] Taking logarithms yields \[ \log \mathcal L(\beta_0,\beta_1) = -\frac{m}{2}\log(2\pi)-m\log\sigma -\frac{1}{2\sigma^2}\sum_{i=1}^m (y_i-(\beta_0+\beta_1x_i))^2. \] The first two terms do not depend on \((\beta_0,\beta_1)\), and the factor \(1/(2\sigma^2)\) is positive, so maximizing the log-likelihood is equivalent to minimizing the residual sum of squares.

In matrix notation, \[ \|Xw-y\|_2^2 = (Xw-y)^\top(Xw-y). \] Differentiating with respect to \(w\) gives \[ \nabla_w \|Xw-y\|_2^2 = 2X^\top(Xw-y). \] Setting the gradient equal to zero yields the normal equations \[ X^\top Xw = X^\top y, \] and if \(X^\top X\) is invertible then \[ w=(X^\top X)^{-1}X^\top y. \]

Corollary. Least squares is the correct estimator whenever the noise is modeled as Gaussian. This is why the Gaussian confidence-interval formulas and least-squares estimation belong in the same section of the theory.

14.4 Hoeffding’s Inequality and Its Interval

Let \(Z_1,\dots,Z_m\) be i.i.d. random variables with \(a\le Z_i\le b\) almost surely and mean \(\mu\). Then

\[ \mathbb{P}\!\left(\left|\frac1m\sum_{i=1}^m Z_i-\mu\right|\gt\epsilon\right) \le 2\exp\!\left(\frac{-2m\epsilon^2}{(b-a)^2}\right). \]

Proof. Let \(X_i = Z_i - \mathbb{E}[Z_i]\). Then \(\mathbb{E}[X_i]=0\), and since \(a\le Z_i\le b\), we also have \(X_i\in [a-\mu,b-\mu]\). For any \(\lambda\gt 0\), \[ \mathbb{P}\!\left(\frac1m\sum_i X_i \ge \epsilon\right) = \mathbb{P}\!\left(e^{\lambda \sum_i X_i} \ge e^{\lambda m\epsilon}\right) \le e^{-\lambda m\epsilon}\prod_i \mathbb{E}[e^{\lambda X_i}]. \] By Hoeffding's lemma, \[ \mathbb{E}[e^{\lambda X_i}] \le e^{\lambda^2(b-a)^2/8}. \] Hence \[ \mathbb{P}\!\left(\frac1m\sum_i X_i \ge \epsilon\right) \le \exp\!\left(-\lambda m\epsilon + \frac{m\lambda^2(b-a)^2}{8}\right). \] The right-hand side is minimized at \[ \lambda = \frac{4\epsilon}{(b-a)^2}, \] giving \[ \mathbb{P}\!\left(\frac1m\sum_i X_i \ge \epsilon\right) \le \exp\!\left(\frac{-2m\epsilon^2}{(b-a)^2}\right). \] The same argument applied to \(-X_i\) yields \[ \mathbb{P}\!\left(\frac1m\sum_i X_i \le -\epsilon\right) \le \exp\!\left(\frac{-2m\epsilon^2}{(b-a)^2}\right). \] A union bound over the upper and lower tails proves \[ \mathbb{P}\!\left(\left|\frac1m\sum_i X_i\right|\gt\epsilon\right) \le 2\exp\!\left(\frac{-2m\epsilon^2}{(b-a)^2}\right). \] Since \(\frac1m\sum_i X_i = \frac1m\sum_i Z_i - \mu\), the theorem follows.

Solving for \(\epsilon\) yields the standard confidence interval

Hoeffding confidence interval. With probability at least \(1-\delta\), \[ \mu \in \left[ \bar Z-(b-a)\sqrt{\frac{\log(2/\delta)}{2m}}, \bar Z+(b-a)\sqrt{\frac{\log(2/\delta)}{2m}} \right]. \]

Derivation. Set the Hoeffding tail bound equal to \(\delta\): \[ 2\exp\!\left(\frac{-2m\epsilon^2}{(b-a)^2}\right)=\delta. \] Taking logarithms and solving for \(\epsilon\) gives \[ \epsilon = (b-a)\sqrt{\frac{\log(2/\delta)}{2m}}. \] Substituting this value into the inequality \(|\bar Z - \mu|\le \epsilon\) yields the confidence interval.

Corollary. For bounded losses, empirical means converge to true means at rate \(O(\sqrt{\log(1/\delta)/m})\). In particular, validation error concentrates around true error at the familiar \(1/\sqrt{m}\) scale.

In classification with 0-1 loss, \(a=0\) and \(b=1\), so this becomes a confidence interval for the true error around the empirical error.

Example. If a classifier makes \(12\) mistakes on \(100\) validation examples, then \(\bar Z=0.12\). A \(95\%\) Hoeffding interval has radius \[ \sqrt{\frac{\log 40}{200}}\approx 0.136. \] So a valid interval is approximately \([0,0.256]\) after clipping to \([0,1]\).

14.5 Bernstein-Type Bounds

Hoeffding only uses boundedness. Bernstein-type inequalities also use variance, so they become tighter when the loss is already small.

Risk-dependent form. For a fixed hypothesis \(h\), with probability at least \(1-\delta\), \[ L_S(h)\le L_D(h)+\sqrt{\frac{2L_D(h)\log(1/\delta)}{m}}+\frac{2\log(1/\delta)}{3m}, \] and \[ L_D(h)\le L_S(h)+\sqrt{\frac{2L_S(h)\log(1/\delta)}{m}}+\frac{4\log(1/\delta)}{m}. \]

Proof. Let \(z_1,\dots,z_m\) be i.i.d. examples, and define \[ X_i = \ell(h,z_i)-L_D(h). \] Then \(\mathbb{E}[X_i]=0\), \(|X_i|\le 1\), and because \(0\le \ell \le 1\), \[ \mathrm{Var}(X_i) = \mathbb{E}[\ell(h,z_i)^2]-L_D(h)^2 \le \mathbb{E}[\ell(h,z_i)] = L_D(h). \] Bernstein's inequality therefore gives, with probability at least \(1-\delta\), \[ \frac1m\sum_{i=1}^m X_i \le \sqrt{\frac{2L_D(h)\log(1/\delta)}{m}} + \frac{2\log(1/\delta)}{3m}. \] Since \(\frac1m\sum_i X_i = L_S(h)-L_D(h)\), this rearranges to \[ L_S(h)\le L_D(h)+\sqrt{\frac{2L_D(h)\log(1/\delta)}{m}}+\frac{2\log(1/\delta)}{3m}. \]

For the second inequality, begin from the one-sided inequality already proved: \[ L_S(h)\le L_D(h)+\sqrt{\frac{2L_D(h)\log(1/\delta)}{m}}+\frac{2\log(1/\delta)}{3m}. \] Rearranging gives \[ L_D(h) \le L_S(h)+\sqrt{\frac{2L_D(h)\log(1/\delta)}{m}}+\frac{2\log(1/\delta)}{3m}. \] Write \[ p=L_D(h), \qquad q=L_S(h), \qquad a=\frac{2\log(1/\delta)}{m}, \qquad b=\frac{2\log(1/\delta)}{3m}. \] Then \[ p\le q+\sqrt{ap}+b. \] We now eliminate the dependence on \(p\) inside the square root. If \(p\le q+b\), then certainly \[ p\le q+\sqrt{aq}+4\frac{\log(1/\delta)}{m}, \] so there is nothing to prove. Otherwise define \[ x=p-q-b\gt 0. \] Then \(p=x+q+b\), and from \(p\le q+\sqrt{ap}+b\) we get \[ x\le \sqrt{a(x+q+b)}. \] Squaring both sides yields \[ x^2\le ax+a(q+b). \] Since \(x\ge 0\), this implies \[ x\le a+\sqrt{a(q+b)}. \] Therefore \[ p \le q+b+a+\sqrt{a(q+b)}. \] Using \(\sqrt{a(q+b)}\le \sqrt{aq}+\sqrt{ab}\), we obtain \[ p \le q+\sqrt{aq} + a+b+\sqrt{ab}. \] Now \[ a+b+\sqrt{ab} = \frac{2\log(1/\delta)}{m} + \frac{2\log(1/\delta)}{3m} + \frac{2\log(1/\delta)}{m\sqrt 3} \le \frac{4\log(1/\delta)}{m}. \] Substituting back for \(a\) gives \[ L_D(h)\le L_S(h)+\sqrt{\frac{2L_S(h)\log(1/\delta)}{m}}+\frac{4\log(1/\delta)}{m}, \] which is the desired second inequality.

Corollary. When the empirical error is already very small, the uncertainty radius becomes significantly tighter than Hoeffding's. This is why Bernstein-style bounds are often preferred for low-noise regimes.

The second inequality is often the most useful one operationally, since it bounds true error directly in terms of empirical error.

Hoeffding is safer and more general. Bernstein is sharper when variance or error is small.

14.6 Validation and Interpretation

Confidence intervals matter especially in model comparison. If two validation errors differ by less than the uncertainty radius, the observed difference may be sampling noise. Thus “smaller validation error” does not by itself imply “better model.”

When comparing two models, first compare the gap in validation error to the concentration radius. This is the mathematically correct way to decide whether the difference is meaningful.

Part XV: Principal Component Analysis

15.1 The Optimization Problem

Given vectors \(x_1,\dots,x_m\in\mathbb{R}^d\), PCA seeks a linear map into \(\mathbb{R}^n\) with \(n<d\), together with a linear reconstruction map back into \(\mathbb{R}^d\), so that the total squared reconstruction error is minimized:

\[ \min_{W\in\mathbb{R}^{n\times d},\,U\in\mathbb{R}^{d\times n}} \sum_{i=1}^m \|x_i-UWx_i\|_2^2. \]

This formulation already tells you the meaning of PCA: it is not a generic dimensionality reduction method but the optimal low-dimensional linear reconstruction method under squared error.

15.2 Why Orthogonal Projection Is Optimal

Once an \(n\)-dimensional subspace is fixed, the best reconstruction of a point from that subspace is its orthogonal projection. Hence the encoder-decoder problem collapses to a subspace-selection problem.

Structure theorem. At optimum, the columns of \(U\) are orthonormal and \[ W=U^\top. \] Therefore reconstruction has the form \(UU^\top x\).

Proof. Fix arbitrary matrices \(U\in\mathbb{R}^{d\times n}\) and \(W\in\mathbb{R}^{n\times d}\). The set of all vectors of the form \(UWx\) is an at-most-\(n\)-dimensional subspace \[ R = \{UWy : y\in\mathbb{R}^d\} \subseteq \mathbb{R}^d. \] Let \(V\in\mathbb{R}^{d\times n}\) have orthonormal columns spanning \(R\), so \(V^\top V=I\). Every vector in \(R\) can be written as \(Vy\) for some \(y\in\mathbb{R}^n\). For a fixed \(x\in\mathbb{R}^d\), \[ \|x-Vy\|_2^2 = \|x\|_2^2 + \|y\|_2^2 - 2y^\top V^\top x. \] Differentiating with respect to \(y\) and setting the gradient equal to zero yields the minimizer \(y=V^\top x\). Thus the point in \(R\) closest to \(x\) is \(VV^\top x\). Since \(UWx\in R\), we have \[ \|x-UWx\|_2^2 \ge \|x-VV^\top x\|_2^2 \] for every \(x\). Summing over \(x_1,\dots,x_m\) shows that replacing \((U,W)\) by \((V,V^\top)\) cannot increase the PCA objective. Hence some optimum is achieved by a pair with orthonormal columns and \(W=U^\top\). Therefore the reconstruction map is \(UU^\top x\), i.e. orthogonal projection onto an \(n\)-dimensional subspace.

Corollary. PCA is really a subspace-selection problem. Once the subspace is known, the optimal encoder and decoder are forced: encode by inner products with an orthonormal basis and decode by projection back into that subspace.

This is the first major simplification in PCA: one is really optimizing over orthonormal bases of \(n\)-dimensional subspaces.

15.3 Trace Maximization and the Eigenvector Solution

If \(A=\sum_{i=1}^m x_i x_i^\top\), then the optimization can be rewritten as

\[ \max_{U^\top U=I} \operatorname{trace}(U^\top A U). \]

Since \(A\) is symmetric positive semidefinite, it has an orthonormal eigenbasis. The solution is:

PCA theorem. The optimal \(n\)-dimensional principal subspace is spanned by the eigenvectors of \(A\) corresponding to the largest \(n\) eigenvalues.

Proof. Let \(A=VDV^\top\) be an eigendecomposition, where the diagonal entries satisfy \(D_{11}\ge D_{22}\ge \cdots \ge D_{dd}\ge 0\), and the columns of \(V\) are orthonormal eigenvectors. For any feasible \(U\in\mathbb{R}^{d\times n}\) with \(U^\top U=I\), define \(B=V^\top U\). Then \(B^\top B = U^\top V V^\top U = I\). Hence \[ \operatorname{trace}(U^\top A U) = \operatorname{trace}(B^\top D B) = \sum_{j=1}^d D_{jj}\sum_{i=1}^n B_{ji}^2. \] Let \[ \beta_j = \sum_{i=1}^n B_{ji}^2. \] Since the columns of \(B\) are orthonormal, we have \(0\le \beta_j\le 1\) and \(\sum_{j=1}^d \beta_j = n\). So maximizing the trace is equivalent to maximizing \[ \sum_{j=1}^d D_{jj}\beta_j \] over all choices of \(\beta_j\in[0,1]\) summing to \(n\). Because the eigenvalues are arranged in decreasing order, this linear objective is maximized by taking \[ \beta_1=\cdots=\beta_n=1,\qquad \beta_{n+1}=\cdots=\beta_d=0. \] Indeed, if some mass were placed on a smaller eigenvalue while a larger eigenvalue had \(\beta_j\lt 1\), moving that mass upward would increase the objective.

The maximizing choice of \(\beta\) means precisely that the columns of \(U\) lie in the span of the first \(n\) columns of \(V\), i.e. the top \(n\) eigenvectors of \(A\). Therefore the optimal principal subspace is the span of those eigenvectors.

Corollary. The optimal reconstruction error equals the sum of the discarded eigenvalues. Therefore the spectrum directly quantifies how much information is lost by truncating to \(n\) dimensions.

Thus PCA, projection onto top eigenvectors, and minimization of squared reconstruction error are all equivalent descriptions of the same method.

15.4 Interpretation: Variance and Reconstruction

PCA can be interpreted in two equivalent ways:

  1. It minimizes total squared reconstruction error among all \(n\)-dimensional linear reconstructions.
  2. After centering, it keeps the directions of largest variance.

The second interpretation is often more intuitive; the first is usually the one to cite in a proof-based derivation.

Example. Suppose data in \(\mathbb{R}^2\) lies approximately along the line spanned by \((1,1)\), with small orthogonal noise. Then 1-dimensional PCA picks the direction \((1,1)\), projects each point onto it, and discards the orthogonal noise component.

15.5 Centering and the Gram Matrix Trick

In practice one first centers the data by subtracting the empirical mean. Otherwise the leading component may mostly capture the mean offset instead of true variation.

How to actually do PCA. Suppose the data points are \(x_1,\dots,x_m\in\mathbb{R}^d\).
  1. Compute the empirical mean \[ \bar x=\frac1m\sum_{i=1}^m x_i. \]
  2. Center the data: \[ \widetilde x_i=x_i-\bar x. \]
  3. Form the centered data matrix \[ X=\begin{pmatrix} \widetilde x_1^\top\\ \widetilde x_2^\top\\ \vdots\\ \widetilde x_m^\top \end{pmatrix}\in\mathbb{R}^{m\times d}. \]
  4. Form the scatter matrix \[ A=X^\top X=\sum_{i=1}^m \widetilde x_i \widetilde x_i^\top. \] Up to the factor \(1/m\), this is the empirical covariance matrix.
  5. Find the top \(k\) eigenvectors \(v_1,\dots,v_k\) of \(A\). These are the principal directions.
  6. To compress a point \(x\), first center it and then project: \[ z = V^\top (x-\bar x), \qquad V=[v_1\ \cdots\ v_k]. \]
  7. To reconstruct from the compressed coordinates \(z\), use \[ \widehat x = \bar x + Vz. \]
Equivalent reconstruction formula. Since \[ z=V^\top(x-\bar x), \] substituting into the decoder gives \[ \widehat x = \bar x + VV^\top(x-\bar x). \] Thus PCA reconstruction is exactly orthogonal projection of the centered vector onto the principal subspace, followed by adding the mean back.
How to reconstruct an entire data matrix. Let \(X\in\mathbb{R}^{m\times d}\) be the centered data matrix and let \[ V_k=[v_1\ \cdots\ v_k]. \] Then the rank-\(k\) PCA reconstruction of the centered data is \[ \widehat X = XV_kV_k^\top. \] If \(X_{\mathrm{raw}}\) is the original uncentered data matrix and \(\mathbf{1}\bar x^\top\) is the mean matrix, then \[ \widehat X_{\mathrm{raw}} = \mathbf{1}\bar x^\top + (X_{\mathrm{raw}}-\mathbf{1}\bar x^\top)V_kV_k^\top. \]
Step-by-step reconstruction of one point. To reconstruct a new point \(x\) from its first \(k\) principal coordinates:
  1. Subtract the mean: \(x_c=x-\bar x\).
  2. Compute the coordinates: \[ z_j=v_j^\top x_c,\qquad j=1,\dots,k. \]
  3. Rebuild the point from those coordinates: \[ \widehat x=\bar x+\sum_{j=1}^k z_j v_j. \]
So reconstruction is just “keep the components of the centered vector along the top principal directions and discard the rest.”
One-dimensional PCA recipe. If the question asks only for the first principal component, then after centering you compute the top eigenvector \(v_1\) of \[ A=X^\top X. \] The scalar coordinate of a point \(x\) on that component is \[ v_1^\top(x-\bar x), \] and its rank-1 reconstruction is \[ \bar x + v_1 v_1^\top (x-\bar x). \]
Worked PCA example. Take three points in \(\mathbb{R}^2\): \[ x_1=\begin{pmatrix}2\\ 0\end{pmatrix}, \qquad x_2=\begin{pmatrix}0\\ 2\end{pmatrix}, \qquad x_3=\begin{pmatrix}4\\ 4\end{pmatrix}. \] We will do rank-1 PCA.
  1. Compute the mean: \[ \bar x=\frac{x_1+x_2+x_3}{3} = \begin{pmatrix}2\\ 2\end{pmatrix}. \]
  2. Center the data: \[ \widetilde x_1=\begin{pmatrix}0\\ -2\end{pmatrix}, \qquad \widetilde x_2=\begin{pmatrix}-2\\ 0\end{pmatrix}, \qquad \widetilde x_3=\begin{pmatrix}2\\ 2\end{pmatrix}. \]
  3. Form the centered data matrix and scatter matrix: \[ X= \begin{pmatrix} 0 & -2\\ -2 & 0\\ 2 & 2 \end{pmatrix}, \qquad A=X^\top X. \] \[ A=\sum_{i=1}^3 \widetilde x_i\widetilde x_i^\top = \begin{pmatrix}0 & 0\\ 0 & 4\end{pmatrix} + \begin{pmatrix}4 & 0\\ 0 & 0\end{pmatrix} + \begin{pmatrix}4 & 4\\ 4 & 4\end{pmatrix} = \begin{pmatrix}8 & 4\\ 4 & 8\end{pmatrix}. \]
  4. Find the eigenvalues and eigenvectors of \(A\). First compute \[ \det(A-\lambda I) = \det\begin{pmatrix}8-\lambda & 4\\ 4 & 8-\lambda\end{pmatrix} = (8-\lambda)^2-16 = \lambda^2-16\lambda+48. \] So the eigenvalues are \(12\) and \(4\). For \(\lambda=12\), \[ (A-12I)v = \begin{pmatrix}-4 & 4\\ 4 & -4\end{pmatrix} \begin{pmatrix}a\\ b\end{pmatrix} = \begin{pmatrix}0\\ 0\end{pmatrix}, \] which gives \(-4a+4b=0\), so \(a=b\). Thus one eigenvector is \((1,1)^\top\), and the first principal direction is \[ v_1=\frac{1}{\sqrt{2}}\begin{pmatrix}1\\ 1\end{pmatrix}. \] For \(\lambda=4\), \[ (A-4I)v = \begin{pmatrix}4 & 4\\ 4 & 4\end{pmatrix} \begin{pmatrix}a\\ b\end{pmatrix} = \begin{pmatrix}0\\ 0\end{pmatrix}, \] which gives \(4a+4b=0\), so \(a=-b\). Thus one eigenvector is \((1,-1)^\top\), and the second principal direction is \[ v_2=\frac{1}{\sqrt{2}}\begin{pmatrix}1\\ -1\end{pmatrix}. \]
  5. Compress \(x_1\) using only the first principal component: \[ z_1=v_1^\top(x_1-\bar x) = \frac{1}{\sqrt{2}}(1,1)\cdot(0,-2) = -\sqrt{2}. \]
  6. Reconstruct from this 1-dimensional code: \[ \widehat x_1=\bar x+z_1v_1 = \begin{pmatrix}2\\ 2\end{pmatrix} - \sqrt{2}\cdot \frac{1}{\sqrt{2}} \begin{pmatrix}1\\ 1\end{pmatrix} = \begin{pmatrix}1\\ 1\end{pmatrix}. \]
  7. Compress \(x_1\) using both principal components: \[ z= \begin{pmatrix} v_1^\top(x_1-\bar x)\\[2mm] v_2^\top(x_1-\bar x) \end{pmatrix} = \begin{pmatrix} -\sqrt{2}\\[2mm] \sqrt{2} \end{pmatrix}. \] This is the 2-dimensional PCA encoding of \(x_1\) in the orthonormal basis \(\{v_1,v_2\}\).
  8. Reconstruct from both principal components: \[ \widehat x_1 = \bar x + z_1v_1 + z_2v_2 = \begin{pmatrix}2\\ 2\end{pmatrix} - \begin{pmatrix}1\\ 1\end{pmatrix} + \begin{pmatrix}1\\ -1\end{pmatrix} = \begin{pmatrix}2\\ 0\end{pmatrix} =x_1. \] Keeping both components gives exact reconstruction, because in \(\mathbb{R}^2\) the two principal directions form a complete orthonormal basis.
So rank-1 PCA keeps only the direction spanned by \((1,1)\), and after centering it replaces \(x_1=(2,0)\) by its projection \((1,1)\). Using both principal components keeps all coordinates in the centered orthonormal basis, so no information is lost.
What the reconstruction error is. The residual after rank-\(k\) PCA is \[ r = x-\widehat x = (I-V_kV_k^\top)(x-\bar x). \] This is exactly the component orthogonal to the principal subspace. Hence \[ \|x-\widehat x\|_2^2 = \|(I-V_kV_k^\top)(x-\bar x)\|_2^2. \] Over the entire sample, PCA chooses \(V_k\) to make the sum of these squared residuals as small as possible.
PCA does permit reconstruction, but only after deciding which \(k\)-dimensional subspace to keep. Unlike random projection, the decoder is built into the method: project onto the learned principal subspace, then expand back in that same basis.

If \(d\gg m\), diagonalizing the \(d\times d\) matrix \(X^\top X\) is expensive. Instead diagonalize the smaller Gram matrix \[ G=XX^\top\in\mathbb{R}^{m\times m}, \] then recover the principal directions from its eigenvectors. This is the computational shortcut underlying kernel PCA as well.

Gram-matrix trick for the first principal component. Let \(\alpha_1\in\mathbb{R}^m\) be a unit eigenvector of \[ G=XX^\top \] with largest eigenvalue \(\lambda_1\). Define \[ v_1=\frac{1}{\sqrt{\lambda_1}}X^\top\alpha_1. \] Then \(v_1\) is the first principal direction in \(\mathbb{R}^d\).
Proof. Since \[ G\alpha_1 = XX^\top \alpha_1 = \lambda_1 \alpha_1, \] multiplying by \(X^\top\) gives \[ X^\top X (X^\top\alpha_1)=\lambda_1 (X^\top\alpha_1). \] Therefore \(X^\top\alpha_1\) is an eigenvector of \(X^\top X\) with eigenvalue \(\lambda_1\). Moreover, \[ \|X^\top\alpha_1\|_2^2 = \alpha_1^\top XX^\top \alpha_1 = \alpha_1^\top G\alpha_1 = \lambda_1 \|\alpha_1\|_2^2 = \lambda_1, \] since \(\|\alpha_1\|_2=1\). Hence dividing by \(\sqrt{\lambda_1}\) normalizes the vector, so \[ v_1=\lambda_1^{-1/2}X^\top\alpha_1 \] is a unit eigenvector of \(X^\top X\) for the largest eigenvalue. That is exactly the first principal direction.
How to use the Gram trick on an exam. If the ambient dimension \(d\) is huge but the sample size \(m\) is small:
  1. Center the rows of \(X\), or equivalently subtract the empirical mean from every sample.
  2. Compute the \(m\times m\) Gram matrix \(G=XX^\top\).
  3. Find its top eigenpair \((\lambda_1,\alpha_1)\).
  4. Recover the first principal direction by \[ v_1=\lambda_1^{-1/2}X^\top\alpha_1. \]
  5. Project any centered point \(x-\bar x\) by computing \(v_1^\top(x-\bar x)\).
The point is that diagonalizing \(G\) costs work in \(m\), not in \(d\).
The first principal component always lies in the span of the centered data points. The Gram-matrix trick works because it first finds the right coefficients \(\alpha_1\) for that linear combination in sample space, then converts them back into an ambient-space vector \(v_1\).
Forgetting to center is one of the standard PCA mistakes. It can completely change the leading direction.

15.6 PCA and SVD: Which Singular Vectors Are the Principal Components?

Let \(X\in\mathbb{R}^{m\times d}\) be the centered data matrix used in these notes, with one sample per row. Write its singular value decomposition as \[ X=U\Sigma V^\top. \] Then \[ X^\top X = V\Sigma^2 V^\top, \qquad XX^\top = U\Sigma^2 U^\top. \] Since PCA diagonalizes \(X^\top X\), the principal directions are the eigenvectors of \(X^\top X\), which are the columns of \(V\).

Main identification. For the centered data convention \(X\in\mathbb{R}^{m\times d}\):
  1. The right singular vectors \(v_1,v_2,\dots\) are the principal directions in feature space.
  2. The eigenvalues of the PCA scatter matrix \(X^\top X\) are \(\sigma_1^2,\sigma_2^2,\dots\).
  3. The principal-component score matrix is \[ XV = U\Sigma. \] Its \(j\)-th column gives the scores of all samples on the \(j\)-th principal component.

This is why PCA and SVD are often presented as the same computation in different language: SVD gives the orthonormal directions \(v_i\), and PCA keeps the top few of them.

In these notes, samples are rows. So the feature covariance/scatter matrix is \(X^\top X\), the principal directions are the right singular vectors, and the left singular vectors describe orthonormal directions in sample space rather than feature space.

Part XVI: Random Projections

16.1 The Core Idea

Random projection is a dimension reduction method that does not fit the data. Instead it multiplies by a random matrix and relies on concentration of measure to preserve norms and distances with high probability.

16.2 Single-Vector Concentration

If \(W\) is a Gaussian random matrix and \(x\) is fixed, then

\[ \mathbb{P}\!\left( \left| \frac{\|(1/\sqrt n)Wx\|_2^2}{\|x\|_2^2}-1 \right|\gt\epsilon \right) \le 2e^{-\epsilon^2 n/6}. \]

The proof comes from concentration of \(\chi^2\) random variables. This is the norm-preservation guarantee for one vector.

16.3 Johnson-Lindenstrauss Lemma

A union bound upgrades the one-vector guarantee to a finite set \(Q\). The Johnson-Lindenstrauss conclusion is that dimension \(O(\log(|Q|/\delta)/\epsilon^2)\) suffices to preserve all norms in the set, and hence all pairwise distances, within factor \(1\pm\epsilon\) with probability at least \(1-\delta\).

JL principle. The target dimension depends logarithmically on the number of points whose geometry must be preserved and does not depend on the original ambient dimension.

Proof. Let \(Q\) be a finite set of vectors. For each fixed nonzero \(u\in Q\), the single-vector concentration result says \[ \mathbb{P}\!\left( \left| \frac{\|(1/\sqrt n)Wu\|_2^2}{\|u\|_2^2}-1 \right|\gt\epsilon \right) \le 2e^{-\epsilon^2 n/6}. \] Therefore, by the union bound, \[ \mathbb{P}\!\left( \exists u\in Q: \left| \frac{\|(1/\sqrt n)Wu\|_2^2}{\|u\|_2^2}-1 \right|\gt\epsilon \right) \le 2|Q|e^{-\epsilon^2 n/6}. \] To make the right-hand side at most \(\delta\), it suffices that \[ 2|Q|e^{-\epsilon^2 n/6}\le \delta, \] equivalently \[ n \ge \frac{6}{\epsilon^2}\log\!\left(\frac{2|Q|}{\delta}\right). \] Thus, with probability at least \(1-\delta\), every vector in \(Q\) has its norm preserved within multiplicative error \(1\pm \epsilon\). Applying this to the difference set \[ Q = \{x-y : x,y\in S,\ x\neq y\} \] for a finite point set \(S\) proves that all pairwise distances in \(S\) are preserved simultaneously under the same dimension bound. This is the Johnson-Lindenstrauss lemma.

Corollary. Since pairwise distances are controlled, nearest-neighbor structure and other geometric relationships are approximately preserved in the projected space. This explains why random projections can be useful even when the ambient dimension is enormous.

This is the main surprise: high ambient dimension alone is not the bottleneck.

16.4 What Random Projection Really Is

Random projection is not trying to discover the best low-dimensional structure in the data. It chooses a random linear map \[ R:\mathbb{R}^d\to\mathbb{R}^n \] and uses \[ x\longmapsto \frac{1}{\sqrt n}Rx \] as a fast embedding. The point is not optimal reconstruction. The point is that with high probability this blind map approximately preserves Euclidean geometry on the finite set of vectors you care about.

What is actually preserved. If norms are preserved for all differences \(x-y\), then pairwise distances are preserved: \[ \left\|\frac{1}{\sqrt n}R(x-y)\right\|_2^2 \approx \|x-y\|_2^2. \] So nearest neighbors, cluster separation, and margin geometry are approximately unchanged in the projected space.
Example. Suppose \(1000\) points lie in \(\mathbb{R}^{100000}\), and you only need pairwise distances up to \(10\%\) distortion. Johnson-Lindenstrauss says that a target dimension on the order of \[ n=O\!\left(\frac{\log 1000}{0.1^2}\right) \] is enough, independent of the original dimension \(100000\). So the geometry can often be compressed to a few hundred or thousand coordinates without learning any model from the data.
How to actually use random projection.
  1. Choose a target dimension \(n\) from the JL bound.
  2. Draw a random matrix \(R\in\mathbb{R}^{n\times d}\), often Gaussian with entries \(N(0,1)\).
  3. Project each vector by \[ z=\frac{1}{\sqrt n}Rx. \]
  4. Run the downstream geometric task in the projected space: nearest neighbors, clustering, margin computations, or approximate distance queries.
Can you reconstruct the original vector? In general, no exact reconstruction is possible when \(n\lt d\), because the map \[ x\mapsto \frac1{\sqrt n}Rx \] is not injective. Distinct vectors can have the same projection. Random projection is therefore not a compression-and-perfect-recovery method like an invertible change of basis.
Best approximate reconstruction from the projection. If \(z=(1/\sqrt n)Rx\) is known and \(R\) is known, a standard approximate lift is the minimum-norm solution of \[ \frac1{\sqrt n}Ru=z. \] This is \[ \widehat x = \sqrt n\,R^+ z, \] where \(R^+\) is the Moore-Penrose pseudoinverse. This recovers the orthogonal projection of \(x\) onto the row space of \(R\), not the full original vector.
Why the pseudoinverse gives the row-space component. Let \(P=R^+R\). Then \(P\) is the orthogonal projector onto the row space of \(R\). Since \[ z=\frac1{\sqrt n}Rx, \] we have \[ \widehat x=\sqrt n\,R^+z = R^+Rx = Px. \] So \(\widehat x\) is exactly the part of \(x\) visible through the rows of \(R\). Any component of \(x\) in the null space of \(R\) is lost.
Finite-set recovery viewpoint. If the vector is known in advance to lie in a finite set \(S\), then reconstruction can mean: project every candidate in \(S\), then given \(z\), choose the candidate whose projection is closest to \(z\). Because JL preserves pairwise distances, this nearest-projected-point rule can identify the original point approximately, and sometimes exactly if the original points were well separated.

16.5 Contrast With PCA

PCA solves an optimal reconstruction problem. Random projection solves a geometry-preservation problem. These are different objectives, so the methods are not interchangeable.

Example. For the standard basis vectors \(e_1,\dots,e_d\), random projection can preserve geometry in \(O(\log d)\) dimensions, whereas PCA performs badly because no low-dimensional subspace gives small average reconstruction error for all basis vectors.
If the prompt says “preserve distances,” think JL/random projection. If it says “best low-rank reconstruction,” think PCA.

Part XVII: Singular Value Decomposition

17.1 Singular Vectors and Singular Values

For a matrix \(A\in\mathbb{R}^{m\times n}\), unit vectors \(v\in\mathbb{R}^n\) and \(u\in\mathbb{R}^m\) are right and left singular vectors with singular value \(\sigma\gt 0\) if

\[ Av=\sigma u, \qquad A^\top u=\sigma v. \]

This means that the matrix maps input direction \(v\) to output direction \(u\), scaled by \(\sigma\).

17.2 The Decomposition Theorem

SVD theorem. If \(A\) has rank \(r\), then \[ A=UDV^\top, \] where the columns of \(U\) and \(V\) are orthonormal left and right singular vectors and \(D\) is diagonal with the positive singular values on its diagonal.

Proof. Consider \(A^\top A\), which is symmetric positive semidefinite. By the spectral theorem, there exists an orthonormal basis \(v_1,\dots,v_n\) of eigenvectors with corresponding eigenvalues \(\lambda_1,\dots,\lambda_n\ge 0\). Since \(A\) has rank \(r\), exactly \(r\) of these eigenvalues are positive. Relabel so that \(\lambda_1,\dots,\lambda_r\gt 0\). For each \(i\le r\), define \[ \sigma_i = \sqrt{\lambda_i}, \qquad u_i = \sigma_i^{-1} A v_i. \] Then \[ Av_i = \sigma_i u_i, \qquad A^\top u_i = \sigma_i v_i, \] because \[ A^\top u_i = \sigma_i^{-1}A^\top A v_i = \sigma_i^{-1}\lambda_i v_i = \sigma_i v_i. \] Thus \((u_i,v_i,\sigma_i)\) are singular triples.

We claim that \(u_1,\dots,u_r\) are orthonormal. Indeed, for \(i,j\le r\), \[ \langle u_i,u_j\rangle = \sigma_i^{-1}\sigma_j^{-1}\langle Av_i,Av_j\rangle = \sigma_i^{-1}\sigma_j^{-1}\langle v_i,A^\top A v_j\rangle = \sigma_i^{-1}\sigma_j^{-1}\lambda_j \langle v_i,v_j\rangle. \] Since the \(v_i\) are orthonormal and \(\lambda_j=\sigma_j^2\), this equals \(0\) if \(i\neq j\) and \(1\) if \(i=j\). So the \(u_i\) are orthonormal.

Let \(U\) be the matrix with columns \(u_1,\dots,u_r\), let \(V\) be the matrix with columns \(v_1,\dots,v_r\), and let \(D\) be the diagonal matrix with entries \(\sigma_1,\dots,\sigma_r\). Then \[ UDV^\top = \sum_{i=1}^r \sigma_i u_i v_i^\top. \] For each \(v_i\) with \(i\le r\), we have \[ UDV^\top v_i = \sigma_i u_i = Av_i. \] For each eigenvector \(v_j\) with \(j\gt r\), the eigenvalue is \(0\), so \(A^\top A v_j=0\), which implies \(\|Av_j\|_2^2 = v_j^\top A^\top A v_j = 0\), hence \(Av_j=0\). Also \(UDV^\top v_j=0\). Therefore \(A\) and \(UDV^\top\) agree on the basis \(v_1,\dots,v_n\), so they are equal. Thus \[ A = UDV^\top, \] which is the singular value decomposition.

Corollary. Truncating the SVD gives the best low-rank approximation in spectral and Frobenius norm. This is the linear algebra statement that sits behind PCA and latent semantic indexing.

The right singular vectors are eigenvectors of \(A^\top A\), and the singular values are square roots of the corresponding nonzero eigenvalues. The left singular vectors are then recovered by \(u_i=\sigma_i^{-1}Av_i\).

Algorithm for computing the SVD in \(O(m^2n)\) when \(m\le n\). Let \(A\in\mathbb{R}^{m\times n}\).
  1. Form the symmetric matrix \(B=AA^\top\in\mathbb{R}^{m\times m}\).
  2. Compute an orthonormal eigenbasis \(u_1,\dots,u_m\) of \(B\) with eigenvalues \(\lambda_1,\dots,\lambda_m\ge 0\).
  3. Set \(\sigma_i=\sqrt{\lambda_i}\). These are the singular values of \(A\).
  4. For each \(i\) with \(\sigma_i\gt 0\), define \[ v_i=\sigma_i^{-1}A^\top u_i. \] Then \(Av_i=\sigma_i u_i\) and \(A^\top u_i=\sigma_i v_i\), so \((u_i,v_i,\sigma_i)\) is a singular triple.
  5. If a full SVD is required, extend the nonzero singular vectors to orthonormal bases of \(\mathbb{R}^m\) and \(\mathbb{R}^n\).
Why the running time is \(O(m^2n)\). Forming \(AA^\top\) costs \(O(m^2n)\), since each of the \(m^2\) entries is a dot product of length \(n\). Diagonalizing the \(m\times m\) symmetric matrix \(B\) costs \(O(m^3)\). Recovering each right singular vector \(v_i=\sigma_i^{-1}A^\top u_i\) costs \(O(mn)\), so recovering all \(m\) of them costs \(O(m^2n)\). Hence the total cost is \[ O(m^2n)+O(m^3)+O(m^2n). \] Because \(m\le n\), we have \(m^3\le m^2n\), so the total running time is \(O(m^2n)\). If \(m\gt n\), apply the same method to \(A^\top\), compute the SVD of \(A^\top\) in \(O(n^2m)\), and transpose the result back.

17.3 Geometric Meaning

SVD decomposes a matrix into three pieces: an orthogonal change of input coordinates, a diagonal scaling, and an orthogonal change of output coordinates. Thus it gives the canonical geometric description of a linear map.

Rank becomes transparent: the number of nonzero singular values is exactly the rank. All input directions orthogonal to the right singular vectors corresponding to nonzero singular values are sent to zero.

Example. If \(A\) has rank \(r\), then only \(r\) orthogonal input directions survive under \(x\mapsto Ax\). Everything orthogonal to those directions lies in the null space.

17.4 Why SVD Matters: Three Standard Uses

In proofs and applications, SVD is useful because it reduces an arbitrary matrix to a diagonal one after orthogonal changes of coordinates. Orthogonal matrices preserve Euclidean lengths and inner products, so many questions about \(A\) become simple one-dimensional questions about its singular values.

Spectral facts to remember. If \[ A = UDV^\top, \qquad D=\operatorname{diag}(\sigma_1,\dots,\sigma_r,0,\dots,0), \qquad \sigma_1\ge \sigma_2\ge \cdots \ge \sigma_r\gt 0, \] then: \[ \|A\|_{2\to 2}=\sigma_1, \qquad \|A\|_F^2=\sum_{i=1}^r \sigma_i^2, \qquad \operatorname{rank}(A)=|\{i:\sigma_i\gt 0\}|. \] Also, for every \(x\in\mathbb{R}^n\), \[ \|Ax\|_2^2 = \|DV^\top x\|_2^2 = \sum_{i=1}^r \sigma_i^2 \langle v_i,x\rangle^2. \] This formula shows exactly how each right singular direction contributes to the output norm.
Proof that \(\|A\|_{2\to 2}=\sigma_1\). For any unit vector \(x\), write \(y=V^\top x\). Since \(V\) is orthogonal, \(\|y\|_2=1\). Then \[ \|Ax\|_2^2 = \|UDV^\top x\|_2^2 = \|Dy\|_2^2 = \sum_{i=1}^r \sigma_i^2 y_i^2 \le \sigma_1^2 \sum_{i=1}^r y_i^2 \le \sigma_1^2. \] Hence \(\|A\|_{2\to 2}\le \sigma_1\). Equality is achieved by taking \(x=v_1\), since \[ Av_1=\sigma_1 u_1 \quad\text{and therefore}\quad \|Av_1\|_2=\sigma_1. \] So \(\|A\|_{2\to 2}=\sigma_1\).
Use 1: maximize or bound \(\|Ax\|_2\). If a problem asks for \[ \max_{\|x\|_2=1}\|Ax\|_2, \] the answer is the largest singular value \(\sigma_1\), attained at the top right singular vector \(v_1\). More generally, if you must show that \(\|Ax\|_2\le c\|x\|_2\) for all \(x\), it is enough to show \(\sigma_1\le c\).
Worked example. Let \[ A=\begin{pmatrix}3&0\\0&1\end{pmatrix}. \] The singular values are \(3\) and \(1\). Therefore \[ \max_{\|x\|_2=1}\|Ax\|_2=3. \] The maximizing direction is \(x=e_1\). Geometrically, the unit circle is sent to an ellipse whose semiaxes have lengths \(3\) and \(1\).
Use 2: best low-rank approximation. If \[ A=\sum_{i=1}^r \sigma_i u_i v_i^\top, \] then the best rank-\(k\) approximation is \[ A_k=\sum_{i=1}^k \sigma_i u_i v_i^\top. \] This is what you use when a problem asks for the closest rank-\(k\) matrix to \(A\), the smallest error after discarding directions, or the optimal spectral/Frobenius compression.
Error formulas for truncation. For the truncated SVD \(A_k\), \[ \|A-A_k\|_{2\to 2}=\sigma_{k+1}, \qquad \|A-A_k\|_F^2=\sum_{i\gt k}\sigma_i^2. \] So the singular values tell you exactly how much information is lost when you keep only the top \(k\) directions.
Worked example. Suppose \[ A=\sum_{i=1}^3 \sigma_i u_i v_i^\top \qquad\text{with}\qquad \sigma_1=10,\ \sigma_2=4,\ \sigma_3=1. \] Then the best rank-1 approximation is \(A_1=10u_1v_1^\top\). Its spectral error is \[ \|A-A_1\|_{2\to 2}=4, \] and its Frobenius error is \[ \|A-A_1\|_F=\sqrt{4^2+1^2}=\sqrt{17}. \] On an exam, this lets you answer low-rank approximation questions without reconstructing the full matrix.
Use 3: solve least-squares and pseudoinverse problems. If \(A=UDV^\top\) has nonzero singular values \(\sigma_1,\dots,\sigma_r\), define \[ A^+ = V D^+ U^\top, \] where \(D^+\) has diagonal entries \(\sigma_i^{-1}\) on the nonzero singular values and \(0\) elsewhere. Then \(x^\star=A^+b\) is the minimum-norm least-squares solution to \[ \min_x \|Ax-b\|_2. \] This matters whenever the system is rectangular or singular.
Why the pseudoinverse solves least squares. Write \(x=Vz\). Since \(U\) and \(V\) are orthogonal, \[ \|Ax-b\|_2 = \|UDV^\top x-b\|_2 = \|Dz-U^\top b\|_2. \] Thus the minimization decouples coordinate-by-coordinate. For each \(i\le r\), the optimal choice is \(z_i=(u_i^\top b)/\sigma_i\), and for \(i\gt r\) the residual cannot be affected by \(z_i\). The minimum-norm choice sets those null-space coordinates to \(0\). Hence \[ z=D^+U^\top b, \qquad x^\star = Vz = VD^+U^\top b = A^+b. \]
Worked example. Let \[ A=\begin{pmatrix}1&0\\0&0\end{pmatrix}, \qquad b=\begin{pmatrix}2\\3\end{pmatrix}. \] Then the SVD has one nonzero singular value \(\sigma_1=1\), and \[ A^+=\begin{pmatrix}1&0\\0&0\end{pmatrix}. \] Therefore \[ x^\star=A^+b=\begin{pmatrix}2\\0\end{pmatrix}. \] Indeed, \[ Ax^\star=\begin{pmatrix}2\\0\end{pmatrix}, \] so the residual is \((0,-3)\), and among all least-squares minimizers this one has smallest Euclidean norm.
Spectral viewpoint. Eigenvalues describe invariant directions of a square operator. Singular values describe stretching behavior of any matrix, square or rectangular. When a problem asks for the most amplified direction, the best rank-\(k\) summary, the norm of a linear map, or the minimum-norm solution of a linear system, SVD is usually the right tool because it turns the problem into diagonal arithmetic.

17.5 Relation to PCA

PCA is the statistical interpretation of SVD on a centered data matrix. The right singular vectors are the principal directions, and the singular values determine how much variance is captured along them.

Part XVIII: Latent Semantic Indexing

18.1 Term-Document Matrices

Start with a term-document matrix \(A\), with entries given by counts or TF-IDF weights. In the raw bag-of-words model, retrieval is based on direct overlap of observed terms. This is brittle: related documents can use different vocabularies, and the same word can appear in different topics.

In a homework or proof setting, the first step is usually completely explicit: choose an ordered vocabulary \(t_1,\dots,t_w\) and an ordered document list \(d_1,\dots,d_m\), then define \[ A_{ij} = \text{number of occurrences of term } t_i \text{ in document } d_j. \] Once the order is fixed, the matrix is determined. Singular/plural normalization or stemming must be stated as part of the preprocessing rule.

Construction procedure. To build a term-document matrix for an exam problem: \[ \text{choose term order} \longrightarrow \text{choose document order} \longrightarrow \text{count occurrences} \longrightarrow \text{fill columns of } A. \] The \(j\)-th column is the term-frequency vector of document \(d_j\).
Mini-example. If the vocabulary is \((\text{apple},\text{computer},\text{economy})\) and the documents are \[ d_1:\ \texttt{apple computer}, \qquad d_2:\ \texttt{economy economy}, \] then \[ A= \begin{bmatrix} 1 & 0\\ 1 & 0\\ 0 & 2 \end{bmatrix}. \] Each column is a document vector; each row tracks one term across documents.

18.2 Column Normalization and Cosine Similarity

Raw counts depend heavily on document length. A standard normalization rescales each document vector to have unit Euclidean norm. If \(a_j\) denotes the \(j\)-th column of \(A\), define \[ b_j = \frac{a_j}{\|a_j\|_2}, \qquad B=[b_1\ \cdots\ b_m]. \]

Proposition. For normalized columns, cosine similarity equals the Euclidean inner product: \[ \cos(b_i,b_j)=\frac{b_i^\top b_j}{\|b_i\|_2\|b_j\|_2}=b_i^\top b_j. \] Hence the document-document similarity matrix is \[ B^\top B. \]

Proof. By construction \(\|b_i\|_2=\|b_j\|_2=1\), so the denominator in the cosine formula is \(1\). Therefore cosine similarity reduces to the dot product. The \((i,j)\)-entry of \(B^\top B\) is exactly \(b_i^\top b_j\), so \(B^\top B\) stores all pairwise cosine similarities.

Example. With \[ A= \begin{bmatrix} 1 & 0\\ 1 & 0\\ 0 & 2 \end{bmatrix}, \] the normalized columns are \[ b_1=\frac{1}{\sqrt 2}(1,1,0)^\top, \qquad b_2=(0,0,1)^\top. \] Their cosine similarity is \[ b_1^\top b_2 = 0, \] which correctly records that the documents share no vocabulary after preprocessing.

18.3 The LSI Construction

Latent Semantic Indexing applies truncated SVD to the term-document matrix:

\[ A \approx U_k D_k V_k^\top. \]

The top \(k\) singular directions define a low-rank latent space. Queries and documents are represented in that space rather than by their raw sparse coordinates.

If one starts from the normalized matrix \(B\), then one similarly writes \[ B = UDV^\top, \qquad B \approx U_k D_k V_k^\top. \] The columns of \(V_k\) describe latent coordinates for documents, the columns of \(U_k\) describe latent coordinates for terms, and the diagonal entries of \(D_k\) measure the importance of those latent directions.

Procedure. The practical workflow for the standard homework-style LSI problem is:
  1. Construct the term-document matrix \(A\).
  2. Normalize columns to obtain \(B\).
  3. Compute \(B^\top B\) for cosine similarities.
  4. Compute the singular value decomposition \(B=UDV^\top\).
  5. Inspect the top singular values and the first few left/right singular vectors.
  6. Use the rank-\(k\) truncation to study latent structure.

18.4 Why Low-Rank Structure Helps

Truncation preserves the dominant co-occurrence patterns while discarding many small, vocabulary-specific fluctuations. Therefore, words and documents that behave similarly across the corpus can become close in the latent space even without exact lexical overlap.

Example. A query about "cars" may retrieve a document emphasizing "automobiles" if the two terms appear in similar patterns across the document collection. Truncated SVD places them near one another in latent space.

18.5 Interpreting the Singular Vectors

In a term-document matrix, the right singular vectors describe document-side directions and the left singular vectors describe term-side directions. Large coordinates in the first right singular vector indicate documents participating strongly in the dominant topic; large coordinates in the corresponding left singular vector indicate the terms most associated with that topic.

This is why plotting the first two left or right singular vectors can reveal clusters. Documents about similar themes tend to have similar coordinates in the leading right singular directions.

Interpretation example. If the corpus splits roughly into a technology topic and a macroeconomy topic, then one leading singular direction often separates those two vocabularies. Documents about software may have positive coordinates while documents about recession and growth have negative coordinates. The sign itself is arbitrary, but the separation is meaningful.

18.6 Relation to PCA and Its Limitation

LSI is the text-retrieval analogue of the low-rank viewpoint behind PCA and SVD. The mathematical idea is the same: keep the dominant low-dimensional structure of a matrix. The limitation is also clear: LSI is linear and global, so it captures coarse co-occurrence structure rather than true contextual semantics.

LSI improves raw keyword matching by replacing exact lexical identity with low-rank co-occurrence structure, but it remains a linear model rather than a contextual semantic model.

Part XIX: Matching Loss and Logistic Regression

19.1 Definition of the Matching Loss

Suppose \[ \mathbb{E}[Y\mid X=x] = u(w_\ast^\top x), \] where \(u:\mathbb R\to\mathbb R\) is monotone increasing and \(1\)-Lipschitz. The matching loss at a point \((x,y)\) is \[ \ell(w;x,y)=\int_0^{w^\top x} (u(z)-y)\,dz. \] The empirical loss on a sample \(S=\{(x_i,y_i)\}_{i=1}^m\) is \[ R_S^\ell(w)=\frac1m\sum_{i=1}^m \ell(w;x_i,y_i). \]

The point of this definition is that differentiating with respect to \(w\) produces the residual \(u(w^\top x)-y\) without an additional factor of \(u'(w^\top x)\). That is why the loss is called a matching loss: it is tailored so that the gradient matches the natural residual expression for the generalized linear model.

19.2 Gradient and Comparison With Square Loss

Proposition. The gradient of the empirical matching loss is \[ \nabla R_S^\ell(w) = \frac1m \sum_{i=1}^m \big(u(w^\top x_i)-y_i\big)x_i. \]

Proof. Fix \(i\). Define \[ g_i(w)=\int_0^{w^\top x_i}(u(z)-y_i)\,dz. \] By the fundamental theorem of calculus and the chain rule, \[ \nabla g_i(w) = (u(w^\top x_i)-y_i)\nabla(w^\top x_i) = (u(w^\top x_i)-y_i)x_i. \] Averaging over \(i\) yields the formula.

Compare this with square loss for the predictor \(u(w^\top x)\): \[ R_S^{\mathrm{sq}}(w)=\frac1m\sum_{i=1}^m \big(u(w^\top x_i)-y_i\big)^2. \] Differentiating gives \[ \nabla R_S^{\mathrm{sq}}(w) = \frac{2}{m}\sum_{i=1}^m \big(u(w^\top x_i)-y_i\big)u'(w^\top x_i)x_i. \] The extra factor \(u'(w^\top x_i)\) can be problematic when \(u'\) is small, as happens for sigmoids in the saturated regime. The matching loss removes that factor.

Matching loss and square loss have the same residual term \(u(w^\top x)-y\), but only square loss carries the extra multiplier \(u'(w^\top x)\). This is the structural reason the matching loss behaves better for monotone link functions.

19.3 Convexity via the Hessian

Theorem. If \(u\) is monotone increasing, then \(R_S^\ell(w)\) is convex in \(w\).

Proof. Differentiate the gradient formula once more: \[ \nabla^2 R_S^\ell(w) = \frac1m\sum_{i=1}^m u'(w^\top x_i)\,x_i x_i^\top. \] Since \(u\) is monotone increasing, \(u'(t)\ge 0\) wherever the derivative exists. Each matrix \(x_i x_i^\top\) is positive semidefinite, and a nonnegative scalar multiple of a positive semidefinite matrix is again positive semidefinite. Therefore \(\nabla^2 R_S^\ell(w)\) is positive semidefinite.

To make the final step explicit, for any vector \(v\), \[ v^\top \nabla^2 R_S^\ell(w) v = \frac1m \sum_{i=1}^m u'(w^\top x_i)\, (v^\top x_i)^2 \ge 0. \] Hence the Hessian is positive semidefinite, so \(R_S^\ell\) is convex.

Corollary. Unlike the empirical square loss for the nonlinear predictor \(u(w^\top x)\), the matching loss remains convex even when \(u\) itself is nonconvex, provided only that \(u\) is monotone increasing.

19.4 Matching Loss Dominates the Squared Error in Expectation

Fix \(x\), and write \[ a=w^\top x, \qquad a_\ast = w_\ast^\top x, \qquad \Delta = u(a)-u(a_\ast). \] Since \(\mathbb{E}[Y\mid X=x]=u(a_\ast)\), one has \[ \mathbb{E}_{Y\sim D_x}[\ell(w;x,Y)]-\mathbb{E}_{Y\sim D_x}[\ell(w_\ast;x,Y)] = \int_{a_\ast}^{a}\big(u(z)-u(a_\ast)\big)\,dz. \]

Theorem. Assume \(u\) is monotone increasing and \(1\)-Lipschitz. Then for every fixed \(x\), \[ \mathbb{E}_{Y\sim D_x}[\ell(w;x,Y)]-\mathbb{E}_{Y\sim D_x}[\ell(w_\ast;x,Y)] \ge \frac12\big(u(w^\top x)-u(w_\ast^\top x)\big)^2. \]

Proof. Assume first that \(a\ge a_\ast\); the case \(a\le a_\ast\) is identical after reversing the roles of the endpoints. Set \[ g(z)=u(z)-u(a_\ast), \qquad z\in[a_\ast,a]. \] Then \(g\) is nonnegative and monotone increasing, and \[ g(a)=u(a)-u(a_\ast)=\Delta. \] Because \(u\) is \(1\)-Lipschitz, for every \(z\in[a_\ast,a]\), \[ \Delta-g(z)=u(a)-u(z)\le a-z, \] hence \[ g(z)\ge \Delta-(a-z)=z-(a-\Delta). \] Since \(g(z)\ge 0\) as well, we obtain the lower bound \[ g(z)\ge \max\{0,z-(a-\Delta)\}. \] Therefore \[ \int_{a_\ast}^{a} g(z)\,dz \ge \int_{a_\ast}^{a} \max\{0,z-(a-\Delta)\}\,dz. \] The right-hand side is exactly the area of a triangle of base \(\Delta\) and height \(\Delta\): \[ \int_{a-\Delta}^{a} (z-(a-\Delta))\,dz = \frac{\Delta^2}{2}. \] Hence \[ \int_{a_\ast}^{a}\big(u(z)-u(a_\ast)\big)\,dz \ge \frac{\Delta^2}{2}. \] Replacing \(\Delta\) by \(u(w^\top x)-u(w_\ast^\top x)\) proves the claim.

Corollary. Minimizing the expected matching loss also controls the squared prediction error \(\big(u(w^\top x)-u(w_\ast^\top x)\big)^2\). Thus the matching loss is a convex upper envelope for the prediction error induced by the link function \(u\).

19.5 Sigmoid Link and Logistic Loss

Take \[ u(z)=\frac{1}{1+e^{-z}}, \] the sigmoid function, and labels \(y\in\{0,1\}\). Then \[ \int u(z)\,dz = \log(1+e^z)+C, \] because \[ \frac{d}{dz}\log(1+e^z)=\frac{e^z}{1+e^z}=\frac{1}{1+e^{-z}}=u(z). \]

Proposition. For the sigmoid link, \[ \ell(w;x,y) = \log(1+e^{w^\top x}) - y\,w^\top x - \log 2. \] Up to the additive constant \(-\log 2\), this is the logistic loss.

Proof. Substitute the antiderivative into the definition: \[ \ell(w;x,y) = \int_0^{w^\top x}(u(z)-y)\,dz = \big[\log(1+e^z)-yz\big]_{z=0}^{z=w^\top x}. \] Therefore \[ \ell(w;x,y) = \log(1+e^{w^\top x})-y\,w^\top x-\log(1+e^0) = \log(1+e^{w^\top x})-y\,w^\top x-\log 2. \] Since additive constants do not affect optimization, the effective loss is \[ \log(1+e^{w^\top x})-y\,w^\top x, \] which is the logistic loss.

Equivalent binary forms. If one instead uses labels in \(\{\pm1\}\), logistic loss is often written as \[ \log(1+e^{-y w^\top x}). \] The two formulas are the same loss written under different label encodings.

19.6 Logistic Regression as Maximum Likelihood

If the labels are encoded as \(y_i\in\{\pm 1\}\) and one models \[ \mathbb{P}[Y=y\mid X=x] = \sigma(y\,w^\top x), \] then the sample likelihood is \[ \mathcal L(w)=\prod_{i=1}^m \sigma(y_i w^\top x_i). \]

Proposition. Maximizing the logistic likelihood is equivalent to minimizing the logistic loss \[ \sum_{i=1}^m \log\!\left(1+e^{-y_i w^\top x_i}\right). \]

Proof. Take logarithms: \[ \log \mathcal L(w)=\sum_{i=1}^m \log(\sigma(y_i w^\top x_i)). \] Since \[ \sigma(z)=\frac{1}{1+e^{-z}}, \] one has \[ \log(\sigma(z)) = -\log(1+e^{-z}). \] Therefore \[ \log \mathcal L(w) = -\sum_{i=1}^m \log(1+e^{-y_i w^\top x_i}). \] Maximizing \(\log \mathcal L(w)\) is thus equivalent to minimizing \[ \sum_{i=1}^m \log(1+e^{-y_i w^\top x_i}), \] which is the logistic loss.

Corollary. Logistic regression is a convex optimization problem because this objective is convex in \(w\). This is the likelihood-theoretic reason one can efficiently optimize it with gradient methods.

19.7 One-Hidden-Layer Neural Networks as a Nonlinear Logistic Model

Logistic regression has the form \[ \mathbb{P}[Y=1\mid X=x]=\sigma(w^\top x), \] while a one-hidden-layer neural network replaces the linear score \(w^\top x\) by a nonlinear feature map: \[ \mathbb{P}[Y=1\mid X=x] = \sigma\!\left(\sum_{j=1}^m a_j\,\sigma((w^{(j)})^\top x)\right). \]

This comparison is structurally important. Logistic regression is convex because the score is linear in the parameters. Once hidden layers are introduced, the score becomes a nonlinear function of the weights, and the optimization ceases to be convex even if the outermost link is still sigmoid.

Part XX: k-Means Clustering

20.1 Objective and Alternating-Minimization Structure

Given points \(x_1,\dots,x_m\in\mathbb R^d\) and a target number of clusters \(k\), the k-means objective is \[ \Phi(c,\mu) = \sum_{i=1}^m \|x_i-\mu_{c_i}\|_2^2, \] where \(c_i\in\{1,\dots,k\}\) is the cluster assignment of point \(x_i\) and \(\mu_1,\dots,\mu_k\in\mathbb R^d\) are the cluster centers.

Lloyd's algorithm alternates between two exact minimization steps:

  1. Assignment step: with centers fixed, assign each point to the nearest center.
  2. Center update step: with assignments fixed, replace each center by the mean of its assigned points.

The theorem that k-means never increases its objective is a direct consequence of these two exact minimizations.

20.2 Monotonicity of the k-Means Objective

Theorem. At every iteration of Lloyd's algorithm, the k-means objective \(\Phi\) does not increase.

Proof. Fix the centers \(\mu_1,\dots,\mu_k\). Then \[ \Phi(c,\mu)=\sum_{i=1}^m \|x_i-\mu_{c_i}\|_2^2 \] is a sum of terms depending on the assignments \(c_i\) separately. For each point \(x_i\), the value \(\|x_i-\mu_{c_i}\|_2^2\) is minimized by choosing \(c_i\) to be the index of the nearest center. Therefore the assignment step cannot increase \(\Phi\).

Now fix the assignments \(c_1,\dots,c_m\). For cluster \(j\), let \[ C_j=\{i:c_i=j\} \] and consider the contribution of that cluster: \[ \Phi_j(\mu_j)=\sum_{i\in C_j}\|x_i-\mu_j\|_2^2. \] Expanding the square gives \[ \Phi_j(\mu_j) = \sum_{i\in C_j}\|x_i\|_2^2 -2\mu_j^\top \sum_{i\in C_j} x_i +|C_j|\,\|\mu_j\|_2^2. \] Differentiating with respect to \(\mu_j\) yields \[ \nabla \Phi_j(\mu_j) = -2\sum_{i\in C_j} x_i + 2|C_j|\,\mu_j. \] Setting the gradient equal to zero gives the unique minimizer \[ \mu_j = \frac{1}{|C_j|}\sum_{i\in C_j} x_i, \] the mean of the assigned points. Hence the center-update step cannot increase \(\Phi\) either.

Since each full iteration consists of these two nonincreasing steps, the objective is nonincreasing from one iteration to the next.

Corollary. Because there are only finitely many assignments \(c\), Lloyd's algorithm eventually terminates at a fixed point. The theorem does not say this fixed point is globally optimal; it only guarantees descent.

20.3 Why k-Means Can Converge to a Bad Local Optimum

The monotonicity theorem proves only local descent, not global optimality. A small explicit example shows the gap clearly.

Example. Consider the four points \[ (0,0),\ (0,2),\ (10,0),\ (10,2) \] with \(k=2\).

The globally optimal clustering groups the left column together and the right column together. The corresponding means are \[ (0,1),\qquad (10,1), \] and the objective value is \[ 1+1+1+1 = 4. \]

However, if Lloyd's algorithm is initialized with centers \[ \mu_1^{(0)}=(0,0), \qquad \mu_2^{(0)}=(0,2), \] then the first assignment step sends the points \((0,0)\) and \((10,0)\) to \(\mu_1^{(0)}\), and the points \((0,2)\) and \((10,2)\) to \(\mu_2^{(0)}\). Thus the algorithm forms the bottom row and top row clusters. The updated centers are \[ \mu_1^{(1)}=(5,0), \qquad \mu_2^{(1)}=(5,2). \]

These centers are stable: the bottom points are closer to \((5,0)\) and the top points are closer to \((5,2)\), so the assignments do not change. Therefore Lloyd's algorithm terminates with objective value \[ 25+25+25+25 = 100. \] This is far worse than the global optimum \(4\).

Conclusion. k-means is an alternating-minimization algorithm with guaranteed descent, but the objective is nonconvex and admits poor local minima. Initialization matters.

Part XXI: PSD Approximation

21.1 The Projection Problem

Let \(A\) be a symmetric matrix. We want the positive semidefinite matrix \(X\) closest to \(A\) in Frobenius norm: \[ \min_{X\succeq 0}\|X-A\|_F^2. \] If \[ A=U\Sigma U^\top \] is the eigenvalue decomposition of \(A\), with \(\Sigma=\mathrm{diag}(\lambda_1,\dots,\lambda_n)\), then the answer is obtained by clipping the negative eigenvalues.

21.2 Best PSD Approximation by Eigenvalue Clipping

Theorem. The unique Frobenius-norm minimizer over PSD matrices is \[ X_\ast = U\,\max(\Sigma,0)\,U^\top, \] where \(\max(\Sigma,0)\) denotes the diagonal matrix with entries \(\max(\lambda_i,0)\).

Proof. The Frobenius norm is orthogonally invariant, so for any matrix \(X\), \[ \|X-A\|_F = \|U^\top XU-\Sigma\|_F. \] Therefore the original problem is equivalent to \[ \min_{Y\succeq 0}\|Y-\Sigma\|_F^2, \qquad Y=U^\top XU. \] Because congruence by an orthogonal matrix preserves positive semidefiniteness, \(X\succeq 0\) if and only if \(Y\succeq 0\).

Now expand the objective. Since \(\Sigma\) is diagonal, \[ \|Y-\Sigma\|_F^2 = \sum_{i=1}^n (y_{ii}-\lambda_i)^2 + \sum_{i\neq j} y_{ij}^2. \] The second term is always nonnegative and vanishes exactly when \(Y\) is diagonal. Hence any off-diagonal entry only increases the objective, so an optimizer must be diagonal.

Write \(Y=\mathrm{diag}(d_1,\dots,d_n)\). Since \(Y\succeq 0\), each diagonal entry must satisfy \(d_i\ge 0\). The problem now separates coordinatewise: \[ \|Y-\Sigma\|_F^2 = \sum_{i=1}^n (d_i-\lambda_i)^2, \qquad d_i\ge 0. \] For each \(i\), the minimizer of \((d_i-\lambda_i)^2\) over \(d_i\ge 0\) is \[ d_i= \begin{cases} \lambda_i & \lambda_i\ge 0,\\ 0 & \lambda_i\lt 0. \end{cases} \] Therefore the unique minimizer is \[ Y_\ast=\max(\Sigma,0). \] Transforming back gives \[ X_\ast = UY_\ast U^\top = U\,\max(\Sigma,0)\,U^\top. \] This proves the theorem.

Corollary. The cone of PSD matrices is the set obtained by zeroing out the negative part of the spectrum. In Frobenius norm, projection onto this cone acts eigenvalue-by-eigenvalue.

Example. If \[ A= \begin{bmatrix} 2 & 0\\ 0 & -3 \end{bmatrix}, \] then the nearest PSD matrix is \[ X_\ast= \begin{bmatrix} 2 & 0\\ 0 & 0 \end{bmatrix}. \] The negative eigenvalue \(-3\) is clipped to \(0\), while the positive eigenvalue \(2\) is preserved.
